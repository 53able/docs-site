HTML Anything

nexu-io/html-anything — Markdown・表・JSON などをローカルのコーディングエージェント CLI に渡し、単一ファイル HTML をストリーミング生成する Next.js アプリ。API キーは不要で、既存の CLI ログインセッションを再利用する。

1 — 概要
位置づけ

README が掲げるコンセプトは「Markdown は下書き、HTML が人間向けの最終成果物」である。エディタはブラウザ上で入力を受け取り、選択した SKILL.md テンプレートとデザインシステムをプロンプトに織り込み、特権プロセスがローカル CLI を起動して stdout の JSON 行を解析し、生成中の HTML を Server-Sent Events でクライアントへ流す。プレビューは sandbox 付き iframe に閉じ、ホストの Cookie や localStorage とは分離する。

3 — コーディング CLI
検出と起動

起動時に PATH を走査し、GUI から起動した Node プロセスで見落としがちな ~/.local/bin や /opt/homebrew/bin なども候補に含める。各 CLI には src/lib/agents/ の薄いアダプタ（argv.ts · invoke.ts 等）が対応し、stream-json 形式の stdout を POST /api/convert が SSE に橋渡しする。

4 — テンプレート（SKILL）
カタログ構造

各スキルは src/lib/templates/skills/<id>/ に SKILL.md（Claude Code の Skills 規約＋拡張 frontmatter）と example.html を持つ。ピッカーは mode（prototype / deck / frame / social / office / doc / mockup / vfx など）と scenario（design / marketing / engineering 等）でグルーピングする。

  • prototype — SaaS LP、ダッシュボード、データレポート、履歴書、長文ドキュメントなどの Web 系
  • deck — 横スワイプのキーノート系（Swiss International、Guizang Editorial、XHS 系など多数）
  • frame — Hyperframes / Remotion 向けの 1920×1080 フレーム列と遷移マーカー
  • social — X、小紅書、Spotify、Reddit 風カード
  • office — PM 仕様、エンジニアリングランブック、財務、人事、請求書、OKR など

README の「9 deliverable surfaces」は、上記モードをユーザー向けに束ねたマーケティング上の区分（記事・スライド・レジュメ・ポスター・SNS カード・Web プロト・レポート・Hyperframes 動画など）に対応する。

5 — 変換・プレビュー・エクスポート
ランタイム動作
  • SSE: エージェント stdout の JSON 行からテキストデルタを抽出し、クライアントが iframe srcdoc を増分更新する（中断・再プロンプト可能）。
  • フォーマット自動判定: Markdown / CSV / TSV / JSON / SQL / プレーンテキストを受け付け、表はブラウザ内パースのみで外部アップロードしない。
  • WeChat / Zhihu: juice による CSS インライン化と、数式の mjx-container を Zhihu 互換の data-eeimg プレースホルダへ変換する経路（README 記載）。
  • X 等の画像貼付: iframe を modern-screenshot で 2× PNG 化し ClipboardItem 経由でコンポーザへ渡す。
ローカル起動 
git clone https://github.com/nexu-io/html-anything.git
cd html-anything
pnpm install
pnpm dev
# → http://localhost:3000

以降はテンプレート選択 → 素材貼り付け → ⌘+Enter（README のクイックスタート）。

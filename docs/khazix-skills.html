Khazix Skills

KKKKhazix/khazix-skills — everyday AI skills, packaged as reusable agent instructions.

1. 概要
何を集めたリポジトリか

Khazix Skills は、作者が日々の作業で使い続け、実用性を確認してから公開した 5 つの Agent Skill 集。[Agent Skills](https://agentskills.io) 標準に沿い、Claude Code・Codex・OpenCode・OpenClaw などへそのまま載せられる。狙いは派手な万能化ではなく、特定の作業を「その人らしいやり方」まで含めて Agent に渡すこと。

5Agent Skills
AgentSkillsOpen standard
4+Agent hosts
MITLicense
中心思想

このリポジトリの面白さは、単なる「便利プロンプト集」ではないところ。知識同期、深い調査、長文の文体再現、AI 资讯の取得、ディスク分析といった、Agent が雑になりやすい領域に対して、判断基準と失敗パターンを明文化している。

読む観点: それぞれの Skill は「何をするか」だけでなく、「何をしないか」「どこから人間が判断するか」まで持っている。

3. ワークフロー
neat-freak の同期対象

neat-freak は、文書を「どこか 1 箇所に追記する」作業ではなく、読者の違う 3 レイヤーを分けて整える。ここを混ぜないので、次の Agent と人間の両方が迷いにくい。

01変更を把握今回の会話と差分を見て、新しい事実と古い前提を分ける。
02人向け docsREADME や docs に、外部読者が使う手順と背景を反映する。
03Agent 向け rootCLAUDE.md / AGENTS.md に、次回作業の判断材料を残す。
04記憶レイヤー跨セッションの好みや非自明な前提だけを短く更新する。
05矛盾チェック重複、期限切れ、相対日付、リンク切れを消す。
hv-analysis vertical

縦軸: 時間の深さ

起源、誕生、変化、意思決定、危機、転換点を追う。単なる年表ではなく、なぜその形になったのかをストーリーとして復元する。

hv-analysis horizontal

横軸: 同時代の広さ

現在の競合、代替、隣接概念、ユーザー評価を横並びで見る。そこで初めて、その対象の生態的位置が見える。

writer の品質ゲート
層見るもの失敗例役割
L1禁句・禁用標点・空泛工具名AI っぽい常套句が残る機械的に落とす
L2開頭、節奏、口語化、断裂報告書のように均一な文章文体の型を保つ
L3具体例、知識の出し方、文化的接続主張だけで人物や場面がない内容の厚みを作る
L4温度、独自性、姿勢、心流情報は正しいが人間味がない最後に「活人感」を見る
aihot の API ルーティング

aihot は「训练数据 ではなく今日の AI 事实」を取る Skill。ユーザーの言い方に応じてエンドポイントを切り替える。デフォルトは mode=selected の精选条目。「日报」と明示されたときだけ daily API を使う。

ユーザーの意図エンドポイント備考
「今天 AI 圈有什么」など広い質問/api/public/items?mode=selected&since=…24h / 7d など時間窗で絞る
「AI 日报」「今天的日报」/api/public/dailyUTC 日次の編集成品
「全部 / 完整 / 所有」/api/public/items?mode=all未精选も含む全量
「OpenAI 最近发的」/api/public/items?q=OpenAIサーバー側キーワード検索
必須: API 呼び出し時はブラウザ UA を付ける。default curl/X.Y UA は nginx の UA 黑名单で 403 になる。
storage-analyzer の三色分级

CleanMyMac のような固定ルールではなく、Agent が各パスを「删了会怎样」まで説明する。扫描は只读、删除はブラウザ上の二次确认後のみ。

🟢可自动清理纯缓存・临时文件。网页可一键移废纸篓或直接删。
🟡需人工判断离线视频・下载・项目代码。访达打开 + 可选移废纸篓。
🔴谨慎清理运行中 App 核心数据。打开文件夹のみ、删除按钮なし。
01scan.py只读 JSON 扫描。macOS / Windows 自動判別。
02server.py127.0.0.1 + 随机端口 + token。白名单付き削除 API。

design lesson

この Skill 集から学べるのは、Agent Skill は「手順」だけでなく「人格」「禁則」「検査方法」までまとめると強い、ということ。特に khazix-writer は、声を真似るという曖昧な作業を、HKR 选题、自检層、禁句、節奏ルールへ分解している。

実務への応用: 自分用 Skill を作るなら、良い例だけでなく「やると即ダメになる癖」を先に言語化すると、出力のブレが大きく減る。

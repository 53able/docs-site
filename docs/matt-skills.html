1 — 概要
何か

Matt Pocock Skills は、Matt Pocock（Total TypeScript 作者）が公開する開発ワークフロー向けエージェントスキルコレクション。 PRD 作成・TDD・インターフェイス設計・Git ガードレールなど、開発サイクルのあらゆる場面をカバーする 18 スキルを収録している。 agentskills.io オープン標準に準拠しており、Claude Code・Cursor・GitHub Copilot など主要プラットフォームで即座に使える。

各スキルは YAML フロントマター + Markdown 本文 の構成。フロントマターの name・description でエージェントが自動的に適切なスキルを選択し、本文のワークフローを実行する。 参照ドキュメント（deep-modules.md・tests.md など）を bundled resources として同梱することで、必要なコンテキストをスキル単体で完結させる設計。

2 — Planning & Design
コードを書く前に考える

問題・設計・要件を固めるためのスキル群。エージェントとのインタビューを通じて、実装前に設計を整理する。

plan write-a-prd
ユーザーインタビュー・コードベース探索・モジュール設計を経て PRD を作成し、GitHub Issue として提出する。
npx skills@latest add mattpocock/skills/write-a-prd
plan prd-to-plan
PRD をトレーサーバレット（縦断スライス）方式の多フェーズ実装プランに変換する。
npx skills@latest add mattpocock/skills/prd-to-plan
plan prd-to-issues
PRD を独立してアサイン可能な GitHub Issue に分解する（縦断スライスベース）。
npx skills@latest add mattpocock/skills/prd-to-issues
plan grill-me
プランや設計についてあらゆる決定分岐が解決されるまで容赦なくインタビューされる。思考の穴を潰す。
npx skills@latest add mattpocock/skills/grill-me
plan design-an-interface
並列サブエージェントで 3 つ以上の根本的に異なるインターフェイス設計を同時生成し比較する（Design It Twice）。
npx skills@latest add mattpocock/skills/design-an-interface
plan request-refactor-plan
ユーザーインタビューを経てリファクタリング計画を作成し、小さなコミット単位で GitHub Issue として提出する。
npx skills@latest add mattpocock/skills/request-refactor-plan
Design It Twice パターン

design-an-interface は John Ousterhout の「A Philosophy of Software Design」にある Design It Twice 思想を実装している。 最初のアイデアが最善である可能性は低い。並列サブエージェントに 根本的に異なる アプローチを採らせることで、比較可能な設計候補を素早く得られる。

3 — Development
コードを書く・直す・整える

実装・リファクタリング・バグ修正を行うためのスキル群。TDD の哲学とトレーサーバレット手法を中心に据える。

dev tdd
Red-Green-Refactor ループで機能追加・バグ修正を行う。1 テスト → 1 実装の縦断スライスで進む TDD スキル。
npx skills@latest add mattpocock/skills/tdd
dev triage-issue
コードベースを探索してバグの根本原因を特定し、TDD ベースの修正プランを GitHub Issue として提出する。
npx skills@latest add mattpocock/skills/triage-issue
dev improve-codebase-architecture
コードベースをアーキテクチャ観点で探索し、浅いモジュールを深くする・テスタビリティを上げる改善機会を特定する。
npx skills@latest add mattpocock/skills/improve-codebase-architecture
dev migrate-to-shoehorn
テストファイル内の as 型アサーションを @total-typescript/shoehorn に移行する。
npx skills@latest add mattpocock/skills/migrate-to-shoehorn
dev scaffold-exercises
セクション・問題・解答・解説を含む演習ディレクトリ構造を作成する（教材・ワークショップ向け）。
npx skills@latest add mattpocock/skills/scaffold-exercises
TDD の哲学 — 縦断スライスで進む

tdd スキルは「全テスト先行 → 全実装」のような 水平スライス を明示的に禁止している。 代わりに 1 テスト → 1 実装 → 繰り返し の縦断スライス（トレーサーバレット）を採用。 テストは「実装の形」ではなく「ユーザー向けの振る舞い」を検証すべきであるという立場から、 パブリック API を通じた統合スタイルのテストを推奨する。

4 — Tooling & Setup
環境を守る・整える

開発環境のセットアップとリスク低減のためのスキル群。一度設定すれば長期的に効く。

tool setup-pre-commit
Husky + lint-staged + Prettier + 型チェック + テスト実行を含むプリコミットフックをセットアップする。
npx skills@latest add mattpocock/skills/setup-pre-commit
tool git-guardrails-claude-code
Claude Code の PreToolUse フックで危険な git コマンド（push / reset --hard / clean / branch -D 等）を実行前にブロックする。
npx skills@latest add mattpocock/skills/git-guardrails-claude-code
git-guardrails がブロックするコマンド
コマンドブロック理由
git push（--force 含む）意図しないリモート書き換えを防止
git reset --hardコミット履歴の破壊的巻き戻しを防止
git clean -f / -fd追跡外ファイルの永久消去を防止
git branch -Dブランチの強制削除を防止
git checkout . / git restore .作業ツリーの一括破棄を防止

6 — クイックスタート
インストール方法

個別スキルのインストールには npx skills@latest add を使う。コレクション全体を取得する場合は git clone が最も速い。

個別スキルのインストール

# 例: TDD スキルだけ追加
npx skills@latest add mattpocock/skills/tdd

# 例: PRD 作成スキルを追加
npx skills@latest add mattpocock/skills/write-a-prd

# 例: Git ガードレールを追加
npx skills@latest add mattpocock/skills/git-guardrails-claude-code

リポジトリ全体をクローン

git clone https://github.com/mattpocock/skills.git

使い方の例（Claude Code）

インストール後、自然言語で指示するだけでエージェントが適切なスキルを自動選択する。

# PRD を作成したい
"このプロジェクトにユーザー認証機能を追加したい。PRD を書いて GitHub Issue にしてほしい。"
# → write-a-prd が起動、インタビュー後に Issue を作成

# TDD で実装したい
"tdd でユーザーログイン機能を実装してほしい。"
# → tdd が起動、Red-Green-Refactor ループを開始

# インターフェイスを設計したい
"この API のインターフェイスを複数パターン考えてほしい。"
# → design-an-interface が起動、並列サブエージェントで 3 案を生成
スコープの選択: git-guardrails-claude-code はインストール時に「このプロジェクトのみ（.claude/settings.json）」か「全プロジェクト（~/.claude/settings.json）」かを選択できる。

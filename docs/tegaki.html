Tegaki

手書き（tegaki）— 任意フォントを筆記アニメに変換する TypeScript モノレポ

1 — 概要
位置づけ

Tegaki は、手動で SVG パスを書かずに、フォントグリフからストローク順とタイミングを推定し、一筆ずつ描画するアニメーションを再生するライブラリだ（README）。npm パッケージ名は tegaki。ネイティブ依存はなく、任意のフォント用データはジェネレータ（または公式サイトのインタラクティブ生成）で用意する。

2 — モノレポ構成（AGENTS.md）
パッケージ一覧
パス / npm役割
packages/renderer（tegaki）公開パッケージ。共有エンジン＋ React / Svelte / Vue / Solid / Astro / WC / Vanilla / Remotion 向けアダプタ。
packages/generator（tegaki-generator）フォントからグリフデータを生成する CLI・内部ライブラリ（npm 非公開）。
packages/website（@tegaki/website）Astro + Starlight。ドキュメント・例・/tegaki/generator/ の対話型ジェネレータ。
開発コマンド: リポジトリでは bun dev（サイトウォッチ）、bun start（ジェネレータ CLI）、bun run test / bun typecheck / bun check などが AGENTS.md に整理されている。
3 — ジェネレータのパイプライン（グリフ単位）
処理の流れ

AGENTS.md に記載のステージを要約する。各グリフに対してフォント取得 → パース（opentype.js）→ ベジエ平坦化 → ラスタライズ → スケルトン化 → トレース → 幅計算 → ストローク順序 → JSON 出力、という流れだ。

flowchart TD
  A[フォント取得] --> B[Parse]
  B --> C[Bezier 平坦化]
  C --> D[ラスタライズ]
  D --> E[スケルトン化]
  E --> F[トレース]
  F --> G[幅・ストローク順]
  G --> H[JSON / バンドル]
4 — レンダラの入口
サブパス export

コアは tegaki/core の TegakiEngine。フレームワーク別は tegaki/react などのサブパス。素の tegaki エントリは後方互換のため React アダプタを再エクスポートする（AGENTS.md）。

import 例用途
tegaki/react など各フレームワーク用 TegakiRenderer
tegaki/coreVanilla の TegakiEngine
tegaki/wcWeb Components 登録
tegaki/fonts/<family>同梱プリ生成バンドル
5 — クイックスタート（README）
インストールと React 例 
npm install tegaki
import { TegakiRenderer } from 'tegaki';
import caveat from 'tegaki/fonts/caveat';

function App() {
  return (
    <TegakiRenderer font={caveat} style={{ fontSize: '48px' }}>
      Hello World
    </TegakiRenderer>
  );
}

テキストがストローク単位で自然なタイミングで描画される、と README が述べている。

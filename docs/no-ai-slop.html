01概要
目的

no-ai-slop は、下書きを明確で直接的な文章に整える Agent Skill。 目標は「人間らしさ」の演出ではなく、書き手がもともと持つ語彙、リズム、ユーモア、迷いを残しながら、 冗長なセットアップや誇張、機械的なリズムを減らすことにある。

AI 判定器ではない。 Detect は文章の出自を推測せず、読者が確認できる具体的な文体パターンだけを名前・引用・修正案とともに示す。
022つの仕事
Default job

Edit

下書き全体を読み、意味と声を守ったまま必要最小限の編集を入れる。

  • 全文を編集して返す
  • 「What changed」を短く添える
  • 最後に eval を通して失敗箇所を直す
Audit only

Detect

文章は書き換えず、見つかったパターンを検査結果として返す。

  • パターン名と該当箇所を示す
  • 短い修正方針を添える
  • 点数化・AI 著者判定はしない
03Edit の処理フロー
Step 01 全文を読む

文脈を切らず、下書き全体を先に把握する。

Step 02 核をつかむ

主張、読者、残すべき声の信号を内部で特定する。

Step 03 最小編集

意味を変えず、効果のある箇所だけを直す。

Step 04 eval 実行

原則、禁止語、パターン、最終読解を確認する。

Step 05 失敗を修正

評価で落ちた箇所に限定して再編集する。

Step 06 全文を返す

編集後の全文と短い変更要約を出力する。

不明点は一度だけ確認する。 読者や形式が分からない場合は質問を1つに絞る。狙いが曖昧なら、読者に何を考え、感じ、してほしいかを聞く。
04検出するパターン
分類 検出対象 編集の方向
構造
binary contrast negative listing throat-clearing colon reveal rhetorical setup
前置きを削り、主張や具体例から始める。
誇張
superficial analysis importance puffery weasel attribution fake-strong verbs
根拠のない重要性を外し、誰が何をしたかを書く。
声とリズム
synonym cycling dramatic fragments robotic rhythm fake-profound kicker summary recap
語彙を無理に散らさず、文の長さと終わり方を自然に戻す。
書式
emoji headings bold sprinkle unnecessary bullets header sprawl em-dash overuse
装飾を情報構造に合わせ、段落で読める内容は段落に戻す。
05評価基準と制約
最終チェック
  • 中心となる主張と意味が残っている
  • 能動態と直接的な動詞を使っている
  • 抽象語より具体的な詳細を優先している
  • 文のリズムに単調さがない
  • 声に合うユーモア、強さ、迷いが残っている
やらないこと
  • 事実、統計、例、意見を新しく作らない
  • 文章を無理に短く、無難に、上品にしない
  • Detect で文章を書き換えたり採点したりしない
  • AI が書いたと断定しない
  • 別の evaluator agent に最終確認を委ねない
語彙チェックの例

評価表は delve、foster、leverage、utilize、 facilitate、empower、streamline、robust、 cutting-edge、paradigm shift、game changer などを要注意語として挙げる。 単語を機械的に置換するのではなく、文が具体的で直接的かを最終的に読み直す。

06使い方と出力
Edit
Prompt
/no-ai-slop

[編集したい下書き]

返却: 編集後の全文 + 短い What changed。

Detect
Prompt
/no-ai-slop is this AI slop?

[検査したい文章]

返却: パターン名 + 引用 + 短い修正方針。本文は変更しない。

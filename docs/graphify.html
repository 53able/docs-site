1 — 概要
何か

graphify は AI コーディングアシスタント向けスキル。Claude Code・Codex・OpenCode・OpenClaw・Factory Droid で /graphify とタイプするだけで、コード・ドキュメント・論文・画像など任意のファイル群を読み取り、ナレッジグラフとして構造化する。知らなかったつながりが見えてくる。

71.5x トークン削減（大規模コーパス）
Multi コード / Docs / PDF / 画像
3.10+ Python バージョン
Local サーバー不要・完全ローカル

Andrej Karpathy の /raw フォルダワークフロー（論文・ツイート・スクリーンショット・コードをひとつの場所に集める）の問題を解決するツール。グラフは永続化されセッション跨ぎでクエリできる。すべてのエッジは EXTRACTED（根拠あり）/ INFERRED（推論・信頼スコア付き）/ AMBIGUOUS（要確認）で区別され、何が見つかったのか・何が推測なのかが常に分かる。

2 — パイプライン
処理フロー

7 段階のパイプラインで構成される。各ステージは独立したモジュールの単一関数として実装され、共有ステートなし・副作用なし（graphify-out/ 外へ書き出さない）の設計。

detect() → extract() → build_graph() → cluster() → analyze() → report() → export()
2 パス抽出

Pass 1 — AST（決定論的、LLM コスト不要）: tree-sitter でコードファイルを構文解析し、クラス・関数・インポート・コールグラフ・ドックストリング・rationale コメントを抽出。
Pass 2 — セマンティック（Claude サブエージェント・並列）: ドキュメント・論文・画像をサブエージェントが並列処理し、概念・関係・設計意図を抽出。SHA256 キャッシュにより変更ファイルのみ再処理。

クラスタリングはグラフトポロジー基盤。Leiden コミュニティ検出はエッジ密度で動作する。Claude が抽出したセマンティック類似エッジ（semantically_similar_to、INFERRED）がグラフ構造の中に含まれるため、エンベディングや別のベクター DB は不要。

抽出スキーマ 
{
  "nodes": [
    { "id": "unique_string", "label": "human name",
      "source_file": "path", "source_location": "L42" }
  ],
  "edges": [
    { "source": "id_a", "target": "id_b",
      "relation": "calls|imports|uses|semantically_similar_to|rationale_for|...",
      "confidence": "EXTRACTED|INFERRED|AMBIGUOUS",
      "confidence_score": 1.0 }
  ]
}

EXTRACTED エッジは confidence_score = 1.0 固定。INFERRED は 0.4–0.95 の範囲でモデルが根拠から個別推論。AMBIGUOUS は 0.1–0.3。

3 — インストール
前提条件

Python 3.10+ と、対応する AI コーディングアシスタント（Claude Code / Codex / OpenCode / OpenClaw / Factory Droid）のいずれか。

pip install graphifyy && graphify install
注: PyPI パッケージ名は graphifyy（名前返還手続き中）。CLI と スキルコマンドは graphify のまま。

4 — コマンド
スキルコマンド（AI アシスタント内） 
/graphify .                        # カレントディレクトリ → グラフ生成
/graphify ./raw                    # 指定フォルダ
/graphify ./raw --mode deep        # INFERRED エッジをより積極的に抽出
/graphify ./raw --update           # 変更ファイルのみ増分再抽出・マージ
/graphify ./raw --cluster-only     # 既存グラフを再クラスタリング
/graphify ./raw --no-viz           # HTML スキップ、レポート + JSON のみ
/graphify ./raw --obsidian         # Obsidian vault も生成（オプトイン）
/graphify ./raw --wiki             # コミュニティごとの wiki（index.md + 記事）
/graphify ./raw --watch            # ファイル変更を監視して自動同期
/graphify ./raw --svg              # graph.svg も出力
/graphify ./raw --graphml          # graph.graphml（Gephi / yEd 向け）
/graphify ./raw --neo4j            # cypher.txt 生成
/graphify ./raw --neo4j-push bolt://localhost:7687  # Neo4j に直接 Push
/graphify ./raw --mcp              # MCP stdio サーバー起動

/graphify add <url>               # URL 取得してコーパスに追加・グラフ更新
/graphify add <url> --author "Name"
/graphify add <url> --contributor "Name"

/graphify query "attention と optimizer を繋ぐものは？"

5 — 出力形式
graphify-out/ の構成
graph.html
インタラクティブグラフ。ノードをクリック・検索・コミュニティでフィルタ。ブラウザで開くだけ（サーバー不要）。vis.js 使用。5,000 ノード超では自動スキップ。
GRAPH_REPORT.md
プレーンテキストの監査レポート。God Nodes（最高次数ノード）・Surprising Connections（驚くべき接続）・Suggested Questions・トークンコストを含む。
graph.json
永続化グラフ（NetworkX node-link 形式）。セッション跨ぎでクエリ可能。GraphRAG ワークフローの入力として使用できる。
cache/
SHA256 ベースのキャッシュ。変更ファイルのみ再処理。大規模コーパスの更新コストを最小化。
obsidian/
オプトイン（--obsidian）。コミュニティごとの Markdown ノート + graph.canvas。Obsidian で vault として開くと構造レイアウトを確認できる。
cypher.txt / graph.graphml / graph.svg
Neo4j（--neo4j）/ Gephi・yEd（--graphml）/ SVG 埋め込み（--svg）向けの追加エクスポート形式。
グラフが提供するもの
  • God Nodes — 最も多くのノードと接続する概念。すべてがそこを経由する。
  • Surprising Connections — コード-論文間のエッジを最高スコアとした複合スコアでランク付け。各結果に平易な説明付き。
  • Suggested Questions — このグラフだから答えられる 4–5 個の質問。
  • Rationale Nodes — # NOTE: # WHY: # HACK: などのコメントや設計意図ドキュメントを rationale_for エッジ付きノードとして抽出。コードがなぜ書かれたかを保存。
  • Hyperedges — 共通プロトコルを実装する全クラス、認証フローの全関数など、2 点間エッジでは表現できない 3+ ノードのグループ関係。
  • Confidence Scores — INFERRED エッジは個別の確信度スコア（0.0–1.0）を持つ。何が推測かが定量的に分かる。
  • Token Benchmark — 実行後自動でコーパス全体 vs グラフのトークン比を算出・表示。

セキュリティ

外部入力はすべて graphify/security.py を経由してバリデーション:

  • URL → validate_url()（http/https のみ）+ _NoFileRedirectHandler（file:// リダイレクトをブロック）
  • 取得コンテンツ → サイズ上限・タイムアウト
  • グラフファイルパス → graphify-out/ 内に解決されることを検証
  • ノードラベル → 制御文字除去・256 文字上限・HTML エスケープ

コードファイルは tree-sitter AST のみでローカル処理。ファイル内容がネットワークに送信されるのは LLM セマンティック抽出（Docs / 論文 / 画像）のみ。テレメトリ・使用状況追跡はなし。

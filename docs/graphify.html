graphify

any input → knowledge graph → query / path / explain · Graphify-Labs · v0.9.14

1 — 概要
何か

graphify は AI コーディングアシスタント向けスキル兼 Python CLI。アシスタントで /graphify .（Codex は $graphify）を実行すると、コード・ドキュメント・PDF・画像・動画などを読み取り、ナレッジグラフとして構造化する。ベクトル索引ではなく、エッジを辿る実グラフ。

0.497 LOCOMO recall@10
$0 コード AST の LLM コスト
20+ 対応アシスタント
3.10+ Python

コードは tree-sitter AST で完全ローカル・決定論的（LLM 不要）。Docs / PDF / 画像などはアシスタントのモデル（または設定した API）でセマンティック抽出。各エッジは EXTRACTED（ソースに明示）/ INFERRED（解決・推論）/ AMBIGUOUS（要確認）で区別する。

公式パッケージ名は graphifyy（二重 y）。PyPI 上の他 graphify* は非公式。CLI コマンドは graphify。Y Combinator S26。
30 秒スタート 
uv tool install graphifyy      # または: pipx install graphifyy
graphify install               # スキルをアシスタントに登録
# アシスタント内で:
/graphify .

出力は graphify-out/ に 3 ファイル: graph.html（ブラウザで操作）・GRAPH_REPORT.md（ハイライト）・graph.json（再クエリ用）。

3 — パイプライン
ライブラリ段階

各ステージは独立モジュールの単一関数。共有ステートなし。副作用は graphify-out/ 内への書き出しに限定（ARCHITECTURE.md）。

detect() → extract() → build_graph() → cluster() → analyze() → report() → export()
3 パス抽出
  • Pass 1 — コード構造（無料・API なし）: tree-sitter AST。クラス・関数・imports / calls / inherits など。コードのみのコーパスでは Pass 3 をスキップ。
  • Pass 2 — 動画・音声（ローカル）: faster-whisper。トップ god nodes をプロンプトにシード。キャッシュあり。
  • Pass 3 — Docs / 論文 / 画像（セマンティック）: アシスタントのサブエージェント（または headless backend）が JSON 断片を並列出力しマージ。
コミュニティ検出は Leiden。埋め込みやベクトル DB は不要。semantically_similar_to などのエッジがトポロジーに直接効く。
主要モジュール
モジュール関数入力 → 出力

抽出スキーマ 
{
  "nodes": [
    { "id": "unique_string", "label": "human name",
      "source_file": "path", "source_location": "L42" }
  ],
  "edges": [
    { "source": "id_a", "target": "id_b",
      "relation": "calls|imports|uses|...",
      "confidence": "EXTRACTED|INFERRED|AMBIGUOUS" }
  ]
}

EXTRACTED は confidence 1.0。INFERRED はルーブリック（0.95〜0.55）。AMBIGUOUS はレポートで要レビュー。

4 — インストール
パッケージとスキル登録 
# 推奨（隔離環境）
uv tool install graphifyy
# 代替: pipx install graphifyy  /  pip install graphifyy

graphify install                 # ユーザー向けスキル登録
graphify install --project       # カレントリポジトリへ（例: .claude/skills/graphify/）
graphify install --platform codex
uvx --from graphifyy graphify install   # パッケージ名は graphifyy
graphify: command not found → uv tool update-shell または pipx ensurepath のあとシェル再起動。~/.local/bin が PATH に無いことが多い。

5 — コマンド
スキル・CLI（よく使うもの） 
/graphify .                        # カレントをグラフ化
/graphify ./docs --update          # 変更ファイルのみ再抽出
/graphify . --cluster-only         # クラスタのみ再実行
/graphify . --no-viz               # HTML スキップ
/graphify . --wiki                 # グラフから markdown wiki

graphify query "what connects auth to the database?"
graphify path "UserService" "DatabasePool"
graphify explain "RateLimiter"

/graphify add https://arxiv.org/abs/1706.03762
graphify hook install              # コミット後に自動再ビルド
graphify merge-graphs a.json b.json
graphify export callflow-html      # Mermaid コールフロー HTML

graphify prs                       # PR ダッシュボード
graphify prs 42                    # PR #42 のグラフ影響
graphify prs --triage              # レビューキュー順位付け
graphify prs --conflicts           # コミュニティ共有の衝突リスク
.graphifyignore とチーム

.gitignore は自動尊重。.graphifyignore は同じ構文で追加除外（マージ時は後者が優先）。チームは graphify-out/ をコミットし、各自がすぐクエリできるようにする。フック導入で AST のみの自動再ビルドと graph.json のマージドライバが有効になる。

6 — 出力とクエリ
graphify-out/
graph.html
クリック・検索・コミュニティフィルタ可能なインタラクティブ図。サーバー不要。
GRAPH_REPORT.md
God nodes・驚く接続・Suggested questions・確信度の監査レポート。
graph.json
NetworkX node-link。再読み込みなしで query / path / explain / MCP。
cache/ · callflow · Obsidian
SHA256 キャッシュ。オプションで callflow HTML、Obsidian vault、Neo4j / GraphML / SVG。

MCP サーバー 
python -m graphify.serve graphify-out/graph.json
python -m graphify.serve graphify-out/graph.json \
  --transport http --host 0.0.0.0 --port 8080 --api-key "$SECRET"

ツール例: query_graph / get_node / shortest_path / list_prs / triage_prs。共有ホストでは --host 0.0.0.0 と --api-key を併用。

7 — 入力・プライバシー
扱える入力（要約）
種別内容
コード多数の tree-sitter 文法（Python / TS / JS / Go / Rust / Java / C/C++ / Ruby / C# / Kotlin / Swift / …）。オプションで Terraform / Pascal / DreamMaker 等
Docs.md / .mdx / HTML / YAML 等。Markdown リンク・wikilink → references
Office / Google[office] / [google] + gws、--google-workspace
PDF / 画像 / 動画セマンティックまたはローカル転写（[video]）
マニフェスト / MCPパッケージ依存・MCP サーバー設定をノード化
データの行き先
  • コード — マシン内 AST のみ。コードのみなら API キー不要でオフライン可。
  • 動画・音声 — faster-whisper でローカル転写。
  • Docs / PDF / 画像 — スキル利用時は IDE セッションのモデルへ。headless graphify extract は設定した backend（Gemini / Kimi / Claude / OpenAI / Ollama / Bedrock 等）へ。
  • テレメトリなし。クエリログはオプトイン（既定オフ、または GRAPHIFY_QUERY_LOG_DISABLE=1）。
外部入力は security.py 経由（http/https のみ、file:// リダイレクト遮断、サイズ上限、グラフパスは graphify-out/ 内、ラベルのサニタイズ）。

Claude Context

zilliztech/claude-context — コードベースをベクトル化し、MCP 経由で意味検索コンテキストをコーディングエージェントへ渡す OSS

1 — 概要
何を解決するか

Claude Context は、巨大リポジトリでも「関連コードだけ」をセマンティック検索で取り出し、Claude Code やその他の MCP 対応エージェントのコンテキストに載せるための仕組みだ。

ディレクトリ丸ごと読み込む代わりに、分割・埋め込み・ベクトルストアに載せたあと、クエリに近いチャンクだけを返す。コストとトークンを抑えつつ、発見性を上げるのが狙い。

2 — モノレポ構成
レイヤー

公式リポジトリは packages/ 配下にコアロジックと配布物を分けたモノレポ。evaluation/ で検索品質の実験、examples/ で最小利用例、docs/ で詳細ドキュメントが置かれている。

packages/core

AST ベースの分割、同期（Merkle 等）、埋め込み抽象、Milvus REST / SDK クライアント。エージェントに依存しない中核。

packages/mcp

MCP サーバー実装。スナップショット取得・検索ツールをエージェントへ公開。Node で npx 実行が前提の導線。

packages/vscode-extension

エディタ内の意味検索 UI とコマンド。Marketplace からインストール可能な製品版の入り口。

packages/chrome-extension

ブラウザ側の補助（GitHub 上のコード閲覧など）とストレージ連携。開発用スタブでオフライン検証も可能。

3 — データの流れ
インデックスから検索まで

ローカルのワークスペースや指定パスを走査し、無視パターンでファイルを落としたうえでチャンク化する。各チャンクを埋め込みモデルへ通し、ベクトル DB に upsert。エージェントがクエリを投げると近傍検索でチャンクとメタデータが返る。

flowchart LR subgraph ingest["取り込み"] A["ソースツリー"] --> B["分割・フィルタ"] B --> C["Embedding API"] C --> D[("Milvus / Zilliz")] end subgraph serve["配信"] D --> E["MCP ツール"] E --> F["コーディングエージェント"] end
非同期インデックス — 大規模リポジトリではバックグラウンド同期や増分更新の設計が重要。公式の docs/dive-deep/ にワークフローとファイル包含ルールの詳細がある。
4 — MCP セットアップ
Claude Code 向けの典型例

ベクトル DB のエンドポイントとトークン、埋め込み用の API キーを環境変数で渡し、npx で MCP パッケージを起動する。値はプレースホルダに置き換えて使うこと。

claude mcp add
claude mcp add claude-context \
  -e OPENAI_API_KEY=sk-your-openai-api-key \
  -e MILVUS_ADDRESS=your-zilliz-cloud-public-endpoint \
  -e MILVUS_TOKEN=your-zilliz-cloud-api-key \
  -- npx @zilliz/claude-context-mcp@latest

詳細は Claude Code MCP ドキュメント と、zilliztech/claude-context の README を参照。

6 — クライアント対応
設定ファイルの差

README では Claude Code のほか、OpenAI Codex CLI（TOML）、Gemini CLI、Qwen Code、Cursor、Void など、MCP クライアントごとの設定断片が並んでいる。キー名やトップレベル構造がクライアントで微妙に異なるので、そのままコピペせず公式の MCP 仕様と突き合わせること。

  • Cursor は ~/.cursor/mcp.json またはプロジェクト直下の .cursor/mcp.json が一般的。
  • Codex CLI は mcp_servers というトップレベルキーに注意（README 内に明記あり）。
  • 起動タイムアウトを延ばす例（startup_timeout_ms）も掲載されている。

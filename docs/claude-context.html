Claude Context

AI coding agents need less blind crawling and more semantic memory.

1 — 概要
何を解決するか

Claude Context は、コードベースをベクトルデータベースへインデックスし、Claude Code や Cursor などの AI コーディングエージェントへ意味検索を渡す MCP プラグイン。巨大なディレクトリを毎回読ませる代わりに、関連コードだけを検索してコンテキストへ差し込む。

MCPagent integration
BM25 + vectorhybrid search
Node 20+runtime
~40%token reduction claim
注意: README は Node.js >=20.0.0 かつ <24.0.0 を要求している。Node 24 系は明示的に非互換として扱われている。
2 — クイックスタート
最小セット

最短ルートは MCP サーバーをクライアントへ登録し、埋め込み API と Milvus / Zilliz Cloud の接続情報を渡すこと。

claude mcp add claude-context \
  -e OPENAI_API_KEY=sk-your-openai-api-key \
  -e MILVUS_ADDRESS=your-zilliz-cloud-public-endpoint \
  -e MILVUS_TOKEN=your-zilliz-cloud-api-key \
  -- npx @zilliz/claude-context-mcp@latest
Step 1Claude Code などの MCP クライアントを開く
Step 2コードベースを絶対パスでインデックス
Step 3進捗ステータスを確認
Step 4自然言語で関連コードを検索
Step 5取得結果をエージェントの判断材料にする
3 — MCPツール
4つの入口
ツール必須入力役割追加指定
index_codebasepathコードベースをインデックスするforce, splitter, customExtensions, ignorePatterns
search_codepath, query自然言語クエリで検索するlimit は最大 50、extensionFilter で絞り込み
clear_indexpath対象コードベースの検索インデックスを消す同じ絶対パス単位で扱う
get_indexing_statuspath進捗、完了、失敗、未登録状態を返す完了後にファイル数とチャンク数が見える
stdioの制約

MCP プロトコルの stdout を壊さないよう、起動直後に console.log と console.warn を stderr へ退避している。MCP サーバーらしい、地味だけど重要な実装ポイント。

@zilliz/claude-context-core

Context、splitter、embedding、vectordb、sync を re-export する中核。AST 分割、ハイブリッド検索、Merkle ベースの増分同期を担当。

@zilliz/claude-context-mcp

MCP の tool schema と handler を公開。設定生成、embedding 初期化、snapshot 管理、background sync をまとめる。

IDE / browser surfaces

VS Code 拡張、Chrome 拡張、Python ブリッジ、examples があり、MCP 以外の入口も用意されている。

6 — インデックス対象
Core rule

対象ファイルは「対応拡張子の和集合」から「除外パターンの和集合」を引いたもの。デフォルト拡張子に MCP 引数と環境変数を足し、除外パターンも同じく複数ソースを合成する。

Final Files = All Supported Extensions - All Ignore Patterns
Supported extensions

TypeScript、JavaScript、Python、Java、C/C++、C#、Go、Rust、PHP、Ruby、Swift、Kotlin、Scala、Markdown、Notebook などをデフォルト対応。

Ignore sources

build 出力、依存ディレクトリ、IDE設定、VCS、cache、log、環境ファイル、minified bundle などを既定で除外。さらに MCP 引数、環境変数、プロジェクト/ユーザー側の ignore ファイルで拡張できる。

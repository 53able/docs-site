n.eko

Docker 内の仮想ブラウザを WebRTC でストリーミング。複数人で同時アクセス・共同閲覧・ウォッチパーティ。

1 — 概要
何か

n.eko（Neko）は、Docker コンテナ内で動く仮想ブラウザ（またはデスクトップ／アプリ）を、WebRTC でストリーミングするセルフホスト型のツール。ブラウザだけでなく VLC や XFCE/KDE などのデスクトップ環境も動かせ、複数ユーザーが同時に同じ画面にアクセス・操作できる。rabb.it のオープンソース代替として始まり、ウォッチパーティ・共同プレゼン・サポート／教育・プライバシー用途などで使われる。

WebRTCストリーミング
Dockerコンテナ
複数人同時アクセス
音声付き低遅延

Guacamole や noVNC のようなクライアントレスゲートウェイと比べ、WebRTC によるスムーズな動画と内蔵音声、複数参加者での操作共有が特徴。

2 — ユースケース
複数人
  • ウォッチパーティ — 動画をみんなで同時視聴・チャット・絵文字（giggl.app / Hyperbeam の OSS 代替）。
  • インタラクティブプレゼン — 画面共有だけでなく、参加者が画面を操作できる。
  • 共同ツール — ブレスト・共同ブラウジング・リモートでコードデバッグ。
  • サポート／教育 — 決まった環境で相手の操作をガイド。
  • 埋め込み — Web アプリに仮想ブラウザを埋め込み（Hyperbeam API の OSS 代替）。neko-rooms で API からルーム発行可能。
単体利用
  • 個人ワークスペース — コンテナ内アプリ／デスクトップをストリーミング（Kasm 的）。
  • 永続ブラウザ — クッキーをサーバ側に持ち、どこからでも同じブラウザ状態（Mighty 的）。接続終了後もホストに状態は残らない。
  • 捨てブラウザ — Tor Browser + VPN で匿名性。コンテナで OS フィンガープリントやブラウザ脆弱性のリスクを隔離。
  • RTMP 配信 — ルーム内容を Twitch/YouTube などへブロードキャスト。nginx-rtmp でファイル保存すればセッション録画も可能。
  • ジャンプホスト — VPN なしで内部アプリへ安全にアクセス。
  • 自動化 — Playwright / Puppeteer をコンテナ内に入れ、操作を能動的に傍受しながら自動実行。
3 — 構成
リポジトリ

m1k1o/neko はモノレポで、主要なディレクトリは次のとおり。

ディレクトリ説明
serverWebRTC シグナリング・セッション管理・認証などのバックエンド。
clientブラウザ側のフロントエンド（接続・映像表示・操作送信）。
runtimeコンテナ内で動く X セッション・画面キャプチャ・入出力の橋渡し。
webpageユーザーがアクセスする Web UI（ログイン・ルーム入室など）。
appsFirefox / Chromium / XFCE など、用途別の Docker イメージ定義。

設定は config.yml で行い、解像度・メンバー（admin/user パスワード）・セッション（クッキー等）を指定する。

対応ブラウザ・アプリ

公式イメージ例: Firefox, Tor Browser, Waterfox, Chromium, Google Chrome, Ungoogled Chromium, Edge, Brave, Vivaldi, Opera。その他 XFCE / KDE / Remmina / VLC などは m1k1o/neko-apps を参照。

4 — インストール
Docker

Docker と Docker Compose が使える環境で、公式イメージを pull して起動する。

docker pull ghcr.io/m1k1o/neko/firefox
# または docker-compose.yaml で services に neko を定義して up

ルーム管理や HTTPS 付きの Zero-knowledge インストールは neko-rooms を利用する。

ドキュメント: 移行（V2→V3）・クイックスタート・インストール・設定・FAQ・トラブルシュートは neko.m1k1o.net を参照。

Kami

tw93/kami の要約ページ。スローガン、三部作での位置、テンプレ八種、デザイン制約、導入コマンドを並べた。

概要
スローガンと役割

Good content deserves good paper. Kami は UI フレームワークではなく、紙面のトーンとレイアウトを固定する制約セットだ。パーチメント地（#f5f4ed）とインクブル（#1B365D）だけで八つの文書タイプを回す。

英語・中国語テンプレが第一級。日本語は CJK 経路でベストエフォート。出す前に目視で崩れを見る想定。

なぜ Kami か
問題設定

下書き速度は十分なのに、セッションごとにフォントと余白が変わると読みにくい。Kami は再現しやすいルールを薄く積む。複雑さはテンプレ側に閉じる。

brand.md の順位: あなたのプロンプトが先。続いて編集判断、習慣メモ、YAML 既定、ビルトイン既定。任意の brand.md は欠けた部分だけ埋め、プロンプトを上書きしない。
三部作のなかの Kami
Kaku · Waza · Kami

作者が切り分けた三ツール。名前だけ日本語で揃っている。

  • Kaku（書く） — コードを書くための高速ターミナル環境（Kaku）
  • Waza（技） — 習慣と反復を鍛える（Waza）
  • Kami（紙） — 完成した思考を、紙面として届ける（本ページの主役）
アイデアが紙になるまで
処理の流れ

自然言語で依頼すると、スキルが言語・用途・テンプレを選び、必要なら素材確認や蒸留（distill）を挟んでから、HTML ベースの組版へ進みます。スライドは WeasyPrint 経由の PDF を既定に、編集可能な PPTX が要望にだけ応じます。

デザインの約束
キャンバスとタイポのルール
要素ルール（要約）
キャンバスパーチメント #f5f4ed。純白は使わない
アクセントインクブル #1B365D のみ。二色目の彩度は足さない
ニュートラル黄みがかったグレー系。青みの強いクールグレーは避ける
セリフ見出し 500、本文 400。安易なフェイクボールドを避ける
行間タイトルはきゅっと、本文は読みやすい密度に
影リングかごく薄い陰のみ。ドロップシャドウは控えめに

言語ごとにセリフを一本化（中国語: TsangerJinKai02、日本語: YuMincho、英語: Charter など）。細部は references/design.md と CHEATSHEET.md。

はじめの一歩
Claude Code から

Claude Code なら一行。

npx skills add tw93/kami -a claude-code -g -y

汎用エージェント向けには -a '*'、Claude Desktop ではリリースの ZIP を Skills にそのままアップロードする方法もあります。更新は同じ URL から差し替えるだけ、とのことです。

任意: ~/.config/kami/brand.md に名前・トーン・通貨ロケールなどを書いておくと、曖昧な依頼のときだけ参照される。雛形は brand.example.md。

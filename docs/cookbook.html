Cursor Cookbook

cursor/cookbook — Cursor SDK をアプリ・CLI・クラウド運用へ接続する小さな実例集

1 — 概要
何を学ぶリポジトリか

Cursor Cookbook は、Cursor の coding agent を TypeScript から動かす @cursor/sdk の実例集。単発のローカル実行から、Ink 製 CLI、Next.js のプロトタイピング UI、Cloud Agent を一覧・作成する Kanban まで、抽象説明ではなく「動く小品」で SDK の使い方を示している。

中心にある概念はシンプル。Agent.create() で実行場所とモデルを決め、agent.send() でプロンプトを送り、run.stream() と run.wait() で進行中のイベントと結果を扱う。

Agent.create
local workspace or cloud repositories
agent.send
prompt, model, force, run options
run.stream
assistant, thinking, tool, task, status
Cloud API
agents, repositories, artifacts, PR links

2 — 4つのサンプル
sdk/quickstart

最小構成。現在の作業ディレクトリにローカル agent を作り、固定プロンプトを送り、assistant の text block だけを標準出力へ流す。

sdk/coding-agent-cli

ターミナル実装。one-shot prompt、stdin prompt、Ink TUI、モデル選択、local/cloud 切り替え、キャンセル、セッションリセットを扱う。

sdk/app-builder

ローカルのアプリ生成ループ。API key を受け取り、Vite React のプレビュー環境を作り、チャット UI から agent に編集させる。

sdk/agent-kanban

Cloud Agent の運用 UI。agent をステータスやリポジトリでグループ化し、artifact preview と新規 cloud agent 作成を提供する。

3 — SDKの基本フロー
Local agent

quickstart と coding-agent-cli の基本はローカル実行。local: { cwd } に作業ディレクトリを渡すことで、agent はそのワークスペースを前提に回答・編集・検証できる。

sdk/quickstart/src/index.ts
const agent = await Agent.create({
  apiKey: process.env.CURSOR_API_KEY,
  name: "SDK quickstart",
  model: { id: process.env.CURSOR_MODEL ?? "composer-2" },
  local: { cwd: process.cwd() },
})

const run = await agent.send("Explain this project in one paragraph.")

for await (const event of run.stream()) {
  if (event.type !== "assistant") continue
  // text block を stdout へ流す
}
Cloud agent

coding-agent-cli の TUI と agent-kanban は、クラウド実行も扱う。Cloud 側では repository URL、branch、PR 作成、artifact など「実行結果を運用する」ための情報が前面に出る。

  • /local と /cloud で実行先を切り替える。
  • /model で Cursor.models.list() の候補を選ぶ。
  • Agent.create({ cloud: { repos } }) で Cloud Agent を作る。
  • artifact は認証付きのローカル API route 経由で preview する。

5 — 実行時の注意
Secrets

README は共通して CURSOR_API_KEY を必要条件にしている。Web UI 系の2例は key をローカル設定ファイルへ保存できるが、共有サービスとして公開する前提ではない。

注意: App Builder の README は、認証・ユーザー別ストレージ・より強い secret handling を足さずに共有公開しないよう明記している。
Runtime
  • CLI 系は Node.js 22 以上を要求する。
  • Next.js 系はローカルデモとして pnpm dev で起動する。
  • App Builder の agent 指示では、既存 preview server が壊れていない限り別の長時間 dev server を起動しない。
  • Agent Kanban の repository listing は Cloud Agents API の rate limit を考慮し、短時間メモリキャッシュを使う。

CLI のコマンド面

coding-agent-cli は、one-shot と TUI の分岐を入口で決める。プロンプトがあれば即実行、stdin があれば stdin を prompt として実行、TTY なら Ink のフルスクリーン TUI に入る。

TUI slash commands
/help   Show available commands
/local  Run future prompts in the local workspace
/cloud  Run future prompts in Cursor cloud
/model  Open a picker with available Cursor models
/reset  Start a fresh agent and clear context
/exit   Exit the TUI

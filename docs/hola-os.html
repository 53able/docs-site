holaOS

Open Agent Computer · holaboss-ai/holaOS · TypeScript / Electron

1 — 概要
何をするプロジェクトか

holaOS は、デジタル作業を人間と AI エージェントが同じブラウザ・ファイル・アプリ上で分担するためのオープンソース環境。メモリ・実行・ゴールを一つのワークスペースに閉じ、文脈が毎回リセットされない「共有コンピュータ」として設計されている。

2 — システム構成
データの流れ（概念図）

ユーザーが触れる Electron デスクトップが、ローカルの api-server と対話し、エージェント実行は harness-host と harnesses 経由でツール・スキルに接続する。ワークスペース状態は state-store 側で永続化される想定の分割になっている。

4 — ランタイムの焦点
api-server と周辺

runtime/api-server には大規模な app.ts を中心に、ターン処理・メモリ書き戻し・統合（Composio 等）・ワーカー・ブラウザツール橋渡しなどのモジュールが集約されている。ハーネス側は runtime/harnesses にポリシー・ネイティブ Web 検索・ワークスペーススキル等のファイルがまとまり、harness-host が実行時の境界として機能する。

  • メモリとリコール: 埋め込み・マニフェスト・バックフィル系ワーカーなど、長期コンテキストを前提としたファイル群が存在する。
  • セッションとタスク: チェックポイント、TODO、サブエージェント、ターミナルセッションなど「作業の継続性」を支えるコンポーネントが並ぶ。
  • 進化（evolve）: スキルレビューやポストラン処理など、エージェント出力を次の行動に変換するパイプラインがコードベース上に配置されている。
5 — デスクトップ（Electron）
UI とブラウザペイン

desktop/electron 以下にメインプロセス、プリロード、ブラウザペイン（タブ状態・HTTP サービス・インポート等）がまとまっている。Renderer 側は Vite + TypeScript で、ダッシュボード・マーケットプレイス・公開フローなどの UI モジュールが desktop/src に配置される構成になっている。

ローカル開発の入口

リポジトリ README の手順では、ルートでデスクトップ依存を入れ、desktop/.env を整えたうえでランタイムのローカル準備と型検査を行い、必要なら開発サーバを起動する流れが案内されている。詳細手順は INSTALL.md および desktop/README.md を正とする。

6 — SDK レイヤー
公開クライアント

sdk/app-sdk には Kubb 等で生成された型・React フック・Zod スキーマが大量に含まれ、マーケットプレイスやワークスペース操作の契約が機械可読になっている。sdk/runtime-client はランタイム API への薄いクライアント層、sdk/bridge はデスクトップとランタイムの橋渡し（DB・プレゼンテーション・ターンコンテキスト等）を担う。

7 — クイックスタート
ワンライナー（macOS / Linux / WSL）

README に記載の公式インストーラ。Git / Node が未導入の場合もスクリプト側で整備する想定になっている。

curl -fsSL https://raw.githubusercontent.com/holaboss-ai/holaOS/main/scripts/install.sh | bash -s -- --launch

コーディングエージェントへの一括指示

README に掲載の「ワンラインエージェントセットアップ」用テキストは、スクリプト URL と desktop:* npm スクリプトの順序をエージェントに明示する用途。長文はリポジトリの README を参照。

Run the holaOS install script from https://raw.githubusercontent.com/holaboss-ai/holaOS/main/scripts/install.sh. Follow README for desktop:install, env setup, prepare-runtime:local, and typecheck; only start desktop:dev if asked for --launch.

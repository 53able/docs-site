OpenScreen

Electron + React + Vite — 画面録画とデモ編集のオープンソースアプリ

1 — 概要
プロジェクトの位置づけ

OpenScreen は、有料のスクリーン録画・デモ制作ツールに近い体験を、シンプルな機能セットで無料・オープンソースとして提供するデスクトップアプリ。公式 README の位置づけでは、プロダクトのウォークスルーやチュートリアル向けの録画・ズーム・トリミング・書き出しに焦点がある。

2 — 技術スタック
ランタイムと主要依存

デスクトップシェルは Electron、UI は React 18 と Vite 7。タイムラインは dnd-timeline、キャンバス合成やエフェクトに PixiJS と関連フィルタ、モーションに GSAP / Motion。メディア処理・書き出しには mediabunny、mp4box、web-demuxer などが列挙されている。品質管理に Biome、テストに Vitest と Playwright が入る。

4 — 導入と制限
配布物と OS ごとの注意

インストーラは GitHub Releases から入手する想定。macOS では未署名ビルムのため Gatekeeper 回避の手順が README に記載されている。

macOS（README 抜粋）

インストール後にターミナルで quarantine 属性を削除し、その後システム設定で画面収録・アクセシビリティを許可する流れ。

xattr -rd com.apple.quarantine /Applications/Openscreen.app

Linux

AppImage を実行権付与して起動。サンドボックスエラー時は --no-sandbox が案内されている。

chmod +x Openscreen-Linux-*.AppImage
./Openscreen-Linux-*.AppImage
システム音声まわりの制限
OS条件・挙動
macOS13 以上。14.2 以降は音声キャプチャの許可プロンプト。12 以下はシステム音声非対応（マイクは可）。
WindowsREADME 上は追加要件なしで動作と記載。
LinuxPipeWire 推奨（Ubuntu 22.04+、Fedora 34+ 等）。古い PulseAudio のみではシステム音声が取れない場合あり。
Linux 表示サーバ: 上流の electron/main.ts では Wayland 検出時に ozone-platform=wayland と PipeWire キャプチャ用フラグを付与している。
5 — 実行時構成
メインプロセスとレンダラーの関係

メインプロセスは registerIpcHandlers で IPC を登録し、エディタ・ソース選択・カウントダウン HUD など複数の BrowserWindow を生成する。レンダラーは Vite ビルドの React アプリで、録画データはユーザーデータ配下の recordings に保存される設計（定数 RECORDINGS_DIR）。

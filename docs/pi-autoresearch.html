pi-autoresearch

davebcn87 / MIT — pi 向けの自律ベンチマーク・最適化ループ

1 — 概要
位置づけ

pi-autoresearch は、ターミナルで動く AI エージェント pi 用の拡張だ。アイデアを試し、指標で測り、改善だけを残して退行は戻し、その繰り返しを自動で回すワークフローを提供する（README）。karpathy/autoresearch に着想を得ている。

3 — ツールとスラッシュコマンド
Extension tools
ツール説明（README 抜粋）
init_experimentセッション設定（名前・指標・単位・向き）を一度だけ初期化
run_experiment任意コマンドを実行しウォールクロック時間を計測、出力を取得
log_experiment結果を記録、自動コミット、ウィジェットとダッシュボードを更新

/autoresearch サブコマンド

サブコマンド説明
/autoresearch <text>モード突入 or autoresearch.md があれば文脈付きで再開
/autoresearch offモード解除。ランタイム状態はクリア、jsonl は保持
/autoresearch clearautoresearch.jsonl 削除・全リセット
/autoresearch exportブラウザでライブダッシュボード

ショートカット: Ctrl+X でウィジェット拡張の切替、Ctrl+Shift+X で全画面オーバーレイ（README）。

5 — セッションファイル
README に列挙されたファイル
ファイル役割
autoresearch.md目的・指針・試したことのセッション文書。エージェントが単独で再開可能
autoresearch.shベンチマーク用シェル。METRIC name=number 行を出力
autoresearch.checks.sh（任意）ベンチ通過後のテスト・型・lint 等。失敗時は keep をブロック
autoresearch.jsonlランごとに追記されるログ
6 — ループと finalize
流れ

/skill:autoresearch-create で目標・コマンド・指標・スコープを確定し、ブランチ作成・autoresearch.md / autoresearch.sh 作成・ベースライン実行後にループ開始（README）。エージェントは編集 → コミット → 実験 → ログ → keep/revert を自律的に繰り返す。

/skill:autoresearch-finalize は autoresearch.jsonl を読み、保持した実験を論理単位でグループ化し、マージベースから独立ブランチを生成する（README）。

7 — 信頼度スコア（README）
MAD ベース

同一セグメント内の指標について 中央値絶対偏差（MAD） をロバストなノイズ推定に使い、信頼度は「最良改善の絶対値 / MAD」として解釈する（README）。3 ラン以上で表示。エージェントへの助言であり、自動破棄はしない。

信頼度意味（README）
2.0× 以上改善はおそらく実質的
1.0–2.0×ノイズより上だが差は小さい
1.0× 未満ノイズ内 — 再実行の検討
8 — 任意設定（README）
autoresearch.config.json

pi のセッションディレクトリに置き、workingDir（操作対象ディレクトリの上書き）と maxIterations（実験回数の上限）を指定できる（README）。

コスト: README はループのトークン消費に注意し、プロバイダの API 上限と maxIterations の併用を推奨している。

Archon

AI コーディングエージェント向けワークフローエンジン（YAML DAG・worktree 分離・Orchestrator）

1 — 概要
位置づけ

Archon は開発プロセス（計画・実装・検証・レビュー・PR 作成など）を YAML ワークフローとして定義し、同じ手順を何度でも再現できるようにするオープンソースのハーネスだ。Dockerfile がインフラに、GitHub Actions が CI に相当するイメージで、AI コーディングの「型」をリポジトリにコミットして共有する。

各ランは git worktree で隔離され、複数タスクを衝突なく並列実行できる。決定的な処理（bash・テスト・git 操作）と AI ノード（プロンプト駆動）を DAG で混在させ、ループや人間承認ゲートも表現できる。

2 — なぜ Archon か
問題意識

「このバグを直して」と頼むだけでは、モデルによって計画の有無・テスト実行・PR 体裁が毎回ぶれる。Archon は フェーズ・検証ゲート・アーティファクトをワークフロー側で固定し、各ステップの知性だけを AI に任せる。

  • Fire and forget: ワークフローを起動して離席し、完了時に PR やレビューコメントが揃った状態にできる。
  • マルチ入口: 同じ定義を CLI・Web UI・Slack / Telegram / GitHub などから起動可能（プラットフォームアダプタ経由）。
3 — ワークフロー例（YAML）
ノードと依存

.archon/workflows/*.yaml に nodes を並べ、depends_on で DAG を構成する。loop で AI 反復や人間承認（until: APPROVED など）を表現する例は README にある。

# 抜粋: plan → implement ループ → テスト → レビュー → 承認ループ → PR
nodes:
  - id: plan
    prompt: "Explore the codebase and create an implementation plan"
  - id: implement
    depends_on: [plan]
    loop:
      prompt: "Read the plan. Implement the next task. Run validation."
      until: ALL_TASKS_COMPLETE
      fresh_context: true
  - id: run-tests
    depends_on: [implement]
    bash: "bun run validate"
  - id: create-pr
    depends_on: [approve]
    prompt: "Push changes and create a pull request"
4 — アーキテクチャ
データの流れ

公式ドキュメントの要約: プラットフォームアダプタ（Web / CLI / Telegram 等）がメッセージを Orchestrator に渡し、スラッシュコマンドは Command Handler、対話は AI アシスタントクライアント（Claude Code / Codex 等）、ワークフローは Executor、作業環境は Isolation プロバイダ（worktree 等）が担う。永続化は SQLite または PostgreSQL（コードベース・会話・セッション・ワークフロー実行・イベント等）。

6 — セットアップ
二つの入り方

フルセットアップ: リポジトリを clone し bun install のあと Claude Code で「Set up Archon」と指示。認証・CLI・スキル配置をウィザードが案内する。

クイック: curl https://archon.diy/install | bash または Homebrew brew install coleam00/archon/archon でスタンドアロン CLI を入れられる。

注意: コーディング対象のプロジェクト側で Claude Code を開くこと。スキルは対象リポジトリにコピーされ、Archon リポジトリ直下だけで作業しても意図どおり動かない。
7 — Web UI
Mission Control

バイナリ利用時は archon serve で UI を取得・起動。チャットに加え、Dashboard（実行中ワークフロー）、Workflow Builder（DAG 編集）、Workflow Execution（進捗）がある。サイドバーには CLI や Slack 経由の会話も横断表示される（README 記載）。

Hallmark

Nutlope/hallmark — AI が吐きがちな UI スロップを拒否するデザインスキル

Nutlope/hallmark は Claude Code / Cursor / Codex 向けのデザインスキル。色を変えただけのテンプレではなく、構造の多様性を強制する。マクロ構造を先に選び、テーマを被せ、58 ゲートのスロップテストと pre-emit 自己批判を通してから返す。

差別化: 見た目の差し替えではなく、hero → 3 features → CTA → footer という同一リズムを拒否する。2 つのブリーフで 2 つのサイトに見えることが目標（README / SKILL.md）。
4つの動詞
呼び出し面

デフォルトは新規 UI 構築。明示動詞は audit / redesign / study。画像や URL だけ渡された場合は study か参照ビルドかを一度確認する。

(default) Design
新規 UI。マクロ構造 → テーマ → ルールセット → スロップテスト。コンポーネント単体なら Component-scope に分岐。
hallmark audit <target>
既存コードを anti-pattern に照らして採点。パンチリストのみ。編集しない。
hallmark redesign <target>
コピー・IA・ブランドは残し、視覚構造を作り直す。既存ルートやコンポーネント所有権はデフォルトで保持。
hallmark study <screenshot|URL>
憧れデザインの DNA（マクロ構造・書体ペア・色アンカー）を抽出。ピクセルコピーと有料テンプレ URL は拒否。
実装セーフティレール
  • 本番ファイルやルートツリーの削除は、明示確認がない限り禁止。
  • 既存プロジェクトでは名前付きファイルの in-place 編集か、既存ルートへの加算を既定にする。
  • PDF / README / ブリーフは参照資料。ユーザーが verbatim 指定しない限り本文へ丸写ししない。
  • 編集前に作成・変更・削除予定ファイルを列挙する。削除は確認必須。
Design flow
既定パイプライン

ページ向けデフォルトは次の順序。コンポーネント単体ブリーフはマクロ構造・ナビ・フッター・hero enrichment・project memory をスキップする。

0 Pre-flight→ 1 Context gate→ 2 Macro + nav/ft→ 2.5 Memory→ 2.6 Theme→ 3 Rules→ 4 Enrich?→ 5 Preview→ 6 Build→ 7 Slop test

テーマ多様化の 3 軸

連続出力は次のうち少なくとも 1 軸が前と異なる必要がある。

  • Paper band — dark / mid / light（--color-paper の明度）
  • Display style — high-contrast-serif / grotesk-sans / mono / display-heavy など
  • Accent hue — warm / cool / neutral / chromatic-other

Custom 分岐は、ブランド色指定・多属性の雰囲気・明示の custom 要求があるときだけ表面化する。無言ならカタログ。

規律とスロップテスト
全動詞共通の 6 規律
  1. Pre-emit self-critique — Philosophy / Hierarchy / Execution / Specificity / Restraint / Variety を 1–5。いずれか < 3 なら改訂。
  2. Honest copy — ユーザー未提供の指標・証言・ロゴ数を捏造しない（gate 46）。
  3. Locked tokens — テーマ決定後は var(--color-*) / var(--font-*) のみ。インライン色・直書き font-family 禁止（gate 48）。
  4. Re-drawn chrome 禁止 — 偽ブラウザバー・偽 IDE 枠を手描きしない（gate 47）。
  5. Mobile — 320 / 375 / 414 / 768 で検証。横スクロール禁止、overflow-x: clip（hidden ではない）。
  6. Typography purity — 見出しに italic を使わない（gate 38a）。強調はウェイト・アクセント・下線。
スロップテストの束

references/slop-test.md は Step 7 専用。ゲートは視覚・構造・マイクロインタラクション・多様化・実装・hero enrichment・レスポンシブ・a11y などに分かれる。ジャンルで緩むゲートもある（例: atmospheric の radial bloom、modern-minimal のゼロクロマニュートラル）。

スタンプ: CSS 先頭に /* Hallmark · macrostructure: … · theme: … */。続いて .hallmark/log.json に追記（最大 20 件）。次回の多様化がこれを読む。
Component-scope

ボタン・入力・カードなど単一 UI のとき。マクロ / ナビ / フッター / enrichment / log 追記をスキップ。代わりに 8 状態（default · hover · focus · active · disabled · loading · error · success）必須と、プレビュー用ラッパーを出す。

インストール
skills CLI 
npx skills add nutlope/hallmark

または SKILL.md と references/ を手動配置:

  • Claude Code: ~/.claude/skills/hallmark/
  • Cursor: .cursor/rules/hallmark.mdc（frontmatter なしの本文）
  • Codex: ~/.codex/skills/hallmark/（個人）または .codex/skills/hallmark/（プロジェクト）

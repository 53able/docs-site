NativeBase（GeekyAnts/NativeBase）

v3.4.28 · React Native · Styled System · React ARIA — Android / iOS / Web

1 — 概要
位置づけ

メンテナンスモード NativeBase は GeekyAnts が開発する、React と React Native 向けのモバイルファースト UI コンポーネントライブラリ。v3 系では約 40 コンポーネントが Android・iOS・Web で一貫した見た目と挙動を提供する（README および公式ドキュメントに基づく）。

新規プロジェクトでは後継の gluestack-ui を推奨する。当初 v4 として計画されていた内容は gluestack-ui として提供されている。

~40コンポーネント
v3.4現行メジャー
ARIAアクセシビリティ
RN Webクロスプラットフォーム
設計の動機: React Native をゼロから組むとスタイル・インタラクション・状態・レスポンシブ・アクセシビリティを毎回足す必要がある。NativeBase は Material UI や Chakra UI に近い「制約付きユーティリティ props + 豊富な部品」で、その初期コストを下げる（README の Motivation 節）。

3 — 主な機能
v3 の特徴
機能内容
アクセシビリティReact ARIA / React Native ARIA と統合。フック経由で ARIA 属性を部品に載せる
ユーティリティ propsStyled System ベース。制約付きの bg・p・width 等で rapid な UI 組み立て
テーマextendTheme でトークンとコンポーネント variant を上書き。ダークモード対応
レスポンシブprops にオブジェクト・配列を渡しブレークポイント別スタイルを指定（useBreakpointValue と連携）
クロスプラットフォームReact Native Web により Web とネイティブで同じコンポーネント API

技術スタック（README より）

JavaScript、React Native、Styled System。依存は React Native / Expo エコシステム。Web は react-native-web 経由。

4 — 導入
セットアップ

Expo または React Native CLI で初期化したアプリに native-base を追加する。詳細手順は 公式 Installation ガイド を参照。

npm install native-base react-native-svg react-native-safe-area-context
# または
yarn add native-base react-native-svg react-native-safe-area-context

ルートに NativeBaseProvider を置き、必要なら extendTheme でテーマを拡張する。

import { NativeBaseProvider, extendTheme } from 'native-base';

const theme = extendTheme({
  config: { initialColorMode: 'dark' },
});

export default function App() {
  return (
    <NativeBaseProvider theme={theme}>
      {/* アプリ本体 */}
    </NativeBaseProvider>
  );
}
互換バージョン: v3.0.0 以降は React Native 0.63.0 以上が前提（README の Compatible Versions 表）。パッケージ現行版は 3.4.28（package.json）。

6 — テーマと hooks
テーマ解決の流れ
extendTheme → NativeBaseProvider → usePropsResolution → Styled props → RN style

src/theme/index.ts は base トークンと components テーマ定義を束ねる。各コンポーネント theme は baseStyle・sizes・variants・defaultProps の 4 層を持つ（ComponentTheme 型）。

よく使う hooks（src/hooks/index.ts）

Hook用途
useTheme現在のテーマオブジェクト取得
useToken色・サイズなどのデザイントークン解決
useColorMode / useColorModeValueライト/ダーク切替（core から再エクスポート）
useBreakpointValueブレークポイントに応じた値の選択
useMediaQueryメディアクエリ条件の監視（Web）
useContrastText背景色に対する読みやすい文字色
useToastトースト通知の表示（composites から）

v3-compatible-theme（v33x-theme）は旧テーマとの互換レイヤーとして同梱される。

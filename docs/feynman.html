概要

Feynman は、論文・ウェブ・コードリポジトリを横断してエビデンスを収集し、引用付きの再現性ある研究成果物を生成するオープンソース CLI エージェント。 Pi をエージェントランタイムとし、 alphaXiv を論文検索・解析の基盤として使う。 すべての出力はソースに根ざしており、論文・ドキュメント・リポジトリへの直接 URL を含む。

1 — 概要・設計思想
何ができるか

Feynman に調査トピックを渡すと、次の流れで研究成果物を生成する。

bash — quick examples
$ feynman "scaling laws とは何か"
→ 論文・Web を検索し、引用付きリサーチブリーフを生成

$ feynman deepresearch "mechanistic interpretability"
→ 並列 Researcher エージェントによる多角的調査、検証・合成まで自動

$ feynman lit "RLHF alternatives"
→ 文献レビュー：コンセンサス・対立点・未解決問題を整理

$ feynman audit 2401.12345
→ 論文の主張と公開コードベースの乖離を監査

$ feynman replicate "chain-of-thought improves math"
→ ローカルまたはクラウド GPU で実験を再現
設計原則
  • Evidence over fluency — 流暢な文章より論拠を優先する。
  • 観察と推論を分離 — 確認できたことと推論したことを明示的に区別する。
  • 不確実性を明示 — 最新文献や不安定な事実に関する問いにはツールを使う。
  • 検証なしに "verified" と言わない — 実際にチェックし、証拠を示せる時だけ使う。
  • 成果物ファースト — 読書リスト・監査・実験ログ・草稿など具体的な成果物を届ける。
2 — インストール
スタンドアロン CLI

1 行のシェルスクリプトで最新リリースをインストールできる。Node.js は不要。

macOS / Linux
curl -fsSL https://feynman.is/install | bash
Windows PowerShell
irm https://feynman.is/install.ps1 | iex

バージョンを固定する場合は末尾に -s -- 0.2.16 のように渡す。
pnpm / bun でインストールする場合は Node.js 20.19.0 以上が必要。

スキルのみインストール

フル CLI が不要でリサーチスキルだけ欲しい場合：

グローバル（~/.codex/skills/feynman）
curl -fsSL https://feynman.is/install-skills | bash
リポジトリローカル（.agents/skills/feynman）
curl -fsSL https://feynman.is/install-skills | bash -s -- --repo
Node.js バージョン: npm / pnpm / bun 経由でインストールする場合は node >= 20.19.0 が必要。スタンドアロンバンドルを使う場合は Node.js は不要。
3 — コマンド一覧
スラッシュコマンド

自然言語でも入力できるが、スラッシュコマンドをショートカットとして使える。

コマンド動作出力先
/deepresearch <topic> ソース重視の多エージェント調査。Researcher × N を並列起動し、Verifier → Reviewer で検証。 outputs/
/lit <topic> 文献レビュー。論文検索からコンセンサス・対立点・未解決問題を整理。 outputs/
/review <artifact> 査読シミュレーション。深刻度グレードと改訂プランを付与。 outputs/
/audit <paper> 論文の主張と公開コードベースの乖離・再現性リスクを監査。 outputs/
/replicate <paper> ローカルまたはクラウド GPU で実験を再現。 experiments/
/compare <topic> ソース比較マトリクスを生成。 outputs/
/draft <topic> リサーチノートから論文スタイルの草稿を作成。 papers/
/autoresearch <idea> 自律実験ループ。仮説→実験→検証を繰り返す。 experiments/
/watch <topic> 定期的なリサーチ監視スケジュールを設定。 outputs/
成果物の命名規則

すべてのワークフローはトピックからスラッグ（lowercase・ハイフン・フィラーワードなし・最大 5 語）を導出し、同一ラン内のすべてのファイルにプレフィックスとして使う。

例: slug = "cloud-sandbox-pricing"
outputs/.plans/cloud-sandbox-pricing.md       # 実行計画
cloud-sandbox-pricing-research-web.md        # Web 調査中間ファイル
cloud-sandbox-pricing-research-papers.md     # 論文調査中間ファイル
outputs/.drafts/cloud-sandbox-pricing-draft.md
outputs/cloud-sandbox-pricing.md             # 最終成果物
outputs/cloud-sandbox-pricing.provenance.md  # 出所記録
4 — マルチエージェント構成
エージェント構成

Feynman は 4 つのサブエージェントを状況に応じて組み合わせる。Lead Agent が計画・委譲・合成・納品を担い、内部のオーケストレーションはユーザーに見えない。

researcher
論文・Web・リポジトリ・ドキュメントを横断してエビデンスを収集。N 並列で異なる調査軸をカバー。
writer
リサーチノートから構造化ドラフトを生成。レポート・文献レビュー・論文スタイルに対応。
verifier
ドラフトにインライン引用を追加し、すべてのソース URL を検証。デッドリンクをクリーンアップ。
reviewer
引用付きドラフトに対し、証拠根拠不足・論理的矛盾・単一ソースの重要主張などを検証。FATAL はフィックスループへ。
ソースと計算資源
Papers
alphaXiv — 論文検索・Q&A・コードリーディング・アノテーション（alpha CLI 経由）
Web
Gemini または Perplexity — ゼロコンフィグで使える Web 検索。製品・最新情報・規制・モデルベンチマーク向け。
Compute
Docker — ローカル安全実行。Modal — サーバーレス GPU バースト。RunPod — 永続 GPU ポッド + SSH。

6 — ディープリサーチフロー
/deepresearch の 7 ステップ

/deepresearch は Feynman の主力ワークフロー。Lead Agent が以下を順番に実行する。

1
Plan
拡張思考でリサーチ戦略を策定。スラッグを導出し outputs/.plans/<slug>.md に計画・タスク台帳・検証ログを書く。ユーザーに確認を求める。
2
Scale decision
クエリの複雑さに応じてサブエージェント数を決定。単一事実なら自分で処理（0 spawn）、広範調査なら 3–6 並列 Researcher。
3
Spawn researchers
調査軸（ソース種別・時期・技術角度）が重複しないよう Researcher を並列起動。各 Researcher はファイルに出力し、大量の中間結果を Lead のコンテキストに流し込まない。
4
Evaluate & loop
未回答の問い・単一ソース主張・矛盾を評価。不足があれば追加 Researcher バッチを起動。タスク台帳と検証ログを都度更新。
5
Write report
Lead Agent が直接レポートを書く（Writer に委譲しない）。主要主張を検証ログにマッピングし、根拠のないものをダウングレード。outputs/.drafts/<slug>-draft.md に保存。
6
Cite
Verifier がドラフトにインライン引用を追加し、全ソース URL を検証。デッドリンクをクリーンアップして <slug>-brief.md を生成。
7
Verify & Deliver
Reviewer が FATAL / MAJOR / MINOR で問題を分類。FATAL はフィックスループへ。最終成果物を outputs/ または papers/ に保存し、出所記録 .provenance.md を付与。
バックグラウンド実行: 長時間ワークフローは clarify: false, async: true でサブエージェントを非同期起動できる。非同期 ID と subagent_status で進捗確認が可能。

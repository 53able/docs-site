gsd-pi

GSD Pi — open-gsd コミュニティのローカルファーストコーディングエージェント（CLI・TUI・Web）

1 — 概要
プロジェクト
@opengsd/gsd-pi 1.1.x MIT Node 22+

GSD Pi（GitHub: open-gsd/gsd-pi）は、計画・実装・検証・進捗追跡をターミナルから行うローカルファーストのコーディングエージェント。Pi 系エージェント基盤の上に、マイルストーン／スライス／タスクのワークフロー、worktree 分離、.gsd/ へのプロジェクト記憶、拡張面を載せている。

README では「アイデアからレビュー可能な実装まで、手動調整を減らす」ことを目的に掲げ、TUI をデフォルトとしつつ gsd --web で Web コントロールプレーンも使える。

gsdCLI バイナリ
.gsd設定・状態ディレクトリ
Unit最小実行ステップ
pnpmモノレポ
コミュニティ継続: VISION.md によると、本リポジトリは元 GSD プロジェクトの open-gsd 組織によるコミュニティ主導の継続開発ライン。v1 系は gsd-core として別途メンテされる。
2 — 主な機能
ガイド付きターミナルエージェント

gsd で起動し、プロバイダー設定のあと計画セッションやクイックタスクを実行。

自律プロジェクトワークフロー

マイルストーン・スライス・タスクへ分解し、auto モードで計画→実装→検証→前進を繰り返す。

worktree 対応 Git

実装を worktree に隔離し、メイン checkout はレビュー可能な状態を保つ。

ローカルプロジェクト記憶

要件・決定・ランタイムメモ・計画・検証証跡を .gsd/ に保存。DB と Markdown 投影の両方。

マルチプロバイダー

フェーズごとのモデル設定。Anthropic / OpenAI / Google など複数経路をルーティング。

拡張面

バンドル拡張・スキル・MCP・ブラウザ自動化・リモート質問などを src/resources/extensions/ から提供。

TUI / Web

対話は TUI が既定。gsd --web でブラウザ UI と API 面を起動。

Cloud MCP Gateway

ローカル runtime とクラウド MCP エンドポイントをペアリングし、外部 MCP クライアントから GSD ツール面を公開。

3 — インストールとクイックスタート
推奨インストール 
npx @opengsd/gsd-pi@latest

CI やスクリプト向けには --yes を付ける。グローバル npm なら npm install -g @opengsd/gsd-pi@latest。旧 unscoped gsd-pi から移行する場合は README のマイグレーション手順（アンインストール → 状態ファイル削除 → 再インストール）に従う。

  1. 1シェルで gsd を実行しセットアップフローを完了する。
  2. 2モデルプロバイダーを選び、作業ディレクトリを開く。
  3. 3計画・実行状態はプロジェクト直下の .gsd/ に蓄積される。

ブラウザで設定を組み立てる場合は GSD Pi web configurator を参照。

4 — ワークフローとコマンド
階層 
flowchart TB
  M[Milestone] --> S[Slice]
  S --> T[Task]
  T --> U[Unit]
  U --> A[Auto Orchestration]
  A --> W[Worktree 実装]
  A --> V[検証・closeout]

CONTEXT.md の用語では Unit が最小実行ステップ（plan slice / execute task / complete slice など）。Auto Orchestration が dispatch 前の不変条件パイプラインとライフサイクル遙測を担い、State Reconciliation → Dispatch → Tool Contract → Worktree Safety の順で pre-dispatch を通す設計（ADR-014 / ADR-015）。

5 — アーキテクチャ
ランタイムモジュール

CONTEXT.md が採用する深いモジュール:

  • Auto Orchestration — start / advance / resume / stop / getStatus。pre-dispatch パイプラインのオーケストレーション。
  • State Reconciliation — DB・ディスク・メモリ間の drift 検出と修復（ADR-017）。
  • Worktree Safety / Lifecycle / State Projection — ソース書き込み Unit の worktree 根保証と状態ファイルの方向性ルール。
  • Recovery Classification — プロバイダー・ツール・Git 失敗を retry / escalate / abort に分類。
  • Tool Contract — Unit プロンプト・ツールスキーマ・ポリシーの整合。

CodeGraph

@colbymchenry/codegraph · semantic code intelligence for Claude Code

1 — 概要
目的

CodeGraph は、Claude Code の Explore エージェントが行う grep / glob / Read によるファイル走査を、事前に構築したコード知識グラフへのクエリに置き換えるツール。シンボル関係・呼び出しグラフ・構造情報をローカルの SQLite に保持し、MCP サーバー経由でエージェントに公開する。

94% ツール呼び出し削減（README 公称）
77% 探索時間短縮（README 公称）
19+ 対応言語
100% ローカル（SQLite のみ）
データは外部に出ない。 API キー不要。インデックスはプロジェクト直下の .codegraph/codegraph.db に保存される。tree-sitter（WASM）で AST を解析し、FTS5 で全文検索を提供する。
主な機能

Smart Context

1 回のツール呼び出しでエントリポイント・関連シンボル・コード断片をまとめて返す。

Full-Text Search

FTS5 によりシンボル名をコードベース全体から即時検索。

Impact Analysis

変更前に callers / callees と影響半径をグラフ走査で把握。

Auto-Sync

OS ネイティブのファイルイベント（FSEvents / inotify 等）で保存後に増分同期。デバウンス 2 秒。

Framework Routes

Django / FastAPI / Express / Spring など 13 フレームワークのルート定義を route ノードとしてリンク。

codegraph affected

変更ファイルから import 依存を辿り、影響を受けるテストファイルを CI 向けに列挙。

2 — ベンチマーク（README 記載）
6 コードベース比較

Claude Opus 4.6（1M context）+ Claude Code v2.1.91。同一質問で Explore エージェントを 1 体起動し、CodeGraph あり / なしを比較した結果（README より）。

コードベース CodeGraph あり なし 改善（README）
VS Code（TypeScript） 3 calls, 17s 52 calls, 1m 37s 94% fewer · 82% faster
Excalidraw（TypeScript） 3 calls, 29s 47 calls, 1m 45s 94% fewer · 72% faster
Claude Code（Python+Rust） 3 calls, 39s 40 calls, 1m 8s 93% fewer · 43% faster
Claude Code（Java） 1 call, 19s 26 calls, 1m 22s 96% fewer · 77% faster
Alamofire（Swift） 3 calls, 22s 32 calls, 1m 39s 91% fewer · 78% faster
Swift Compiler（Swift/C++） 6 calls, 35s 37 calls, 2m 8s 84% fewer · 73% faster
観察（README）: CodeGraph 利用時は codegraph_explore の結果を信頼しファイル Read にフォールバックしなかった。最大規模（Swift Compiler: 25,874 files / 272,898 nodes）でも 6 explore calls・ファイル Read 0 で 35 秒と記載。
3 — セットアップ
インストール 
npx @colbymchenry/codegraph

対話型インストーラが codegraph のグローバル導入、~/.claude.json への MCP 登録、ツール auto-allow、~/.claude/CLAUDE.md への指示追記を行う。

  1. 上記を実行
  2. Claude Code を再起動（MCP 読み込み）
  3. プロジェクトで codegraph init -i（.codegraph/ 作成 + 索引）
Explore エージェント向けルール（要約）

.codegraph/ があるプロジェクトでは、メインセッションから codegraph_explore / codegraph_context を直接呼ばない（大量ソースでコンテキストを消費するため）。探索質問は Explore エージェントに委譲し、プロンプトに CodeGraph 利用指示を含める。

メインセッションで直接使える軽量ツール: codegraph_search、codegraph_callers / codegraph_callees、codegraph_impact、codegraph_node。

4 — 動作の仕組み
アーキテクチャ 
flowchart TD
  CC[Claude Code / Explore Agent]
  MCP[CodeGraph MCP Server]
  DB[(SQLite + FTS5
.codegraph/codegraph.db)]
  TS[tree-sitter extraction]
  RES[Resolution + frameworks]
  WATCH[File watcher auto-sync]

  CC -->|MCP tools| MCP
  MCP --> DB
  TS --> DB
  RES --> DB
  WATCH -->|debounced incremental| TS
Extraction → Storage → Resolution → Auto-Sync
  • Extraction: tree-sitter で AST を解析。言語別クエリで関数・クラス・呼び出し・import などをノード/エッジ化。
  • Storage: SQLite + FTS5。プロジェクト内 .codegraph/ に保存。
  • Resolution: 呼び出し先定義・import 先・継承・フレームワーク固有パターンを解決。
  • Auto-Sync: MCP 起動中はソース変更を検知し、約 2 秒の quiet window 後に増分同期。

5 — MCP ツールと CLI
MCP ツール
ツール用途
codegraph_searchシンボル名で検索
codegraph_contextタスク向けコードコンテキスト生成
codegraph_callers呼び出し元の列挙
codegraph_callees呼び出し先の列挙
codegraph_impactシンボル変更の影響範囲
codegraph_node単一シンボルの詳細（ソース付き可）
codegraph_files索引済みファイル構造（fs 走査より高速）
codegraph_statusインデックス統計・健全性
CLI（抜粋） 
codegraph init [path]              # 初期化（--index で同時索引）
codegraph index [path]             # フルインデックス（--force / --quiet）
codegraph sync [path]              # 増分更新
codegraph status [path]            # 統計表示
codegraph query <search>           # シンボル検索
codegraph context <task>           # AI 向けコンテキスト
codegraph affected [files...]      # 影響テスト列挙（--stdin 対応）
codegraph serve --mcp              # MCP サーバー起動

ライブラリ API（抜粋）

import CodeGraph from '@colbymchenry/codegraph';

const cg = await CodeGraph.init('/path/to/project');
await cg.indexAll({ onProgress: (p) => console.log(p.phase) });
const results = cg.searchNodes('UserService');
const context = await cg.buildContext('fix login bug', { maxNodes: 20, includeCode: true });
cg.watch();
cg.close();

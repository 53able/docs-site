architect-loop

Claude Fable が設計・判定、GPT-5.5 Codex が実装・調査 — リポジトリを唯一の記憶に据えたクロスベンダーループ

1 — 概要
何か

architect-loop は、DanMcInerney/architect-loop が提供する Claude Code スキル群。Claude Fable 5（high effort）がアーキテクトとしてスライス仕様・凍結ゲート・判定を担い、GPT-5.5（Codex CLI, xhigh）がビルダーとして実装と調査を fresh context で実行する。デフォルトでは既存の Claude / ChatGPT サブスクリプションだけで動き、API キーは不要。

2 — 役割分担
4 つの役割
Architect — Claude Fable 5
仲裁・凍結ゲート照合・次スライス仕様・kill/continue。実装コードは書かない。ディスパッチしたセッションでは同じ実行を判定しない。
Builder — GPT-5.5 (codex exec)
レーンごとに fresh process + git worktree。PHASE 0 で仕様への異議を必須化し、宣言ファイルのみ変更。生の証拠だけ報告する。
Memory — リポジトリ
docs/HANDOFF.md、docs/gates/、docs/lanes/、git 履歴。リポジトリ外の会話出力は存在しない扱い。
Human — あなた
スコープ・不可逆操作・好みの最終権限。ワークブロック間で handoff を読み、オーバーライドする。

SPEC + GATES
先にゲートを凍結
1 PR サイズのスライスを仕様化し、docs/gates/<slice>.md をディスパッチ前にコミット。ビルダーがゲートを編集したら自動 FAIL。
PARALLEL LANES
worktree 分離
ファイルタッチ集合の重複をチェックし、最大 4 レーンまで並列 codex exec。マージ衝突はレーン設計の欠陥として扱う。
JUDGMENT
アーキテクトがゲート実行
ビルダーの成功報告は参考情報のみ。ゲートコマンドはアーキテクトが自分で走らせ、diff を仕様意図と照合する。
DISAGREEMENT
沈黙遵守は欠陥
PHASE 0 で仕様への異議をファイル引用付きで必須化。黙って従うだけのビルダー出力は欠陥として記録する。

代表的なディスパッチ

codex exec -C <repo> --sandbox workspace-write \
  -m gpt-5.5 -c model_reasoning_effort="xhigh" \
  --json -o .architect/last-run.md \
  "<builder block: PHASE rules + slice spec + frozen gates>"
4 — リサーチループ（/architect-research）
何を build するか決める段階

ブレインストーム・技術選定・SOTA 調査向けの別スキル。ビルドループの副作用にしない（コストが約 15×）。調査結果は docs/research/ にまとめ、必要なら docs/prd/ へ蒸留してビルドループへ渡す。

flowchart TD
  R["/architect-research"] --> S["1 Scope → brief"]
  S --> T["2 Scout 地形マップ
（比較・事実確認は省略可）"]
  T --> U["3 トピック別 3–6 lanes 設計"]
  U --> V["4 並列 read-only researchers"]
  V --> W["5 Gap round + expert 第2波"]
  W --> X["6 Verify ≥2 独立ソース"]
  X --> Y["7 単一著者が report 執筆"]
  • Scout-first: 本番の deep-research 系は固定レーンではなくプランナー設計の分解を採用。scout が用語・主要システム・論点をマップし、lanes はトピックから設計する。
  • Findings discipline: URL + 日付 + 引用 + 信頼度タグ。NOT FOUND は推論より優先。推奨は書かず、オーケストレーターが判断する。
  • Expert wave: 専門家意見は第2波レーン。事実の ≥2 ソースには数えないが、専門家間の対立は未解決論点として記録する。
5 — 12 の設計ルール（要約）
DESIGN.md の機械的強制
IDルール要点
R1Repo memoryHANDOFF.md に無いことは起きていない
R2Gates freeze結果前に docs/gates/ をコミット。ビルダー編集は自動 FAIL
R3No self-gradeビルダーもアーキテクトも自分の実行を同セッションで採点しない
R4Outcome gradingゲートごとに PASS / FAIL / INVALID。パス手順ではなく結果を見る
R5DisagreementPHASE 0 の異議必須。ACCEPT / REJECT / MODIFY を明示
R6Full contract目的・出力形式・ツール指示・境界をディスパッチに含める
R7Fresh contextスライスごとに新しい codex exec。壊れたら rescue より再ディスパッチ
R8Worktree lanesアーキテクトが fan-out を所有。≤4 レーン、ファイル集合は非重複
R9Async supervise長時間ビルドはバックグラウンド。ライブネスと stall triage
R10Grounded claimsステータス報告はツール出力と照合。解釈はアーキテクトの仕事
R11Scale effort自明な作業はループ不要。まずプロジェクトの検証ゲートを読む
R12Thin skill不変条件とインターフェースだけ残し、手順の過剰規定は削る
意図的にやらないこと: 無限自律ループではない（人間がブロック間に介在）。汎用オーケストレーターの置き換えでもない。Goal Mode に加えて、クロスモデル判定・外部ゲート・リポジトリ記憶を足した形。
6 — リポジトリ記憶
永続アーティファクト
パス役割
docs/HANDOFF.md短い目次（~150 行）。TL;DR とゲート・レーンへのポインタ。詳細はアーカイブ
docs/gates/<slice>.md凍結された受け入れゲート（コマンド + 閾値）
docs/lanes/<slice>-<lane>.mdレーンごとの生結果レポート
docs/prd/<slice>.mdリサーチ蒸留後の PRD（ビルダー PHASE 0 の対象）
docs/research/<topic>.md/architect-research の決定指向レポート
.architect/実行テレメトリ（gitignore）。生リサーチ出力など
7 — 導入
前提とコマンド

Claude Code（有料プラン）と Codex CLI（ChatGPT プラン、≥ 0.133）が必要。

git clone https://github.com/DanMcInerney/architect-loop
cd architect-loop && ./install.sh        # Windows: .\install.ps1
npm i -g @openai/codex@latest

./install.sh --project でカレントリポジトリのみにインストール可能。

使い方（2 コマンド）

/architect                                      # ビルドループ
/architect-research <検討中のテーマ>              # リサーチループ

GBrain

Postgres ネイティブのパーソナル知識ベース — Markdown リポジトリ × ハイブリッド検索

1 — 概要
位置づけ

GBrain は、AI エージェントが「あなたの文脈」を読むための 検索・保存レイヤーだ。会議・メール・X・カレンダー・通話などのシグナルを、検索可能な知識ベース（脳）に流し込み、応答の前に読み、会話の後に書き戻す運用を前提にしている。

Bun / TypeScript MIT OpenClaw プラグイン pgvector
0.7.0VERSION（package.json）
37BrainEngine 操作
30MCP ツール
RRFハイブリッド融合

正典はローカルの repos/gbrain（README・docs/・src/）。上流: github.com/garrytan/gbrain。

フロンティアモデル推奨。 README では Claude Opus 4.6 および GPT-5.4 Thinking での検証が前提とされ、小型モデルでは破綻しやすいと明記されている。
2 — アーキテクチャ
三層の関係

Git 管理の Markdown 脳リポジトリがソースの正であり、GBrain が Postgres + pgvector 上に検索インデックスを持つ。エージェントは CLI / MCP 経由で読み書きし、人間は常に Markdown を直接編集できる。

flowchart LR MD["Markdown 脳リポジトリ\ngit = source of truth"] PG["Postgres + pgvector\nPGLite / Supabase"] HY["ハイブリッド検索\nRRF 融合"] AG["AI Agent\nskills / MCP"] MD -->|import / sync| PG PG <--> HY HY <--> AG MD <--> AG
コンパウンドループ（README の要旨）
シグナル到着 → エンティティ検出 → READ: 脳を先に検索 → 応答 → WRITE: ページ更新 → sync / embed

ループを回すほど文脈が蓄積し、同じ人物・会社が次に出たときには既にドシエが揃っている、というのが設計上の価値だ。

3 — ストレージエンジン
Pluggable BrainEngine

すべての操作は BrainEngine 契約を通る。エンジンを差し替えても CLI・MCP・スキルは同じ。v0.7 以降は PGLiteEngine（WASM 埋め込み Postgres 17.5）が既定で、サーバ不要のローカル開始が約 2 秒、という README のトーンだ。

ニーズ向くエンジン
ゼロコンフィグ・ローカルPGLiteEngine（既定）
大規模・多デバイス・マネージド PostgresPostgresEngine + Supabase
移行gbrain migrate --to supabase など（README / ENGINES.md）

検索の共通化: キーワードとベクトルはエンジン実装の差を吸収し、RRF による融合と重複排除は src/core/search/hybrid.ts 側で共有される（ENGINES.md の説明に沿う）。

4 — メモリの層（OpenClaw / Hermes との関係）
世界知識 vs 運用メモリ

README は GBrain を「人物・会社・会議・思想などの長期世界知識」とし、OpenClaw の memory_search 的なものは「運用状態・好み・セッション設計」と対比している。両方を見る設計が推奨される。

層保持するもの問い合わせの例
gbrain人・会社・会議・アイデア・メディアなどgbrain search / gbrain query / gbrain get
エージェントメモリ好み・決定・運用設定memory_search（OpenClaw）
セッション直近の会話文脈（自動）
5 — データイン（レシピ）
レシピ駆動

各種連携は recipes/*.md に手順が書かれ、エージェントがキー取得・検証・配線まで進める。「Markdown がインストーラ」になる、という README の ethos に近い。

レシピ主な用途
ngrok-tunnelMCP や音声用の固定 URL
credential-gatewayGmail / Calendar へのゲートウェイ
twilio-voice-brain電話 → 脳ページ
email-to-brainGmail → エンティティページ
x-to-brainX（Twitter）→ 脳ページ
calendar-to-brainGoogle Calendar → 日次ページ
meeting-syncCircleback 等の議事 → ページ

依存関係は gbrain integrations で一覧・健全性確認ができる。

6 — CLI / MCP
入口
  • CLI: gbrain init → gbrain import → gbrain embed --stale → gbrain query など。スタンドアロンでも利用可能。
  • MCP（stdio）: gbrain serve で約 30 ツール。CLI と同じ操作定義から生成される（README の記述）。
  • リモート MCP: Supabase Edge へのデプロイ・トークン発行で、Desktop / Cowork / Perplexity などから接続可能（docs/mcp/）。
# 例: MCP クライアント設定（README 抜粋）
{
  "gbrain": {
    "command": "gbrain",
    "args": ["serve"]
  }
}
スキルパック

ツールだけでは足りず、GBRAIN_SKILLPACK.md や skills/*/SKILL.md が「いつ読むか・どう書くか」を規定する。運用の正はリポジトリ内ドキュメントに従う。

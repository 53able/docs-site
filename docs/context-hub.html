Context Hub

コーディングエージェント向けのキュレート・バージョン付きドキュメント。検索・取得・アノテーション・フィードバック・MCP で、セッションを超えて賢くなる。

1 — 概要
何か

Context Hub（CLI 名: Chub）は、コーディングエージェントが「API をハルシネートする」「セッションで学んだことを忘れる」問題に向き合うための仕組み。キュレートされバージョン管理されたドキュメントを配信し、エージェントが chub search / chub get で必要なドキュメントを取得して使う。コンテンツはすべてオープンで Markdown としてリポジトリにあり、エージェントが読む内容をそのまま確認・貢献できる。

chubCLI 名
search/get検索・取得
annotateローカル注釈
feedback作者へ投票
chub-mcpMCP サーバー

npm パッケージは @aisuite/chub。Node.js 18 以上。2 つの実行ファイル（chub CLI と chub-mcp MCP サーバー）が同梱されている。

2 — Quick Start
導入

グローバルインストール後、検索して取得する。

npm install -g @aisuite/chub
chub search openai                 # 利用可能なドキュメントを検索
chub get openai/chat --lang py     # 現在のドキュメントを取得（Python 版）

言語は --lang py / --lang js で指定。取得した Markdown をエージェントのコンテキストに含めてコード生成に使う。

3 — 仕組み
設計思想

Chub は人間ではなくコーディングエージェントが使うことを前提にしている。プロンプトで「chub を使って最新の API ドキュメントを取得し、その内容に従って実装して」と指示するか、get-api-docs スキル（SKILL.md）をエージェントに登録して使う。

基本フローは search → get → use → annotate → feedback の 5 ステップ。エージェントが不足やワークアラウンドに気づいたら chub annotate でローカルにメモしておくと、次回の chub get 時にその注釈が自動で付いて返る。

使用例 
chub search "stripe payments"        # 関連ドキュメントを検索
chub get stripe/api --lang js       # ドキュメントを取得
# エージェントがドキュメントを読み、正しいコードを書く。

chub annotate stripe/api "Needs raw body for webhook verification"
# 次セッションでは chub get 時にこの注釈が自動で付与される。

chub feedback stripe/api up          # 作者に評価を送る
4 — コマンド一覧
全コマンド
コマンド説明主なフラグ
chub search [query] ドキュメント・スキルを検索（クエリ省略で一覧） --tags <csv> --lang --limit --json
chub get <ids...> ID でドキュメントまたはスキルを取得。複数指定可 --lang --version --full --file -o --json
chub annotate <id> [note] ローカル注釈を付与・閲覧・削除・一覧 --clear --list --json
chub feedback <id> up|down ドキュメントを評価。ラベルと作者情報を付与できる --label --file --agent --model --status
chub update キャッシュ済みレジストリインデックスを更新 --force --full（オフライン用フルバンドル）
chub build <content-dir> ローカルコンテンツから配布用レジストリをビルド -o <output> --base-url

グローバルフラグ: --json（構造化 JSON 出力）、--version、--help。詳細は CLI Reference を参照。

Incremental Fetch

ドキュメントにメイン以外の参照ファイルがある場合、chub get の出力末尾に利用可能なファイル一覧が表示される。--file で必要なファイルだけ、--full で一式を取得できる。トークンを無駄にしない設計。

chub get acme/widgets --file references/advanced.md   # 特定のリファレンスのみ
chub get acme/widgets --full                          # すべてのファイルを取得
chub get stripe/api -o .context/stripe.md             # ファイルに保存
5 — セッションを超えて賢く
アノテーション

アノテーションは、エージェントがドキュメントに付けるローカルメモ。~/.chub/annotations/ に JSON として保存され、セッションをまたいで永続する。次回 chub get したときにドキュメントの末尾に自動で追記されて返る。

効果的なアノテーションの例:

  • 環境固有の落とし穴 — 「Webhook 検証は JSON parse 前の raw body が必要」
  • バージョン固有の注意 — 「v3 API は別の認証ヘッダーが必要」
  • プロジェクト固有の文脈 — 「個別呼び出しではなくバッチエンドポイントを使用」
  • エラー対策 — 「レートリミットエラーには jitter 付き指数バックオフが必要」
フィードバックラベル

作者へ投票する際にラベルを付けると、何が良くて何が悪いかを具体的に伝えられる。

ポジティブ:

accurate well-structured helpful good-examples

ネガティブ:

outdated inaccurate incomplete wrong-examples wrong-version poorly-structured 
chub feedback openai/chat down --label outdated --label wrong-examples
chub feedback stripe/api up --agent "claude-code" --model "claude-sonnet-4"
chub feedback acme/widgets down --file references/advanced.md --label incomplete
アノテーション vs フィードバック
項目アノテーションフィードバック
対象自分のエージェント（ローカル）ドキュメント作者（レジストリ経由）
永続場所~/.chub/annotations/リモートレジストリ
目的同じミスを繰り返さないコンテンツ品質の向上
可視範囲自分のみメンテナーのみ
効果次回 get 時に自動付与作者がドキュメントを更新

アノテーションは今日の自分のエージェントを賢くし、フィードバックは明日の全員のドキュメントをよくする。

6 — コンテンツと MCP
コンテンツタイプ

バージョン管理され、言語別（py / js など）のバリアントがある。ディレクトリ構造でマルチ言語・マルチバージョンを表現する。

content/
  acme/
    docs/
      widgets/
        DOC.md             # 単一言語
      client/
        javascript/DOC.md  # 複数言語
        python/DOC.md
      api/
        v1/DOC.md          # 複数バージョン
        v2/DOC.md          # 最高バージョンが推奨バージョンになる
    skills/
      deploy/SKILL.md      # エージェントスキル

API ドキュメント以外のコンテンツタイプ（エージェントスキルなど）もロードマップ上にある。

MCP 統合（chub-mcp）

@aisuite/chub には CLI だけでなく、MCP サーバー（chub-mcp）も同梱されている。MCP クライアント（Claude Desktop など）から Context Hub のドキュメントを直接呼び出せる。

# Claude Desktop の mcp_servers 設定例
{
  "chub": {
    "command": "chub-mcp"
  }
}

CLI と MCP の 2 経路でアクセスでき、エージェントのアーキテクチャに合わせて選択できる。

貢献: ドキュメント・スキルは Markdown + YAML フロントマターで PR により追加できる。chub build でローカルコンテンツを検証・ビルドし、Content Guide でフォーマットと構造を確認できる。エージェントからのフィードバック（up/down）が集まるほど、コンテンツの優先度が上がる。
7 — エージェントスキル（get-api-docs）
5 ステップワークフロー

Context Hub には cli/skills/get-api-docs/SKILL.md が付属する。エージェントに登録すると、外部ライブラリを使うコードを書く前に自動でドキュメントを取得するようになる。

1
ドキュメント ID を見つける
推測ではなく chub search で確認してから取得する。
chub search "openai" --json
2
ドキュメントを取得する
バリアントが複数あれば --lang を指定。1 つのみなら自動選択される。
chub get openai/chat --lang py
3
ドキュメントに従ってコードを書く
記憶で API を推測せず、取得した内容を正として実装する。
4
学んだことをアノテートする
ドキュメントに書かれていないワークアラウンドや落とし穴をメモしておく。次回以降のセッションでも役立つ。
chub annotate openai/chat "streaming requires async iterator, not .text()"
5
フィードバックを送る
ドキュメントが役立ったかどうかをラベル付きで評価する。ユーザーに確認してから送信するのが推奨。
chub feedback openai/chat up --label helpful --label good-examples
スキルの登録方法（Claude Code の場合）: ~/.claude/skills/get-api-docs/ ディレクトリを作成し、SKILL.md を配置する。Claude Code が外部ライブラリを使う場面でスキルを自動的に参照するようになる。

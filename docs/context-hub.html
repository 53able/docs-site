Context Hub

chub — コーディングエージェントがキュレーションされたドキュメントを取得し、セッションを超えて賢くなるための CLI。

1 — 概要
何か

Context Hub（略称 chub）は、コーディングエージェントが「正しい API ドキュメント」を手に入れ、セッションをまたいで学びを蓄積するための CLI。エージェントは API を hallucinate したり、前のセッションで得た知識を忘れたりしがちなので、バージョン管理された Markdown のドキュメントを検索・取得し、必要に応じてローカルでアノテーションを付けて次回以降の取得時に自動で付与されるようにする。

chubCLI 名
search / get検索・取得
annotateローカルメモ
feedback作者へ投票

コンテンツはこのリポジトリ内で Markdown と YAML フロントマターで管理されており、誰でも PR で追加・改善できる。エージェント用スキル（例: get-api-docs）を組み合わせると「chub を使って最新の API ドキュメントを取ってきて使え」とプロンプトするだけで済む。

2 — コマンド
一覧
コマンド目的
chub search [query]ドキュメント・スキルを検索。クエリなしで一覧。--tags / --lang / --limit で絞り込み。
chub get <id> [--lang py|js]ID でドキュメントまたはスキルを取得。言語指定でバリアント選択。--file / --full で参照ファイルを追加取得。
chub annotate <id> <note>そのドキュメントにローカルでメモを付与。次回 chub get 時に自動で末尾に付く。
chub annotate <id> --clearアノテーションを削除。
chub annotate --list全アノテーション一覧。
chub feedback <id> up|downドキュメントを up/down で評価。作者にフィードバックが送られ、品質改善に使われる。
グローバル: --json で JSON 出力（エージェントやパイプ向け）。--version / --help でバージョン・ヘルプ。
3 — 検索・取得・学習の流れ
基本フロー

多くの場合は「検索 → 取得 → エージェントが読んでコードを書く」で完結する。

chub search "stripe payments" → chub get stripe/api --lang js → エージェントがドキュメントを読んでコード生成

ギャップに気づいたら chub annotate stripe/api "Webhook 検証は raw body が必要" のようにメモを付ける。次回 chub get stripe/api したとき、取得結果の末尾にそのメモが自動で付与されるので、エージェントは前セッションの学びを忘れない。

With / Without Context Hub

Context Hub なし: ウェブ検索はノイズが多い・コードが壊れやすい・直す手間・知識はセッションで消える → 次のセッションで同じことを繰り返す。Context Hub あり: キュレーションされたドキュメントを取得・コードが動きやすい・ギャップはアノテーションで補い、次回以降に引き継がれる → セッションを重ねるほど賢くなる。

4 — コンテンツ・取得オプション
言語別・バージョン

ドキュメントは言語別（--lang py, --lang js など）のバリアントを持てる。複数言語ある場合は --lang を省略すると選択を促される。1 言語だけの場合は自動でその言語が選ばれる。

Incremental Fetch

1 本のドキュメントに参照ファイル（例: references/advanced.md）がある場合、chub get の出力末尾に「追加ファイルがあります」と案内が出る。--file <path> で必要なファイルだけ取得、--full で全部取得。トークンを無駄にしない。

chub get acme/widgets --file references/advanced.md
chub get acme/widgets --full

Claude-Mem

thedotmack/claude-mem — ツール観察を圧縮し、SQLite と Chroma でハイブリッド検索。Worker とライフサイクルフックでセッションをまたいでコンテキストを注入する OSS

1 — 概要
何を解決するか

Claude-Mem は Claude Code、Gemini CLI、OpenCode（--ide opencode）など向けに、セッションをまたいでプロジェクトの文脈を保持する。

ツール実行の観察を取り込み、セマンティック要約を生成。SQLite と Chroma によるハイブリッド検索で保持し、新しいセッション開始時に関連コンテキストを注入する。トークン効率のため Progressive Disclosure（段階的に詳細を取りに行く）が設計の軸になる。

2 — 構成要素
レイヤー

公式 README が示すコアは「フックで観察を拾う → Worker が処理・検索を提供 → DB とベクトルに載せる → 次セッションで注入」という一本線。ビルド済みフックは plugin/scripts/ に置かれ、各 IDE 向けインストーラが配置する。

Lifecycle Hooks

Claude Code のセッションイベントに接続。Smart Install はライフサイクル外の依存プリチェック。

Worker Service

既定ポート 37777。Web Viewer、検索系 HTTP、SSE など。README では検索系エンドポイントが 10 本と説明される。

SQLite

セッション・観察・要約を ~/.claude-mem/claude-mem.db に保存。スキーマ詳細は公式 Architecture の Database 章へ。

Chroma + Skills

ベクトル同期とハイブリッド検索。plugin/skills/mem-search/ などから HTTP API を叩く。

プライバシー

<private>…</private> で囲んだ範囲は保存対象外。タグ除去はフック層（例: src/utils/tag-stripping.ts）で行い、Worker や DB に渡す前に処理する想定だ。

3 — データの流れ
観察から注入まで

セッション中にフックが観察をキューへ載せ、Worker が要約・インデックス・ベクトル同期を担当。新セッションでは SessionStart やコンテキスト注入設定に従い、関連チャンクだけがプロンプトへ差し込まれる。

flowchart LR subgraph capture["取り込み"] A["IDE / CLI"] --> B["Lifecycle Hooks"] B --> C["Observation Queue"] end subgraph process["処理"] C --> D["Worker HTTP"] D --> E["Summarize / Index"] end subgraph store["永続化"] E --> F[("SQLite")] E --> G[("Chroma")] end subgraph inject["再利用"] F --> H["Context Injection"] G --> H H --> A end
MCP 経路 — エージェントは MCP ツールでメモリを検索し、3 層ワークフローでトークンを抑える。詳細は公式 Search Tools。

5 — MCP 検索（3 層）
トークン効率

公式 README のパターン: search でコンパクトなインデックスと ID → timeline で前後の時系列 → get_observations で必要 ID だけ全文取得。先に絞り込むことで詳細取得のトークンを大きく削る。

層ツール役割
1search全文検索・フィルタ付きのインデックス（目安 50–100 トークン/件）
2timeline注目観察の前後コンテキスト
3get_observationsID 指定で詳細のみ（複数 ID をまとめて）。目安 500–1000 トークン/件

ツール集合の最新一覧や追加の smart_* 系は Search Tools を正とする。

6 — 引用・ベータ・要件
引用と Viewer

観察には ID が付き、http://localhost:37777/api/observation/{id} や Web Viewer 上で参照できる。Claude Desktop から mem-search で辿る流れも README で案内されている。

ベータチャンネル

Viewer の Settings から安定版とベータを切り替え可能。Endless Mode などは Beta Features を参照。

Claude-Mem

Claude Code / Gemini CLI 向けの永続メモリ圧縮。セッションをまたいでコンテキストを保持し、ツール利用の観察を要約して将来のセッションに注入する。

1 — 概要
何か

Claude-Mem は Claude Code や Gemini CLI 向けのプラグインで、セッション終了や再接続後もプロジェクトの知識を継続させる。ツール使用の観察を自動で取り込み、セマンティック要約を生成し、SQLite と Chroma によるハイブリッド検索で保持。新しいセッション開始時に関連コンテキストを注入する。

12.5npm パッケージ版（2026-05 時点）
5+1ライフサイクル + Smart Install
37777Worker ポート
SQLite+ Chroma

主な機能: 永続メモリ、Progressive Disclosure、mem-search スキル、Web Viewer（localhost:37777）、Claude Desktop からの検索スキル、<private> タグ、観察 ID による引用、コンテキスト注入の細かい設定、ベータチャンネル（Endless Mode など）。

2 — アーキテクチャ
構成要素

5 つのライフサイクルフックでセッションイベントにフックする: SessionStart → UserPromptSubmit → PostToolUse → Stop → SessionEnd。あわせて依存関係チェック用の Smart Install（プリフック、ライフサイクル外）があり、フックスクリプトは README では計 6 本と説明される。ビルド成果物は plugin/scripts/ 配下に配置される。

コンポーネント説明
Worker ServiceHTTP API がポート 37777（デフォルト）で待ち受け、Web Viewer と検索系エンドポイント（README では 10 本の検索系）を提供。Bun で管理・起動。
Databasesrc/services/sqlite/。SQLite3 で ~/.claude-mem/claude-mem.db。セッション・観察・要約を保存。
Chromasrc/services/sync/ChromaSync.ts。ベクトル埋め込みでセマンティック検索。
Viewer UIsrc/ui/viewer/。React の Web UI。http://localhost:37777 で表示。ビルド成果は plugin/ui/viewer.html。
mem-search Skillplugin/skills/mem-search/SKILL.md。過去作業の検索用 HTTP API。履歴に関する質問で自動起動。
プライバシー

<private>content</private> でユーザーが指定した範囲は保存されない。タグの除去はフック層（src/utils/tag-stripping.ts）で行い、Worker や DB に渡る前に処理する。

3 — インストールと使い方
推奨（npx）

ターミナルで次を実行すると、フック・Worker・依存関係のセットアップまで一括で行える:

npx claude-mem install

Gemini CLI 向け（~/.gemini を自動検出）:

npx claude-mem install --ide gemini-cli

Claude Code または Gemini CLI を再起動すると、前セッションのコンテキストが新しいセッションに自動で載る。

マーケットプレイス（Claude Code）

IDE 内からインストールする場合:

/plugin marketplace add thedotmack/claude-mem

/plugin install claude-mem
注意: npm install -g claude-mem は SDK / ライブラリのみで、プラグインフックや Worker の登録は行われない。必ず npx claude-mem install か上記 /plugin 手順を使う。
OpenClaw

OpenClaw ゲートウェイに永続メモリとして導入する場合:

curl -fsSL https://install.cmem.ai/openclaw.sh | bash

インストーラが依存関係・プラグイン・AI プロバイダ設定・Worker 起動などをまとめて行う。Telegram / Discord / Slack などへのリアルタイム観察フィードも任意で設定可能。詳細は OpenClaw 連携ガイド。

4 — MCP 検索
3 層ワークフロー（核）

トークン効率のため、search → timeline → get_observations の 3 段階で絞り込んでから詳細を取得する。MCP にはこの流れを強制する説明用ツール __IMPORTANT も登録されている。さらに smart_search / smart_outline / smart_unfold（コードベース探索）やコーパス構築系（build_corpus など）も追加されている。一覧と例は Search Tools を参照。

レイヤーツール役割
1searchコンパクトなインデックスと ID を取得（結果あたり約 50–100 トークン）
2timeline気になる結果の前後の時系列コンテキストを取得
3get_observationsフィルタした ID の観察の詳細のみ取得（結果あたり約 500–1000 トークン）。複数 ID はまとめて指定する。

最初に search でインデックスを取り、必要な ID を選んでから get_observations で詳細を取ることで、トークンが約 10 倍節約できる。

5 — 引用・ベータ・要件
引用と Desktop

過去の観察を ID で参照できる。http://localhost:37777/api/observation/{id} や Web Viewer 上で確認可能。Claude Desktop の会話から mem-search スキルでメモリを検索する流れも README で案内されている。

ベータチャンネル

Web Viewer（localhost:37777）の Settings から安定版とベータを切り替え可能。Endless Mode など実験機能の詳細は Beta Features。

7 — 設定・参考リンク
設定

設定は ~/.claude-mem/settings.json。初回起動時にデフォルトで自動作成される。AI モデル、Worker ポート、データディレクトリ、ログレベル、コンテキスト注入の細かいオプションを変更できる。環境変数を含む完全な一覧は Configuration。

codex-plugin-cc

Claude Code から Codex でレビュー・委譲・ジョブ管理を行う OpenAI 公式プラグイン（marketplace: openai/codex-plugin-cc）

1 — 概要
位置づけ

既存の Claude Code ワークフローを保ったまま、Codex による読み取り専用レビューや、バックグラウンドでの調査・修正依頼（codex:codex-rescue サブエージェント経由）を統合するプラグイン。別ランタイムではなく、マシン上の Codex CLI と Codex app server に委譲する。

2 — 要件と導入
前提
  • 認証: ChatGPT 契約（無料枠含む）または OpenAI API キー。利用量は Codex の上限にカウントされる。
  • Node.js: 18.18 以上。
  • Codex CLI: グローバルに codex が通っていること。未インストールなら npm install -g @openai/codex、未ログインならシェルで codex login（README では !codex login と案内）。

マーケットプレイスと有効化

/plugin marketplace add openai/codex-plugin-cc
/plugin install codex@openai-codex
/reload-plugins
/codex:setup

/codex:setup で Codex の準備状態を確認できる。npm があれば Codex のインストール案内も可能。導入後はスラッシュコマンド一覧とサブエージェント codex:codex-rescue が /agents に現れる。

3 — スラッシュコマンド
一覧
コマンド役割備考
/codex:review未コミット差分または --base 指定ブランチの通常レビュー読み取り専用。--background / --wait 可。カスタムフォーカス文は不可。
/codex:adversarial-review設計・前提・失敗モードを問う steerable レビューフラグの後に自由文で焦点を指定可能。読み取り専用。
/codex:rescueバグ調査・修正・続きの作業を Codex に委譲codex:codex-rescue 経由。--resume / --fresh / --model / --effort / spark（→ gpt-5.3-codex-spark）等。
/codex:status実行中・直近のジョブ一覧または特定 task id—
/codex:result完了ジョブの保存出力（session id があれば codex resume へ）—
/codex:cancelバックグラウンドジョブのキャンセル—
/codex:setup環境チェック・任意の review gate 設定--enable-review-gate / --disable-review-gate
Review gate: 有効化すると Stop フックが Codex レビューを挟み、問題があれば Claude の応答をブロックするループになり得る。利用上限を急速に消費しうるため、監視できるときだけ有効にするのが README の推奨。
4 — 典型フロー
例

出荷前レビュー

/codex:review

長時間ジョブ

/codex:review --background
/codex:status
/codex:result

Codex へ引き継ぎ

/codex:result や /codex:status に表示されるセッション ID を使い、ターミナルで codex resume <session-id> として Codex 側で継続できる。

5 — 連携と設定
config.toml

デフォルトモデルや推論 effort は、ユーザー ~/.codex/config.toml またはプロジェクト直下の .codex/config.toml で Codex 本体と同じ形式で指定する。プロジェクト設定は trusted project のときのみ読み込まれる。

model = "gpt-5.4-mini"
model_reasoning_effort = "high"

エンドポイント変更などは Codex の設定ドキュメント の openai_base_url 等に従う。

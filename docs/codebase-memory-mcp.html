codebase-memory-mcp

Pure C の単一静的バイナリが tree-sitter と Hybrid LSP で知識グラフを構築し、MCP 経由で AI コーディングエージェントに構造クエリを返す。内蔵 LLM は持たず、接続先エージェントが自然言語→グラフクエリの翻訳を担う。

1 — 概要
Positioning

DeusData/codebase-memory-mcp は、リポジトリをフルインデックスして関数・クラス・呼び出しチェーン・HTTP ルート・クロスサービスリンクを永続グラフ化する MCP サーバー。平均規模のリポジトリはミリ秒、Linux カーネル（28M LOC / 75K ファイル）でも約 3 分でインデックス完了。

158languages
14MCP tools
11agents
<1msCypher query

ファイル逐次 grep/read と比べ、5 本の構造クエリで約 3,400 トークン（対 ~412,000 トークン、約 99.2% 削減）。処理は 100% ローカルで、コードは外部送信されない。

Source: README.md, server.json (v0.8.1)

Design stance

LLM を内蔵しない structural analysis backend。MCP クライアント（Claude Code 等）が既に持つ推論で NL→グラフクエリを行い、サーバーは SQLite 上のグラフを高速に返すだけ。

チーム共有: .codebase-memory/graph.db.zst をコミットすれば、clone 後の初回起動でアーティファクトを展開し増分インデックスのみ実行。merge=ours の .gitattributes でバイナリ競合を回避。
2 — 処理の流れ
Agent ↔ MCP ↔ Graph 
sequenceDiagram
  participant U as User
  participant A as MCP Client
(Claude Code 等)
  participant C as codebase-memory-mcp
  participant G as SQLite Graph

  U->>A: "ProcessOrder を呼ぶ関数は?"
  A->>C: trace_path(function_name="ProcessOrder", direction="inbound")
  C->>G: BFS / graph query
  G-->>C: structured call chain
  C-->>A: JSON results

エージェントは get_graph_schema でスキーマを把握してから search_graph / query_graph / trace_path を組み合わせる。コード断片が必要なら get_code_snippet、全体像は get_architecture。

Two-layer parsing

全 158 言語に tree-sitter パスを適用し、9 言語では Hybrid LSP で型解決を追加する二層構成。

Layer 2 — Hybrid LSP（型 aware）
Python, TS/JS/JSX/TSX, PHP, C#, Go, C/C++, Java, Kotlin, Rust。import グラフ + クロスファイル定義レジストリで CALLS / USAGE エッジを精緻化。別プロセスの language server は不要。
Layer 1 — tree-sitter（構文）
158 言語すべて。定義・呼び出し・import を高速抽出。Hybrid LSP 未対応言語はテキスト解決にフォールバック。
Persistence — SQLite @ ~/.cache/codebase-memory-mcp/
WAL モード、RAM-first インデックス後にダンプ。LZ4 圧縮読み込み → in-memory SQLite → 終了時 1 回書き出し。

7 — エージェント連携
install auto-detect

install はインストール済みエージェントを検出し、MCP エントリ・instruction ファイル・hooks を配置する。Claude Code では PreToolUse が Grep/Glob を傍受し、インデックス済みシンボルを search_graph 結果として additionalContext に注入（非ブロッキング、exit 0）。Codex / Gemini CLI / Antigravity は SessionStart リマインダー。

エージェントMCP 設定InstructionsHooks
Claude Code.claude/.mcp.json4 SkillsPreToolUse (Grep/Glob augment)
Codex CLI.codex/config.toml.codex/AGENTS.mdSessionStart
Gemini CLI.gemini/settings.json.gemini/GEMINI.mdBeforeTool + SessionStart
Zed / OpenCode / VS Code / …各 settings——
手動 MCP 設定: ~/.claude/.mcp.json またはプロジェクト .mcp.json にバイナリへの絶対パスを指定。/mcp で 14 tools が見えれば成功。

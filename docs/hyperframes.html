HyperFrames

Write HTML. Render video. Built for agents.

1 — 概要
何か

HyperFrames は、HTML ベースの動画コンポジションを作成・プレビュー・レンダリングするオープンソースの枠組み。エージェントがそのまま扱える 宣言的な HTML + data 属性 を正とし、CLI は非対話で自動化パイプラインに載せやすい設計になっている（heygen-com/hyperframes、Apache 2.0）。

2 — なぜ HyperFrames か
設計の要点
  • HTML-native — コンポジションは HTML + data 属性。React 専用 DSL ではなく、マークアップでタイムラインを表現する。
  • AI-first — エージェントは HTML を既に書ける。CLI はデフォルト非対話で、スクリプトや CI に載せやすい。
  • Deterministic rendering — 同じ入力なら同じ出力。自動生成パイプライン向け。
  • Frame Adapter — アニメーション実行環境を差し替え可能（GSAP、Lottie、CSS、Three.js など）。
カタログ

50 以上のブロック・コンポーネント（ソーシャルオーバーレイ、シェーダー遷移、データ可視化など）を npx hyperframes add <name> で追加できる。一覧は公式サイトの Catalog を参照。

3 — 構成とデータモデル
HTML でのコンポジション例

ルートに data-composition-id、解像度・開始時刻、各メディアに data-start / data-duration / data-track-index を付与し、ブラウザでプレビューしたあとキャプチャして動画化する。

<div id="stage" data-composition-id="my-video" data-start="0"
     data-width="1920" data-height="1080">
  <video id="clip-1" data-start="0" data-duration="5" data-track-index="0"
    src="intro.mp4" muted playsinline></video>
  <img id="overlay" class="clip" data-start="2" data-duration="3"
    data-track-index="1" src="logo.png" />
  <audio id="bg-music" data-start="0" data-duration="9" data-track-index="2"
    data-volume="0.5" src="music.wav"></audio>
</div>
レンダリングの流れ（概念）

コアが HTML を解釈し、エンジンがシーク可能なページ→映像キャプチャにし、プロデューサーがエンコードとオーディオミックスまで担当するイメージ。

Composition (HTML) → @hyperframes/core → @hyperframes/engine → @hyperframes/producer → MP4
Pipeline
flowchart TD H["HTML + data-*"] --> C["@hyperframes/core"] C --> E["@hyperframes/engine"] E --> P["@hyperframes/producer"] P --> M["MP4 / 出力"] C --> V["preview / lint CLI"]

5 — エージェント向け Skills
Vercel Skills 形式

HyperFrames は Vercel Skills として公開されており、npx skills add heygen-com/hyperframes で取得できる。エージェントにフレームワーク固有のパターンを教える用途。

hyperframes-registryhyperframes add によるブロック・コンポーネント導入

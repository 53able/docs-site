Understand Anything

マルチエージェント解析 + tree-sitter で、コード・ビジネスドメイン・Karpathy 型 wiki・Figma デザインをインタラクティブなナレッジグラフに変換

1 — 概要
What it is

Understand Anything（Egonex-AI/Understand-Anything、原作者 Lum1104）は、LLM と web-tree-sitter のハイブリッド解析で、コードベース・ビジネスドメイン・Karpathy 型 LLM wiki・Figma デザインをインタラクティブなナレッジグラフにするプラグイン。構造グラフではファイル・関数・クラス・依存を、ドメインビューでは業務プロセスを横方向に可視化。ダッシュボードはペルソナ（Junior / PM / PowerUser）に合わせた情報密度に調整できる。

分析結果の既定保存先は .ua/knowledge-graph.json。既存プロジェクトに .understand-anything/ があればそのまま継続（マイグレーション不要）。React Flow 製ダッシュボードで閲覧し、Git 差分 + fingerprint によるインクリメンタル更新が既定。初回の全解析はトークン消費が大きいので、大規模リポジトリではサブディレクトリ指定やローカルモデル（Ollama 等）を検討する。

Junior
新規参加の開発者向け。ガイドツアーでコードベースの全体像を素早く把握できる。

PowerUser
AI 開発者向け。自然言語クエリ・差分分析・影響範囲の即時確認。

3 — 解析フロー
/understand 実行時のパイプライン
flowchart TD A["/understand コマンド実行"] --> P0 P0["Phase 0 Pre-flight\nフル/インクリメンタル判定\n$UA_DIR 解決・language/auto-update"] --> B0 B0["Phase 0.5\n.understandignore 生成・確認"] --> B B["Phase 1 SCAN\nproject-scanner\nファイル探索・言語検出・importMap"] --> B15 B15["Phase 1.5 BATCH\nセマンティックバッチ分割"] --> C C["Phase 2 ANALYZE\nfile-analyzer × 最大5並列\ntree-sitter + LLM\n→ merge-batch-graphs.py"] --> C2 C2["Phase 3 ASSEMBLE REVIEW\nassemble-reviewer\nバッチ統合・セマンティック修復"] --> D D["Phase 4 ARCHITECTURE\narchitecture-analyzer\nAPI / Service / Data / UI / Utility"] --> E E["Phase 5 TOUR\ntour-builder\n依存順ガイドツアー"] --> F F["Phase 6 REVIEW\ngraph-reviewer\n既定: インライン / --review: LLM"] --> G G["Phase 7 SAVE\nknowledge-graph.json\nfingerprint baseline・meta.json"] --> H H["ダッシュボード自動起動"] style A fill:#0891b2,color:#fff,stroke:none style H fill:#0891b2,color:#fff,stroke:none style G fill:#0369a1,color:#fff,stroke:none
データディレクトリ: 既定は .ua/。既存の .understand-anything/ があればそちらを継続。

インクリメンタル更新（既定）: git diff <last-commit>..HEAD + fingerprint で変更ファイルを検出 → 該当ノード・エッジのみ再解析。全再解析は --full。file-analyzer はバッチごとに最大 5 並列（バッチあたり約 20〜30 ファイル）。

自動更新: /understand --auto-update で post-commit フックが有効化され、コミットごとにグラフを差分パッチ。--language ja などでノード説明・ダッシュボード UI・ツアーを指定言語で生成（設定は .ua/config.json に保存。初回は会話言語を検出し、英語以外は確認プロンプト）。
4 — コマンド一覧
ユーザー向けコマンド
コマンド機能モデル
/understand コードベース全体を解析してナレッジグラフを生成。フラグ: --full --review --auto-update --no-auto-update --language <lang>（ISO 639-1 または friendly name） Sonnet + Opus
/understand-dashboard ブラウザで React Flow ダッシュボードを起動 —
/understand-chat <質問> コードベースへの自然言語質問（グラフコンテキスト付き） Sonnet
/understand-diff 現在の変更影響範囲をグラフ上で分析 Sonnet
/understand-explain <パス> 特定ファイル・関数の詳細解説を生成 Opus
/understand-onboard 新メンバー向けオンボーディングガイドを生成 Opus
/understand-domain ビジネスドメイン・フロー・プロセスステップを抽出しドメイングラフを生成（既存グラフからの派生や軽量スキャンにも対応） Opus
/understand-knowledge Karpathy 型 LLM wiki（index.md・wikilink 等）を解析し、エンティティ・主張・トピッククラスタのグラフを生成 Sonnet + agents
/understand-figma <url-or-key> Figma REST API でデザインファイルを解析し kind:"design" グラフを生成。FIGMA_TOKEN 必須（オフライン不可）。UNDERSTAND_FIGMA_FORCE=1 で強制再構築 design-analyzer
呼び出しプレフィックス: 多くのプラットフォームは /understand。Codex は $understand（$ プレフィックス）。認識されない場合は自然言語で「understand skill を使ってこのプロジェクトを解析して」と依頼する。

Cursor

公式 README のとおり、本リポジトリをクローンして Cursor で開けば .cursor-plugin/plugin.json が自動検出され、追加インストールなしでスキルが有効になる。拾われない場合は Cursor Settings → Plugins に https://github.com/Egonex-AI/Understand-Anything を貼る。

git clone https://github.com/Egonex-AI/Understand-Anything ~/tools/understand-anything

ローカル開発

前提: Node.js ≥ 22（開発は v24 系で確認）、pnpm ≥ 10（ルート package.json の packageManager に準拠）

# 依存インストール
pnpm install

# core パッケージをビルド（ダッシュボードが依存）
pnpm --filter @understand-anything/core build

# ダッシュボード開発サーバー
pnpm dev:dashboard

# テスト
pnpm --filter @understand-anything/core test
pnpm --filter @understand-anything/skill test

# Lint
pnpm lint
tree-sitter に注意: 環境によってはネイティブ tree-sitter がビルドできないため、web-tree-sitter（WASM 版）を採用。ネイティブバイナリ前提の変更は動作しない。
8 — チームでグラフを共有
グラフは JSON — 一度コミットすれば全員がパイプラインをスキップできる

オンボーディング・PR レビュー・docs-as-code に向く。例: GoogleCloudPlatform/microservices-demo（Go / Java / Python / Node の参照グラフ付き）。

コミット対象

既定の .ua/ 配下のうち、intermediate/ と diff-overlay.json は除外（ローカル作業用）。レガシープロジェクトは .understand-anything/ に読み替える。

# .gitignore に追加する例
.ua/intermediate/
.ua/diff-overlay.json

鮮度を保つ

  • /understand --auto-update — post-commit フックでグラフを差分更新
  • リリース前に /understand を手動再実行
  • 10 MB 超の JSON は git-lfs で追跡

LLM なしでダッシュボードを開く

コミット済みグラフがあれば、Claude Code や API キーなしで viewer を起動できる（Node.js ≥ 18）。

npx https://github.com/Egonex-AI/Understand-Anything/releases/latest/download/understand-anything-viewer.tgz /path/to/analyzed/project

プロジェクト直下に .ua/（またはレガシー .understand-anything/）が必要。トークン付きローカル URL が開き、データはマシン外に出ない。

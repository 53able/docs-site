1 — 概要
What it is

Understand Anything は、Claude（Sonnet / Opus）と web-tree-sitter による静的解析を組み合わせ、コードベースだけでなく ビジネスドメインのフローや Karpathy 型 LLM wiki もインタラクティブなナレッジグラフとして扱えるプラグイン群。構造グラフではファイル・関数・クラス・依存を、ドメインビューでは業務プロセスを横方向に可視化。ダッシュボードはペルソナ（Junior / PM / PowerUser）に合わせた情報密度に調整できる。

分析結果は .understand-anything/knowledge-graph.json に保存され、React Flow 製ダッシュボードで閲覧。Git 差分ベースのインクリメンタル更新に対応。言語コンセプトとして 12 種のプログラミングパターンを文脈付きで解説する機能もある。

対象ユーザー

Junior
新規参加の開発者向け。ガイドツアーでコードベースの全体像を素早く把握できる。
PM / Designer
コードを読まない職種向け。アーキテクチャレイヤーを視覚的に確認できる。
PowerUser
AI 開発者向け。自然言語クエリ・差分分析・影響範囲の即時確認。

5 — コアデータ型
KnowledgeGraph ルート 
interface KnowledgeGraph {
  version: string;
  kind?: "codebase" | "knowledge";  // コード解析 vs ナレッジ wiki など
  project: ProjectMeta;
  nodes:   GraphNode[];   // 21 種のノード型（コード / 非コード / ドメイン / ナレッジ）
  edges:   GraphEdge[];   // 35 種類のエッジ型（8 カテゴリ）
  layers:  Layer[];       // アーキレイヤー (API/Service/Data/UI/Utility)
  tour:    TourStep[];    // ガイドツアーステップ
}

解析対象のプロジェクトだけをワークスペースにしているときは、File › Add Folder to Workspace… でクローンした Understand-Anything/ を追加し、./understand-anything-plugin/skills/ などの相対パスが解決できるようにする。

git clone https://github.com/Lum1104/Understand-Anything ~/tools/understand-anything

ローカル開発

前提: Node.js ≥ 22（開発は v24 系で確認）、pnpm ≥ 10（ルート package.json の packageManager に準拠）

# 依存インストール
pnpm install

# core パッケージをビルド（ダッシュボードが依存）
pnpm --filter @understand-anything/core build

# ダッシュボード開発サーバー
pnpm dev:dashboard

# テスト
pnpm --filter @understand-anything/core test
pnpm --filter @understand-anything/skill test

# Lint
pnpm lint
tree-sitter に注意: 環境によってはネイティブ tree-sitter がビルドできないため、web-tree-sitter（WASM 版）を採用。ネイティブバイナリ前提の変更は動作しない。

qiaomu-anything-to-notebooklm

Claude Code スキル · 多様なソース → Google NotebookLM · MIT

1 — 概要
位置づけ

Anything → NotebookLM（joeseesun/qiaomu-anything-to-notebooklm）は、自然言語で指示したソースを Google NotebookLM に載せ、ポッドキャスト・スライド・マインドマップ・クイズなどへ変換する Claude Code スキル。URL・ローカルファイル・検索キーワードを main.py とエージェント手順が型ごとに正規化し、notebooklm CLI でノートブック操作する。

15+入力ソース種別
300+ペイウォール対応サイト
6段回避カスケード
CLInotebooklm 連携

典型フローは「WeChat 記事をポッドキャストに」「有料記事をマインドマップに」「小宇宙（中国のポッドキャストアプリ）の番組をスライドに」など、ユーザー発話 → ソース取得 → NotebookLM アップロード → （任意）成果物生成、という一本線（README・SKILL.md）。

2 — 処理パイプライン
エンドツーエンド 
flowchart LR
  IN[入力 URL / ファイル / 検索語]
  DET[detect_input_type]
  GET[取得・変換]
  UP[notebooklm source add]
  GEN[generate（任意）]
  OUT[MP3 / PDF / JSON 等]
  IN --> DET --> GET --> UP --> GEN --> OUT
識別 → 取得 → アップロード → 生成（任意）

main.py の detect_input_type() は HTTP URL・拡張子・存在しないパス（検索語扱い）で分岐する。WeChat は MCP、YouTube は URL を NotebookLM に直接渡す（yt-dlp 等は SKILL で禁止）、ポッドキャスト系は Get笔记 API 経由の TXT、一般 URL は fetch_url.sh または NotebookLM 直取り。

デフォルト: 生成意図が無いときはアップロードのみ。明示された自然言語（「ポッドキャストに」「スライドに」など）が SKILL.md の意図表にマップされる。

同梱サブモジュール

feishu-read-mcp/ は Feishu（飞书）ドキュメント連携用 MCP（深度分析 + Feishu へ書き出すフローで参照）。WeChat 用 MCP は SKILL 上 wexin-read-mcp（表記は README と SKILL でディレクトリ名が揺れる）として別途設定する。

4 — ペイウォール回避
6 段階のカスケード

README の回避戦略を要約。失敗時に次段へフォールバックする。

  1. 代理サービス（r.jina.ai / defuddle.md）
  2. サイト別 Bot UA（Googlebot / Bingbot）
  3. UA 偽装 + X-Forwarded-For + Referer + AMP + EU IP
  4. archive.today（CAPTCHA 検知あり）
  5. Google Cache
  6. agent-fetch ローカルツール

対象例: NYT, WSJ, FT, Economist, Bloomberg, Medium, SCMP など（README の表）。実装の多くは scripts/fetch_url.sh に集約されている。

利用上の注意: 各サイトの利用規約・著作権・地域法に従う。本スキルは技術的回避の説明であり、利用者の責任で適法に使うこと。

6 — セットアップ
前提
  • Python 3.9+（README バッジ）
  • notebooklm CLI — 初回 notebooklm login、notebooklm list で認証確認
  • WeChat MCP — Claude の config.json に weixin-reader サーバを登録（SKILL.md の JSON 例。パスは各マシンのスキル配置に合わせる）
  • ポッドキャスト文字起こし — GETNOTE_API_KEY / GETNOTE_CLIENT_ID と Get笔记トークン（SKILL.md）
  • markitdown — Office / PDF / 画像・音声の変換

リポジトリ付属の install.sh / check_env.py で依存チェックができる（ファイル一覧より）。

notebooklm login
notebooklm create "my-notebook"
notebooklm source add https://example.com/article
python main.py /path/to/book.epub --deep-analysis
python main.py /path/to/book.epub --deep-analysis --to-feishu

OpenUI

The Open Standard for Generative UI — ストリーミングファーストの構造化 UI 言語と React ランタイム

1 — 概要
何か

OpenUI はフルスタックのジェネラティブ UI フレームワーク。コンパクトなストリーミングファーストの言語（OpenUI Lang）、React ランタイム、組み込みコンポーネントライブラリ、そのまま使えるチャット UI を提供し、JSON と比べて最大 67% トークン削減を実現する。

OpenUI Lang構造化 UI 言語
Reactランタイム
67%トークン削減
Zod型付きコンポーネント

中核: OpenUI Lang はモデル出力用のコンパクトなストリーミング向け言語。コンポーネントライブラリを定義し、そこからプロンプトを生成。モデルがストリームする OpenUI Lang をパースして React で段階的にレンダリングする。

3 — クイックスタート
CLI で作成 
npx @openuidev/cli@latest create --name genui-chat-app
cd genui-chat-app
echo "OPENAI_API_KEY=sk-your-key-here" > .env
npm run dev

スキャフォールドされたアプリにはストリーミング・組み込み UI・OpenUI Lang 対応が含まれる。プロンプトは許可したコンポーネント集合から自動生成され、ストリーム到着とともに UI が更新される。

4 — 仕組み
フロー

コンポーネントが「モデルが生成できるもの」を決める。

flowchart LR A["Component Library"] --> B["System Prompt"] B --> C["LLM"] C --> D["OpenUI Lang Stream"] D --> E["Renderer"] E --> F["Live UI"]
  1. コンポーネントライブラリを定義または再利用する。
  2. そのライブラリからシステムプロンプトを生成する。
  3. プロンプトをモデルに送る。
  4. OpenUI Lang のストリームをクライアントに返す。
  5. Renderer で段階的にレンダリングする。

Playground でデフォルトコンポーネントライブラリを使ってリアルタイムに UI を生成できる。

5 — OpenUI Lang
設計

OpenUI Lang は構造化かつストリーミング可能なモデル出力用に設計されている。

  • ストリーミング出力 — トークン到着に合わせて UI を段階的に出力。
  • トークン効率 — 同等の JSON より最大 67% 少ないトークン（ベンチマーク参照）。
  • 制御されたレンダリング — 定義・登録したコンポーネントだけに出力を制限。
  • 型付きコンポーネント — Zod スキーマで props と構造を事前定義。

Defuddle

Web ページからノイズを除き、本文だけをきれいに取り出す。Obsidian Web Clipper や HTML→Markdown 変換の前段に。

1 — 概要
何か

Defuddle（de·fud·dle）は、Web ページから「本文」だけを抽出するライブラリ。コメント・サイドバー・ヘッダー・フッター・広告など不要な要素を除き、読みやすい一貫した HTML を出力する。Obsidian Web Clipper 向けに書かれており、Turndown などの HTML→Markdown 変換の入力として使いやすい形を目指している。

Mozilla Readability の代替としても使える。Readability との主な違いは次のとおり。

  • より寛容で、不確実な要素を削りすぎない
  • 脚注・数式・コードブロックなどを一貫した形式に整える
  • ページのモバイル用スタイルを手がかりに不要要素を推測する
  • schema.org などメタデータをより多く抽出する

ブラウザ・Node.js・CLI の 3 通りで利用可能。Playground で動作を試せる: defuddle.md/playground

2 — 使い方
ブラウザ 
import Defuddle from 'defuddle';

const defuddle = new Defuddle(document);
const result = defuddle.parse();

console.log(result.content);
console.log(result.title);
console.log(result.author);
Node.js

package.json に "type": "module" が必要。Node 用は defuddle/node から読み、JSDOM を peer に持つ。

import { JSDOM } from 'jsdom';
import { Defuddle } from 'defuddle/node';

const html = '<html><body><article>...</article></body></html>';
const result = await Defuddle(html);

// URL から
const dom = await JSDOM.fromURL('https://example.com/article');
const result = await Defuddle(dom);

// オプション付き
const result = await Defuddle(dom, url, { debug: true, markdown: true });
CLI 
# ローカル HTML をパース
defuddle parse page.html

# URL をパース
defuddle parse https://example.com/article

# Markdown で出力
defuddle parse page.html --markdown

# JSON（メタデータ付き）で出力
defuddle parse page.html --json

# 特定プロパティだけ
defuddle parse page.html --property title

# ファイルへ保存
defuddle parse page.html --output result.html

# デバッグ
defuddle parse page.html --debug
オプションエイリアス説明
--output <file>-o標準出力の代わりにファイルへ書き出す
--markdown-mMarkdown 形式で出力
--json-jメタデータと content を JSON で出力
--property <name>-p指定プロパティ（title, description, domain など）のみ抽出
--debug—デバッグモード

4 — バンドル
3 種類
バンドルインポート説明
Coredefuddleブラウザ用メイン。依存なし。数式は扱うが MathML↔LaTeX のフォールバックはなし
Fulldefuddle/fullmathml-to-latex と temml で信頼性の高い <math> 生成など追加機能
Nodedefuddle/nodeJSDOM 向け。数式・Markdown 変換を含む。Node ではこちらを推奨

インストール: npm install defuddle。Node 利用時は npm install jsdom も必要。

5 — 標準化・デバッグ
HTML 標準化

standardize: true のとき、見出し・コードブロック・脚注・数式を統一フォーマットに整える。例: 最初の H1/H2 がタイトルと一致すれば削除、H1→H2 に変換、コードは data-lang と class="language-*" を付与、脚注は <sup id="fnref:*"> と <div id="footnotes"> の形式に、数式は MathML に統一する。

デバッグ

debug: true にすると、result.debug.contentSelector（選ばれたメインコンテンツのセレクタ）と result.debug.removals（除去された要素の一覧と理由）が得られる。パイプラインの特定ステップだけ止めたい場合は removeLowScoring: false や removeHiddenElements: false などを指定する。メインコンテンツを明示したい場合は contentSelector: 'article.post-content' のように指定する（一致しなければ自動検出にフォールバック）。

第三者の API: parseAsync() でローカル HTML から本文が取れない場合、Defuddle は FxTwitter API など外部 API にフォールバックすることがある（クライアントレンダリングの SPA など）。無効にしたい場合は useAsync: false を指定する。

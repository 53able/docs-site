Academic Research Skills（Imbad0202/academic-research-skills）

Claude Code 向け学術研究スキル群 · v3.9.0 · CC BY-NC 4.0

1 — 概要
位置づけ

Academic Research Skills（ARS） は、調査から執筆・査読・最終化までをカバーする Claude Code 向けスキルスイート。論文を代筆するのではなく、文献探索・引用整形・整合性検証などの反復作業をエージェントに任せ、研究設計・解釈・主張の文言は人間が担う前提で設計されている（README）。

4コアスキル
10パイプライン段
42+専門エージェント
v3.9現行リリース
人間参加型: 各ステージはユーザー確認チェックポイントで閉じる。Stage 2.5 / 4.5 の整合性ゲートは機械検証のあとユーザーがレポートを承認してから進む（docs/ARCHITECTURE.md）。

プラグイン導入（推奨）:

/plugin marketplace add Imbad0202/academic-research-skills
/plugin install academic-research-skills

チェックポイントの種類

種類例ユーザーが決めること
分岐 Stage 1, 2, 3, 5 RQ 承認、アウトライン、Accept/Minor/Major/Reject、出力形式
整合性 Stage 2.5, 4.5 機械検証後の Integrity Report を承認（スキップ不可）
コーチング 3→4, 3'→4' ソクラテス対話（最大 8 / 5 ラウンド、スキップ可）
Observer はブロックしない: collaboration_depth_agent は各 FULL/SLIM チェックポイントと完了時に助言のみ。Mandatory な 2.5 / 4.5 では意図的にスキップし、コンプライアンス検査の希釈を防ぐ。
4 — 信頼性・データ契約
Material Passport と検証層
raw
deep-research → redacted
academic-paper → verified_only
reviewer / gates

各スキルは data_access_level（raw / redacted / verified_only）と task_type を宣言し、scripts/check_data_access_level.py で検証される（v3.3.2+）。パイプライン横断の状態は Material Passport（Schema 9）に集約。任意で repro_lock を付与できるが、LLM 出力のバイト再現は保証しない（設定ドキュメント扱い）。

AI 研究の失敗モード（Stage 2.5 / 4.5）

Lu et al. (2026, Nature) の完全自動研究の限界を踏まえ、7 モードのブロッキングチェックリストを実行（M1 実装バグ〜 M7 フレームロック）。詳細は academic-pipeline/references/ai_research_failure_modes.md。

引用・Claim Audit（v3.7–v3.9）

  • v3.7.3: 三層 citation anchor（将来の claim 監査用ロケータ）
  • v3.8: ARS_CLAIM_AUDIT=1 で引用先を取得し claim 支持を判定。5 種の HIGH-WARN が formatter の REFUSE 条件
  • v3.9: クロスインデックス triangulation 計測（仕様は design doc 参照）

文献コーパスは Zotero / Obsidian / フォルダスキャン等のアダプタで Material Passport に取り込み、corpus-first の 4 Iron Rules で検索と整合（v3.6.5+）。

5 — 導入と運用
前提と検証
  • Claude Code（プラグインは v3.7.0+ 推奨）、ANTHROPIC_API_KEY
  • 任意: Pandoc（DOCX）、tectonic + Source Han Serif TC（APA PDF）
  • 任意: ARS_CROSS_MODEL（クロスモデル DA）、ARS_CLAIM_AUDIT=1

初回確認: /ars-plan でテーマを述べるか、/ars-lit-review "topic" で単発テスト。

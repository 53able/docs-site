open-slide

自然言語とコーディングエージェントで React スライドを書き切るためのランタイム（MIT）

1 — 概要
位置づけ

各スライドは 1920 × 1080 の固定キャンバス上に描画される任意の React コンポーネントである。DSL に閉じず、エージェントが普段どおり TSX を書ける。ホットリロード、ページ遷移、プレゼンモード、静的 HTML と PDF のエクスポートまでをフレームワーク側が担う。

1920×1080キャンバス
Reactページ表現
Vite開発・ビルド
MITライセンス

公開ソースは GitHub の 1weiho/open-slide。npm では @open-slide/cli などがスキャフォールドとテンプレート配布に使われる。

2 — エージェント連携
同梱スキルとコマンド

初期化されたワークスペースには、スライド作成と実装ルールを読ませるためのエージェントスキルが含まれる。

  • /create-slide — トピック・トーン、枚数、テキスト量、モーションの有無を質問し、デッキ ID と構成を決めてから各ページの TSX を生成する。
  • /slide-authoring — キャンバス寸法、タイポの段階、配色、レイアウト規約など、実装前にエージェントが参照する技術メモ。
  • /apply-comments — インスペクタで付けた @slide-comment をソースに反映し、マーカーを消す。
運用上の前提: Claude Code 向けにスキルが揃えられているが、README の記述どおり Codex や Cursor など他のエージェントでも同じリポジトリを編集する使い方が想定されている。
3 — ランタイム機能
開発・配布
  • インスペクタ: 開発サーバー上で要素をクリックし、コメントを付与。コメントはソース内の @slide-comment として保持される。
  • アセット: 画像・動画・フォントをデッキ単位で管理。ブランド SVG は svgl カタログから検索して取り込める。
  • プレゼン: 全画面再生、キーボード操作、プレゼンターモード（現在／次スライド、ノート、タイマー）。
  • エクスポート: 自己完結した静的 HTML サイト、または印刷向け PDF を CLI から出力。
  • スライドマネージャ: 複数デッキをフォルダと絵文字で整理し、ドラッグで並べ替え。
デプロイ

成果物は静的ファイルのみ。Vercel、Cloudflare Pages、Netlify 等の静的ホスティングにそのまま載せられる（サーバ常駐ランタイムは不要）。

4 — 編集ループ
プレビューとコメントの往復

README が示すループを、処理の順序として整理した。

flowchart LR
  A[プレビュー] --> B[要素クリック]
  B --> C["@slide-comment 付与"]
  C --> D["/apply-comments"]
  D --> E[ソース更新]
  E --> A
5 — モノレポ
パス役割
packages/core（@open-slide/core）ホーム、スライドビューア、プレゼン、インスペクタ、Vite プラグイン、open-slide の dev / build / preview CLI。
packages/cli（@open-slide/cli）npx @open-slide/cli init による最小ワークスペースの生成。設定や React の詳細は core に隠蔽。
apps/demoローカル開発用。workspace 参照で @open-slide/core を消費するサンプルデッキ。
apps/webドキュメント／マーケ用の Web アプリ（公式サイト側の構成）。

ルートは pnpm ワークスペースと Turbo でオーケストレーション。品質ゲートは Biome と Vitest が担う。

6 — セットアップ

新規デッキ

npx @open-slide/cli init my-slide
cd my-slide
pnpm dev

エージェント運用の制約事項はリポジトリ直下の CLAUDE.md にまとまっている。スライド本体は slides/<id>/index.tsx から編集する。

フレームワーク本体の開発

pnpm install
pnpm dev
pnpm build
pnpm check
pnpm lint

peon-ping

PeonPing/peon-ping — AI コーディングエージェントのイベントを音声・大画面オーバーレイ・タブタイトルで知らせる（MIT）

公式サイトは peonping.com。ソースは GitHub: PeonPing/peon-ping。各 IDE のフックから共通の peon.sh 経由で再生・通知し、Coding Event Sound Pack Specification（CESP） に沿ったカテゴリでパックを差し替えられる。

エージェントがバックグラウンドで完了・エラー・権限要求に至っても、別アプリにフォーカスがあると気づきにくい。peon-ping は 聴覚と視覚（オーバーレイ／システム通知）とターミナルタブのタイトルを組み合わせ、状態変化を短いループで取りこぼしにくくする。

macOS では JXA ベースの大画面オーバーレイ、テーマ（neon / glass / sakura / jarvis など）、通知位置・自動消去が peon setup や CLI で切り替え可能。標準通知モードでは terminal-notifier 等との連携も README に記載。

CLI と設定

実行ファイル名は peon。ミュートは /peon-ping-toggle（Claude Code）または peon toggle。対話セットアップは peon setup。設定はグローバルなら $CLAUDE_CONFIG_DIR/hooks/peon-ping/config.json（省略時は ~/.claude/hooks/peon-ping/config.json）、ローカルインストール時は ./.claude/hooks/peon-ping/config.json。

# 例: 音量と通知の確認
peon status
peon volume 0.7
peon notifications overlay

独立した 3 スイッチ: enabled（音声マスタ）、desktop_notifications（デスクトップ）、mobile_notify.enabled（モバイル push）。README の表どおり、音だけ・通知だけといった組み合わせが可能。

MCP サーバ

リポジトリ内の mcp/ に Node 製サーバがある。エージェントから play_sound でキー（例: duke_nukem/Groovy）を指定して再生。カタログは MCP Resource として公開され、モデルが一覧を一度読める設計。

MCP 機能役割
Tool play_soundsound または sounds 配列で再生
Resource peon-ping://catalog全パック・全サウンドのカタログ
Resource peon-ping://pack/{name}指定パックのカテゴリ別一覧

要件は Node.js 18+ と、ホストに peon-ping が入っていること。MCP クライアントの設定では node と peon-mcp.js へのパス（クローンしたプロジェクト内の絶対パス、またはインストール先）を指定する。

Tamagui（tamagui/tamagui）

Universal style · Optimizing compiler · UI kit — Web と Native で UI コードを共有する

1 — 概要
位置づけ

Tamagui は、React と React Native の両方で動く UI を、パフォーマンスとコード品質を落とさずに共有するためのライブラリ群。スタイルライブラリ・最適化コンパイラ・UI キットが一体となり、コンパイル時とランタイムの両方で動作する（README の説明に基づく）。

2 — 三層スタック
パッケージの役割
@tamagui/core + @tamagui/static → tamagui
パッケージ役割典型利用
@tamagui/coreReact 向けユニバーサルスタイルライブラリ（styled()、トークン、テーマ）低レベル API・カスタムデザインシステム
@tamagui/staticcore と tamagui 向け最適化コンパイラVite / Webpack / Next プラグイン経由でビルド時抽出
tamagui全プラットフォーム向け UI キット（Button、Sheet、テーマ等）アプリ本体のコンポーネント import

コア API の要点

  • styled(Component, { ... }) — バリアントは as const 必須。トークンは $4 のように $ 接頭辞。
  • XStack / YStack / ZStack — 行・列・重ね配置のプリミティブ。
  • Theme — ネスト可能なテーマ階層（name + 子テーマ）。
import { View, styled } from '@tamagui/core'
import { YStack, Button, Theme } from 'tamagui'

const Card = styled(View, {
  padding: '$4',
  backgroundColor: '$background',
  borderRadius: '$4',
  variants: { elevated: { true: { shadowRadius: 10 } } } as const,
})

<Theme name="dark">
  <YStack gap="$4">
    <Card elevated />
    <Button>Action</Button>
  </YStack>
</Theme>
3 — モノレポ構成
code/ 配下

  core/         # web, themes, config, animations, cli …

  kitchen-sink # Playwright / Detox 統合テスト
  starters/     # create-tamagui テンプレート
  packages/     # build, lucide-icons, native-ci …
  sandbox      # SSR・ハイドレーション検証

4 — 最適化コンパイラ
ビルド時の変換

コンパイラは styled コンポーネント（複雑なロジックやクロスモジュール import を含む）を、プラットフォーム向けに平坦化する（README）。

出力先ランタイム上の形
Web単純な div + アトミック CSS
NativeView とスタイルオブジェクトのホイスト
 
flowchart LR
  SRC["styled コンポーネント TSX"]
  EX["@tamagui/static 抽出"]
  WEB["Web: div + atomic CSS"]
  NAT["Native: View + hoisted styles"]
  SRC --> EX
  EX --> WEB
  EX --> NAT

README の計測例: 公式サイトのレスポンシブ領域で約 55 個のインライン styled のうち 49 が div に平坦化され、コンパイラ ON で Lighthouse が約 15% 改善。

段階的導入: 最初は import してベース View と styled だけ使う程度から始められ、コンパイラと UI キットは後から足せる。
5 — 開発フロー
新規プロジェクトと CI 
# スターター（学習用〜本番モノレポ）
npm create tamagui@latest

# リポジトリ開発（ルート）
bun install
bun run build          # turbo run build
bun run check          # deps / unused / tamagui check / references
bun run test           # テストスイート

# 公式サイト
bun run dev            # code/tamagui.dev

# kitchen-sink（統合テスト）
cd code/kitchen-sink
bun run start:web      # http://localhost:7979
bun run test:web       # Playwright（複数 animation driver）

kitchen-sink での検証

  • ユースケース: code/kitchen-sink/src/usecases/ — URL ?test=CaseName で単体表示。
  • デモ: code/demos/src/ — ?demo=Select（SelectDemo.tsx なら Select）。
  • アニメーションドライバ: css · native · reanimated · motion を並列プロジェクトで回す。

パッケージを編集したら、ルートで bun run watch を回すか、対象パッケージで bun run build する（CONTRIBUTING / claude.md）。

6 — エージェント向け
Tamagui Skill

リポジトリ内 plans/tamagui-skill/skills/tamagui/SKILL.md は、クロスプラットフォーム UI 実装時のトリガー（styled()、$token、@tamagui/* 等）を定義する。

プロジェクト固有のトークン・テーマ名を推測せず、次で設定をダンプする:

npx tamagui generate-prompt
# → tamagui-prompt.md（tokens, themes, media, shorthands）
ルール内容
トークン$ 付きで参照（$background, $4）
variantsオブジェクトに as const を付けて型推論を固定
設定の正tamagui-prompt.md を優先（デフォルト値の憶測を避ける）

Tamagui（tamagui/tamagui）

Bun モノレポ · @tamagui/core · @tamagui/static · tamagui UI — Web と React Native でコードを共有

1 — 概要
位置づけ

Tamagui は、React と React Native の両方で動くスタイルライブラリ・デザインシステム・コンポーネント群の集合体。Web とネイティブで UI コードを共有しつつ、パフォーマンスと型・DX を維持することを目的とする（README および公式ドキュメントの説明に基づく）。

ランタイムの styled() とテーマトークンに加え、最適化コンパイラがビルド時に styled コンポーネントを Web では atomic CSS 付きの div、ネイティブではスタイルオブジェクトをホイストした View へ平坦化する。公式サイトの Hero セクションでは、コンパイラ有効時に Lighthouse スコアが約 15% 改善した例が示されている。

スタイルの書き方（2 パターン）

インライン props と styled() + variants の両方をサポートする（library 内ドキュメントより）。

import { View, Text, styled } from 'tamagui'

export default () => (
  <>
    <View backgroundColor="red" />
    <Text fontSize={16} color="$color" />
  </>
)

const Card = styled(View, {
  borderRadius: 12,
  variants: {
    size: {
      sm: { padding: 8 },
      lg: { padding: 20 },
    },
  } as const,
})

variants では boolean・文字列キー・'...size' のようなトークン spread など、型安全な条件スタイルを 1 コンポーネントに集約できる。

4 — 最適化コンパイラ
ビルド時 extract

@tamagui/static は単体では使わず、@tamagui/vite-plugin・@tamagui/babel-plugin（Metro）・@tamagui/next-plugin などのバンドラ統合経由で有効化する（library/tamagui.mdx）。

flowchart LR
  SRC[styled コンポーネント TSX]
  EXT[static extractor]
  WEB[Web: div + atomic CSS]
  NAT[Native: View + hoisted styles]
  SRC --> EXT
  EXT --> WEB
  EXT --> NAT
  • 複雑なロジックやクロスモジュール import を含む styled でも、多くが単純な DOM / View に平坦化される
  • コンパイル時とランタイムの両方で動作。段階的に extract 範囲を広げられる
  • 開発時は TAMAGUI_EXTRACT=1 付きの dev:extract で extract 挙動を確認可能（ルート scripts）
パッケージ変更時: ルートで bun run watch を回すか、変更したパッケージディレクトリで bun run build しないと downstream が古いビルドを参照する（CONTRIBUTING / claude.md）。

6 — UI キットとテスト
kitchen-sink

code/kitchen-sink がコンポーネント統合テストの中心。Web は bun run start:web（既定ポート 7979）で起動し、Playwright が複数アニメーション driver（css / native / reanimated / motion）に対して走る（claude.md）。

テスト種別ファイル命名実行回数
通常ComponentName.test.tsx既定 driver で 1 回
アニメーション依存ComponentName.animated.test.tsx全 driver で繰り返し

ブラウザ確認: ?test=SelectFocusScopeCase（usecases 名）や ?demo=Select（demos の Demo 名から Demo を除いたもの）。

周辺プロダクト（ルート scripts より）

  • kitchen-sink — クロスプラットフォーム検証アプリ
  • starter / create — 学習用〜本番向け monorepo テンプレート
  • takeout / studio — 商用テンプレ・スタジオ系（@tamagui スコープ）
  • sandbox — 実験用サンドボックス
7 — 導入
新規プロジェクト 
npm create tamagui@latest
# または
bun create tamagui

最小構成では tamagui を import し View / Text / styled から開始。本番最適化では Vite または Metro の Tamagui プラグインを設定し、テーマ config（tamagui.config.ts）をプロジェクトに置く。

このリポジトリを clone して開発する場合は Bun 1.3 系を使い、初回 bun install 後に bun run build または bun run watch を推奨する。

vercel-labs/skills

npx skills — オープンなエージェントスキルエコシステム向け CLI。GitHub・GitLab・git URL・ローカルパスから SKILL.md を取得し、Claude Code・Cursor・Codex など 70 以上のコーディングエージェントへシンリンクまたはコピーで配布する。

1 — 概要
CLI role

skills は Agent Skills 仕様 に沿った指示セット（SKILL.md）を、リポジトリからエージェント固有の skills ディレクトリへ届ける配布レイヤー。スキル本体は各リポジトリ（例: vercel-labs/agent-skills）にあり、CLI は探索・取得・配置・更新を担う。

70+対応エージェント
v1.5.11 最新
7主要コマンド
MITライセンス

出典: README.md、package.json、src/cli.ts

Typical flow

リポジトリを指定して add → エージェント skills ディレクトリへ配置 → list で確認 → update で最新化。試すだけなら use でインストールせずプロンプト生成もできる。

スキルとは: YAML frontmatter に name と description を持つ Markdown 指示書。PR 作成手順・リリースノート生成・外部ツール連携など、エージェントの専門タスクを再利用可能にする。

3 — use コマンド
Try without install

skills use は add と同じソース解決を行い、選択した 1 スキルを一時ディレクトリへ展開したうえで生成プロンプトを stdout に出力する。--agent を付けると対応エージェントを対話起動する。

examples
# プロンプトをパイプで Claude に渡す
npx skills use vercel-labs/agent-skills@web-design-guidelines | claude

# エージェントを直接起動
npx skills use vercel-labs/agent-skills --skill web-design-guidelines --agent claude-code

形式例

add オプション（代表）
オプション説明
-g, --globalユーザーディレクトリ（~/ 配下）へインストール
-a, --agent <agents>対象エージェント指定（* で全エージェント）
-s, --skill <skills>特定スキルのみ（* で全スキル）
-l, --listインストールせずリポジトリ内スキルを一覧
--copyシンリンクの代わりにファイルコピー
--full-depthルート SKILL.md があっても全サブディレクトリを探索
-y, --yes確認プロンプトをスキップ（CI 向け）
--all--skill '*' --agent '*' -y の省略形
ci-friendly
npx skills add vercel-labs/agent-skills --skill frontend-design -g -a claude-code -y

6 — スキル探索
Discovery rules

CLI はリポジトリ内のコンテナディレクトリを最大 2 階層まで走査する。フラット配置（skills/<name>/SKILL.md）とカタログ配置（skills/<category>/<name>/SKILL.md）の両方に対応。浅い階層で見つかった SKILL.md は深い階層より優先される。

探索対象備考
リポジトリルート（SKILL.md がある場合）単一スキルリポジトリ向け
skills/、skills/.curated/ 等標準コンテナ。エージェント固有パス（.claude/skills/ 等）も列挙
.claude-plugin/plugin.jsonClaude Code プラグインマーケットプレイス互換。manifest 宣言パスは深度制限なし
--full-depthexamples/ や tests/ などコンテナ外も再帰探索
SKILL.md 必須フィールド 
---
name: my-skill
description: このスキルの目的と使うタイミング
---

# My Skill
エージェントが実行する手順を記述する。

metadata.internal: true のスキルは通常探索から除外。INSTALL_INTERNAL_SKILLS=1 で表示・インストール可能。

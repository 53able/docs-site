汎用 AI はテキストを返す。PM Skills Marketplace は構造を返す。各スキルは Teresa Torres・Marty Cagan・Alberto Savoia らが確立した PM フレームワークをエンコードし、ステップバイステップで実行を導く。

ディスカバリーから戦略、実行、ローンチ、グロースまでをカバーする 8 プラグイン構成。Claude Code と Cowork 向けに設計されているが、スキル単体は Gemini CLI / OpenCode / Cursor / Codex CLI / Kiro など任意のエージェントで使える。

2 — 仕組み
3 つの構成要素

「スキル」「コマンド」「プラグイン」が階層的に組み合わさって動く。コマンドは完了時に次の推奨コマンドを提案するため、PM ワークフロー全体が自然につながる。

Skills スキル — 知識の最小単位

各スキルは PM ドメインの知識・分析フレームワーク・ガイドワークフローをエンコードする。会話に関連するスキルは自動ロードされるが、/plugin-name:skill-name で明示的に呼び出すことも可能。

Commands コマンド — スラッシュ起動のフロー

/discover などのコマンドはスキルをチェーンして、エンドツーエンドのプロセスを走らせる。例: /discover は「アイデア出し → 仮定洗い出し → 優先順位付け → 実験設計」の 4 スキルを連続実行する。

Plugins プラグイン — 領域ごとの束

8 プラグインが PM ドメインを分割して担当する。マーケットプレイスをインストールすると全 8 プラグインが一括導入される。複数コマンドが同一スキルを共有することもある。

リファレンススキルについて — prioritization-frameworks や opportunity-solution-tree のようなスキルはコマンドを介さず、会話に関連すれば自動で参照される。スラッシュコマンドは Claude 専用だが、スキルはどの AI ツールでも動く。
3 — プラグイン一覧
pm-product-discovery 13 skills 5 cmds

プロダクトディスカバリー

新規アイデアや既存プロダクトの不確実性を、仮定・機会・実験に分解する領域。ふわっとした「何を作るか」を、検証可能な学習計画に落とす。

Skills

  • brainstorm-ideas-* と brainstorm-experiments-* で、既存/新規プロダクト別にアイデアと実験を広げる。
  • identify-assumptions-* と prioritize-assumptions で、Value / Usability / Viability / Feasibility などのリスクを整理する。
  • opportunity-solution-tree、interview-script、summarize-interview で、顧客理解を構造化する。

Commands

  • /discover は「アイデア出し → 仮定洗い出し → 優先順位付け → 実験設計」を一気通貫で回す。
  • /brainstorm は ideas / experiments と existing / new を組み合わせて発散する。
  • /triage-requests、/interview、/setup-metrics は、要望・調査・指標設計を個別に扱う。

Examples

  • /discover AI-powered meeting summarizer for remote teams
  • ユーザー活性化を上げる Opportunity Solution Tree を作る。
  • エンタープライズ顧客の機能要望をテーマ別に優先順位付けする。
pm-product-strategy 12 skills 5 cmds

プロダクト戦略

ビジョン、価値提案、事業モデル、価格、競争環境をまとめて扱う戦略レイヤー。個別フレームワークを並べるだけでなく、プロダクトの勝ち筋として接続する。

Skills

  • product-strategy は 9 セクションの Product Strategy Canvas で、ビジョンから防御力まで整理する。
  • lean-canvas、business-model、startup-canvas は、事業モデルを粒度別に描き分ける。
  • pricing-strategy、swot-analysis、pestle-analysis、porters-five-forces で市場と価格の前提を検証する。

Commands

  • /strategy は包括的な Product Strategy Canvas を生成する。
  • /business-model は lean / full / startup / value-prop / all から用途に合うキャンバスを選ぶ。
  • /market-scan と /pricing は外部環境と収益化の妥当性を点検する。

Examples

  • /strategy B2B project management tool for agencies
  • AI ライティングツールの JTBD ベースの価値提案を設計する。
  • プロジェクト管理 SaaS 市場を Porter's Five Forces で分析する。
pm-execution 15 skills 10 cmds

実行管理

戦略をチームの実行単位へ翻訳する領域。PRD、OKR、ロードマップ、スプリント、リリース、ステークホルダー管理まで、日々の PM 作業を支える。

Skills

  • create-prd、brainstorm-okrs、outcome-roadmap で、要求・目標・ロードマップを成果中心にまとめる。
  • sprint-plan、retro、release-notes で、スプリント運営とリリースコミュニケーションを支える。
  • user-stories、job-stories、wwas、test-scenarios で、開発が動ける粒度に分解する。

Commands

  • /write-prd、/plan-okrs、/transform-roadmap が計画系の骨格を作る。
  • /sprint は plan / retro / release のサブモードでスプリントの前後を扱う。
  • /write-stories、/test-scenarios、/generate-data は実装・検証の入力を作る。

Examples

  • /write-prd Smart notification system that reduces alert fatigue
  • /sprint retro にスプリントのメモを渡して振り返りを構造化する。
  • 50 件のバックログに合う優先順位付けフレームワークを選ぶ。
pm-market-research 7 skills 3 cmds

市場調査

ユーザー、市場、競合を調査して、誰のどんな課題を狙うのかを明確にする領域。定性調査と市場規模の両方を PM が扱える形へ変換する。

Skills

  • user-personas、market-segments、user-segmentation で顧客像とセグメントを分ける。
  • customer-journey-map で、接点・感情・痛みを時系列に見える化する。
  • market-sizing、competitor-analysis、sentiment-analysis で市場規模・競争・声の傾向を読む。

Commands

  • /research-users はペルソナ、セグメント、カスタマージャーニーをまとめて作る。
  • /competitive-analysis は競合の強み・弱み・差別化余地を整理する。
  • /analyze-feedback は NPS、レビュー、調査回答を感情とテーマで分解する。

Examples

  • /research-users に 12 人分のインタビューデータを渡す。
  • AI コードレビュー市場の TAM / SAM / SOM を見積もる。
  • Q4 の NPS 回答 200 件をテーマ別に分析する。
pm-data-analytics 3 skills 3 cmds

データアナリティクス

PM が意思決定に必要なデータ分析を扱う小さな分析セット。SQL、コホート、A/B テストを、統計と事業判断の橋渡しとして使う。

Skills

  • sql-queries は BigQuery / PostgreSQL / MySQL 向けに自然言語から SQL を作る。
  • cohort-analysis はリテンション、機能採用、エンゲージメントをコホート単位で読む。
  • ab-test-analysis は有意性、サンプルサイズ、MDE、ship / extend / stop 判断を扱う。

Commands

  • /write-query は分析したい問いを SQL に変換する。
  • /analyze-cohorts は行動データから継続率や採用率の差を読む。
  • /analyze-test は実験結果を統計的に評価する。

Examples

  • /write-query Show me monthly active users by country for Q4 2025
  • 95% 信頼水準、2% MDE に必要なサンプルサイズを見積もる。
  • 1 月登録ユーザーと 2 月登録ユーザーの週次リテンションを比較する。
pm-go-to-market 6 skills 3 cmds

ゴートゥーマーケット

最初に攻める市場、届け方、売り方、競合への勝ち方を組み立てる領域。ローンチ計画だけでなく、ICP とチャネル、営業材料までつなげる。

Skills

  • gtm-strategy はチャネル、メッセージ、成功指標、ローンチ計画をまとめる。
  • beachhead-segment と ideal-customer-profile で、最初に狙う顧客を絞り込む。
  • growth-loops、gtm-motions、competitive-battlecard で成長構造と営業材料を作る。

Commands

  • /plan-launch はビーチヘッドからローンチ計画までを一気に作る。
  • /growth-strategy はグロースループと GTM モーションを検討する。
  • /battlecard は競合比較、反論処理、勝ち筋を営業向けにまとめる。

Examples

  • /plan-launch AI code review tool targeting mid-size engineering teams
  • フリーミアム B2B SaaS のグロースループを設計する。
  • 自社 CRM と Salesforce の SMB 向けバトルカードを作る。
pm-marketing-growth 5 skills 2 cmds

マーケティング & グロース

市場にどう伝わり、どの指標で成長を見続けるかを扱う領域。ポジショニング、メッセージ、名前、North Star Metric をひと続きで設計する。

Skills

  • marketing-ideas はチャネルとメッセージを含む、低コストで試せるマーケティング案を出す。
  • positioning-ideas と value-prop-statements は競合との差分と伝え方を整理する。
  • product-name と north-star-metric は、ブランドと言語化、成長指標の核を決める。

Commands

  • /market-product はマーケティング案、ポジショニング、価値提案、名前をまとめて発散する。
  • /north-star は North Star Metric と入力指標を定義する。

Examples

  • /market-product B2B analytics dashboard for e-commerce managers
  • Notion と差別化するポジショニング角度を 5 つ出す。
  • 二面 marketplace の North Star Metric を決める。
pm-toolkit 4 skills 5 cmds

PM ツールキット

中核 PM ワークから少し外れるが、実務で頻繁に発生する文書・法務・キャリア系の補助領域。レビュー、ドラフト、校正を作業単位で支える。

Skills

  • review-resume は PM レジュメを XYZ+S 公式、キーワード、構成の観点で見る。
  • draft-nda と privacy-policy は NDA とプライバシーポリシーのドラフトを作る。
  • grammar-check は文法だけでなく、論理、流れ、読みやすさも対象にする。

Commands

  • /review-resume と /tailor-resume は、PM レジュメのレビューと求人別調整を扱う。
  • /draft-nda、/privacy-policy、/proofread は、文書作成と校正の短いワークフロー。

Examples

  • /tailor-resume にレジュメと求人票を渡して調整する。
  • プロダクト告知文の文法、論理、流れをチェックする。
  • /proofread で Q1 投資家向けアップデート草案を磨く。
4 — インストール
Claude Cowork（推奨）

非開発者向けの最速経路。Customize → Browse plugins → Personal → + → Add marketplace from GitHub の順に進み、phuryn/pm-skills を入力すると全 8 プラグインが自動インストールされる。

Claude Code（CLI） 
# Step 1: マーケットプレイスを追加
claude plugin marketplace add phuryn/pm-skills

# Step 2: 必要なプラグインをインストール
claude plugin install pm-toolkit@pm-skills
claude plugin install pm-product-strategy@pm-skills
claude plugin install pm-product-discovery@pm-skills
claude plugin install pm-market-research@pm-skills
claude plugin install pm-data-analytics@pm-skills
claude plugin install pm-marketing-growth@pm-skills
claude plugin install pm-go-to-market@pm-skills
claude plugin install pm-execution@pm-skills
その他のエージェント（スキル単体）

skills/*/SKILL.md はユニバーサルフォーマット。スラッシュコマンドは Claude 専用だが、スキル自体はどのツールでも動く。

ツール配置先動作範囲
Gemini CLI~/.gemini/skills/スキルのみ
OpenCode.opencode/skills/スキルのみ
Cursor.cursor/skills/スキルのみ
Codex CLI.codex/skills/スキルのみ
Kiro.kiro/skills/スキルのみ
 
# 例: OpenCode 向けに全スキルをコピー（プロジェクトルートから実行）
for plugin in pm-*/; do
  mkdir -p .opencode/skills/
  cp -r "$plugin/skills/"* .opencode/skills/ 2>/dev/null
done

designlang

npm: designlang / リポジトリ: design-extract — ライブ DOM から Design Language を抜き出す

1 — 概要
位置づけ

designlang（ソースは design-extract リポジトリ）は、Playwright のヘッドレス Chromium で任意の URL を開き、実際に計算された CSS から色・タイポ・間隔・影・ブレークポイント・モーション・アクセシビリティ指標などを集約し、17 種類以上の成果物（AI 向け Markdown、W3C トークン、Tailwind 設定、shadcn テーマ、Figma 変数、designlang mcp 用メタ、など）として書き出す Node.js ツールだ。

Node20+
CLIdesignlang
BrowserPlaywright
17+出力スロット

v10 では ページ意図・セクションの役割・Material / 画像表現・コンポーネントライブラリ推定など、他ツールが拾わない「意味」と「文脈」に踏み込んでいる。オプションで --full マルチページ、--smart 低信頼時の LLM 補正（API キー必須）も選べる。

実行前提: 依存に Playwright があり、postinstall で Chromium の取得を試みる。オフライン環境ではブラウザ取得に手を要することがある。npm パッケージ名は designlang、公開バイナリ名は designlang（bin/design-extract.js）。
2 — パイプライン
処理の流れ（概念）

エントリ extractDesignLanguage は、まず crawlPage で（ライト/インタラクション用データを含め）生の計算スタイルと DOM シグナルを採取し、続く段階で extractors/* が色・タイポ・レイアウト・モーション・page-intent などのサブシステムに落とし込み、最後に formatters/* へ渡して各ファイル形式に整形される。

crawl（Playwright） → aggregate styles → extractors 群 → design オブジェクト → formatters & 書き出し

抽出レイヤの例

  • 基礎: colors / typography / spacing / shadows / borders / variables
  • 構造・挙動: layout / breakpoints / components + component-clusters / interaction-states / motion
  • 意味づけ（v10 付近）: page-intent / section-roles / material-language / imagery-style / component-library / voice
  • 運用系: lint / drift / sync / watch / visual-diff / multibrand — CLI サブコマンドから利用
早い試し方 
npx designlang https://example.com
npx designlang https://example.com --full
3 — 主な出力物
ファイル指針

同一抽出につきプレフィックス（例: ドメイン名由来）付きの複数ファイルが一括で生成される。代表的な役割は次のとおり（README 記載のサマリに準拠）。

成果物用途
*-design-language.mdLLM に食わせるための 19 セクション構成の設計要約
*-preview.htmlスウォッチ・タイプスケール等のビジュアルレポート
*-design-tokens.jsonW3C Community Group 形式の Design Tokens
*-tailwind.config.jsTailwind テーマ
*-variables.css / *-shadcn-theme.cssCSS カスタムプロパティ、shadcn 向け
*-figma-variables.jsonFigma Variables 流し込み
*-motion-tokens.json持続時間・easing・スクロール連携など v9+ のモーション言語
*-anatomy.tsxコンポーネント解剖に基づく型付き React スタブ
*-voice.jsonトーン・CTA 動詞・見出しスタイルなど voice 指紋
*-mcp.json後続 designlang mcp 向けのディスク上メタ
4 — v10（Intent）と v9
v10: 意図と文脈
  • Page Intent — landing / pricing / docs 等に分類。デフォはヒューリスティクス、--smart で低信頼時に LLM 補正可能。
  • Section Roles — セマンティック領域に hero / feature-grid / pricing-table などの役割ラベルとスロット抜粋を付与。
  • Material / Imagery / Library — ガラス/フラット等の Material Language、画像の Imagery Style、UI ライブラリ候補（shadcn / MUI など）を検出。
  • Prompt Pack — v0 / Lovable / Cursor / Claude Artifacts 向けの recipe-* など、貼るだけ用プロンプト一式。
v9: モーションとトークン運用

意味トークン化された Motion Language、Component Anatomy v2（スロット＋ variant × size × state）、Brand Voice 抽出、designlang lint（自前トークンファイルの監査）、designlang drift（本番サイトとローカルトークンの比較）、designlang visual-diff（2 URL の可視比較レポート）が一続きのストーリーとして提供される。

5 — 他ツールとの違い（抜粋）
マーケット上の空き

README では、単純に色とフォントだけ出す層と差をつけるため、次のような 構造寄りの能力が前面に出ている。

  • Layout System — グリッド/フレックス/コンテナ幅/ギャップ/整列のパターン採取。
  • Responsive 4 ビューポイント — ブレーク間で 何が 切り替わるか（表示/非表示、段数、フォントサイズ等）の差分を記述。
  • Interaction States — ホバー/フォーカス等をプログラム的に採取してスタイル遷移を具体値で残す（--interactions 系フロー）。
  • Live → Repo — sync / apply で、デプロイ済みサイトをソースオブトゥルースにしたトークン更新を自動化しうる。
  • Multi-Brand / Score / Watch / Clone — 複数ブランド横断、品質採点、差分ウォッチ、Next.js 雛形への clone など、再現用アプリ層まで踏み込む。

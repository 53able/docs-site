OpenReview

セルフホスト型 AI コードレビューボット（Beta）

GitHub PR で @openreview とメンションするだけで、Claude が PR の差分を読み・Sandbox でコードを実行し・インラインコメントと修正提案を投稿するオープンソースボット。Vercel にワンクリックでデプロイできる。

動作フロー

Webhook → Workflow → Agent → GitHub

PR コメントで起動し、Vercel Workflow が各ステップを durable に実行する。Sandbox にリポジトリをクローンしてエージェントを走らせ、完了後に Sandbox を廃棄する。

sequenceDiagram participant U as Developer participant GH as GitHub participant WH as Webhook Handler participant WF as Vercel Workflow participant SB as Vercel Sandbox participant AI as Claude Agent U->>GH: @openreview in PR comment GH->>WH: Webhook event WH->>WF: Start workflow WF->>GH: Check push access WF->>SB: Create sandbox SB->>SB: Clone repo on PR branch SB->>SB: Install dependencies WF->>AI: Run agent with PR context AI->>SB: Read files, run linters SB-->>AI: Command output AI->>GH: Post inline comments WF->>SB: Check uncommitted changes alt Changes made WF->>SB: Commit and push SB->>GH: Push changes end WF->>SB: Stop sandbox

ステップ詳細

1
Webhook 受信
POST /api/webhooks で GitHub の Issue Comment / PR Review Comment イベントを受信。@openreview メンションを検出したら Vercel Workflow を起動する。
2
Push アクセス確認
GitHub App がブランチへの書き込み権限を持つか検証。権限なしの場合は PR にエラーコメントを投稿して FatalError で終了。
3
Sandbox 作成・準備
Vercel Sandbox を起動し、PR ブランチをクローン・依存インストール・git 設定を行う。エージェントはこの隔離環境内でのみ動作する。
4
Agent 実行
Claude Sonnet 4.6 ベースの DurableAgent が bash・readFile・writeFile・reply・loadSkill ツールを使いながら PR を解析。インラインコメントと提案を GitHub に投稿する。
5
変更のコミット（任意）
Sandbox に未コミット変更がある場合、openreview: apply changes として PR ブランチへコミット・プッシュする。フォーマット修正やリント修正はここで自動適用される。
6
Sandbox 停止
finally ブロックで必ず Sandbox を破棄。エラー発生時も PR にエラーコメントを投稿した上で停止する。

機能

オンデマンドレビュー 任意の PR コメントで @openreview とメンション。特定の指示（「セキュリティ脆弱性を確認して」など）も添えられる。
Sandbox 隔離実行 Vercel Sandbox 上でリポジトリをクローンし、リンター・フォーマッター・テストを安全に実行できる。
インラインサジェスチョン GitHub の suggestion ブロックでワンクリック適用可能な修正コードを行単位でコメント投稿する。
直接コード修正 フォーマット・リント・単純バグを直接 writeFile で修正し、PR ブランチへコミット・プッシュできる。
リアクション操作 👍 / ❤️ で提案を承認して再適用。👎 / 😕 でスキップ。Webhook の Reaction イベントを購読して処理する。
Durable ワークフロー Vercel Workflow による耐障害性のある実行。各ステップが独立して再試行可能で、長時間処理も確実に完了する。
拡張可能なスキル .agents/skills/ に SKILL.md を置くだけでカスタムレビュースキルを追加できる。エージェントは必要なときだけスキルをロードする。
状態の永続化 Redis（Upstash 推奨）または in-memory でスレッド状態を管理。PR 番号・ブランチ・リポジトリ情報をリアクション処理で再利用する。

スキルシステム

エージェントはスキル名と description だけをシステムプロンプトに含める。リクエストに関連するスキルと判断したときだけ loadSkill ツールで全文をロードする。これによりコンテキストウィンドウを最小に保ちつつ、専門的なレビューを実現する。

スキル名 説明
next-best-practices ファイル規約・RSC 境界・データパターン・Async API・メタデータ・エラーハンドリング
next-cache-components PPR・use cache ディレクティブ・cacheLife・cacheTag・updateTag
next-upgrade 公式マイグレーションガイドと codemod に従った Next.js アップグレード
vercel-composition-patterns コンポーネントリファクタリングのための React コンポジションパターン
vercel-react-best-practices React / Next.js パフォーマンス最適化ガイドライン
vercel-react-native-skills React Native / Expo のベストプラクティス
web-design-guidelines Web Interface Guidelines / アクセシビリティ準拠のための UI コードレビュー

カスタムスキルの追加

プロジェクトの .agents/skills/<skill-name>/SKILL.md に YAML フロントマターを持つ Markdown ファイルを置くだけ。name と description フィールドが必須。

---
name: my-custom-skill
description: When to use this skill — the agent reads this to decide whether to load it.
---

# My Custom Skill

Your specialized review instructions here...

セットアップ

4 ステップでデプロイ
1
Vercel にデプロイ
リポジトリを Vercel にデプロイ（「Deploy with Vercel」ボタンまたはリポジトリのクローン＆インポート）。
2
GitHub App を作成

Webhook URL を https://your-deployment.vercel.app/api/webhooks に設定。リポジトリ権限（Contents / Issues / Pull Requests: Read & Write、Metadata: Read-only）と、Issue comment・PR review comment イベントを購読する。

3
環境変数を設定
変数名説明
ANTHROPIC_API_KEYClaude API キー
GITHUB_APP_IDGitHub App の ID
GITHUB_APP_INSTALLATION_IDインストール ID
GITHUB_APP_PRIVATE_KEY秘密鍵（\n で改行エスケープ）
GITHUB_APP_WEBHOOK_SECRETWebhook シークレット
REDIS_URL（任意）Upstash Redis URL
4
GitHub App をインストール
対象リポジトリに GitHub App をインストール。PR で @openreview とコメントすればすぐにレビューが走る。

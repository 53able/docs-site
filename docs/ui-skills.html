UI Skills

Design Engineers 向けのエージェントスキル集 — AI が量産しがちな UI slop を、Tailwind / a11y / metadata / motion のルールで抑える。

1 — 概要
Positioning

UI Skills（ibelick/ui-skills）は、フロントエンド実装を AI エージェントに任せるときの品質ガードレールを SKILL.md として配布するプロジェクト。公式サイトは ui-skills.com。

リポジトリは 2 層になっている。ローカル 4 スキル（リポジトリ同梱の skills/*/SKILL.md）と、外部レジストリ（registry.ts に列挙された他リポジトリの SKILL）を Astro サイトが統合カタログとして公開する。

手動コピーskills/<slug>/SKILL.md をエージェントの skills ディレクトリへオフライン・カスタム配置向け。
 
# デフォルト（baseline-ui のみ）
curl -fsSL https://ui-skills.com/install.sh | sh

# 同梱 4 スキル + frontend-design（レジストリ経由）
curl -fsSL https://ui-skills.com/install.sh | sh -s -- --all

# npm CLI
npx ui-skills add fixing-accessibility
npx ui-skills add "baseline-ui fixing-metadata"

配置先（install.sh が検出する代表例）

プロジェクト直下に .cursor/skills、.claude/skills、.codex/skills などが存在すると、各 <slug>/SKILL.md を自動配置する。Cursor / Claude Code では .cursor/commands/<slug>.md、.claude/commands/<slug>.md にコマンド版もコピーされる。

ローカルに SKILL が無いスキル（例: frontend-design）は、https://ui-skills.com/skills/registry.txt の URL か skills/<slug>/llms.txt からダウンロードする。

4 — baseline-ui の制約カテゴリ
MUST / NEVER の軸

baseline-ui は「明示的に依頼されない限りアニメーションを足さない」「紫グラデ・glow を避ける」など、禁止パターンを先に列挙するスタイル。主要カテゴリは次のとおり。

Stack & Components

  • Tailwind デフォルト値を基本とする
  • JS アニメは motion/react
  • キーボード UI は Base UI / React Aria / Radix
  • 破壊的操作は AlertDialog

Animation & Layout

  • transform / opacity のみ（200ms 以内）
  • h-dvh、safe-area、固定 z-index スケール
  • prefers-reduced-motion を尊重
  • 見出し text-balance、本文 text-pretty
カテゴリ代表 NEVER
Designグラデーション、glow を主 affordance にしない、ビューあたりアクセント 1 色
Animationlayout プロパティのアニメ、ループの off-screen 停止なし、カスタム easing
Performance大きな blur / backdrop-filter アニメ、常時 will-change
Interactioninput の paste ブロック、エラーを離れた場所だけに表示
5 — サイトとレジストリ
Astro カタログ

公開サイト本体も同リポジトリ内の Astro 5 + React 19 + Tailwind 4 アプリ。src/data/skills.ts がローカル SKILL（glob で skills/*/SKILL.md）と registry.ts の外部エントリをマージし、トピックタグ付きカタログを生成する。

skills/*/SKILL.md + registry.ts → skills.ts → /skills/[...slug] → llms.txt / registry.txt

registry.ts には Remotion、Vue/Nuxt、Three.js、Next.js など多数の外部 SKILL が登録されている。サイト上では isRegistry: true としてローカル SKILL と区別される。

Matt Pocock Skills

Skills For Real Engineers — 「プロセスを丸ごと奪う巨大フレーム」ではなく、小さく、組み合わせやすく、実開発の失敗モードを潰すための agent skills。

1 — 概要
What it is

mattpocock/skills は、Claude Code などのコーディングエージェントに読み込ませるスキル集。README は「real engineering, not vibe coding」を掲げ、仕様合わせ、用語合わせ、テスト、診断、トリアージ、設計改善を小さなワークフローとして切り出している。

Core stance

GSD / BMAD / Spec-Kit のように全体プロセスを所有するのではなく、制御をユーザー側に残す。各スキルは独立して差し替えやすく、モデル非依存で使える前提。

「小さく適応しやすいスキル」を積む設計。巨大な自動化よりも、フィードバックループを増やす方向に寄せている。

3 — スキル分類
Engineering

daily code work。仕様化、Issue 化、TDD、診断、トリアージ、設計改善を扱う。

  • grill-with-docs — 既存語彙と ADR に照らして計画を詰める
  • tdd — vertical slice の red-green-refactor
  • diagnose — reproduce → minimise → hypothesise → instrument → fix
  • triage — Issue を state machine で進める
Productivity

コードに限らない作業用。会話密度やスキル作成を支える。

  • caveman — 技術精度を保ったまま出力を圧縮
  • grill-me — 設計ツリーを質問で詰める
  • write-a-skill — progressive disclosure な skill 構造を作る
Misc

頻度は低いが便利な保守ツール群。

  • git-guardrails-claude-code — 危険な Git 操作を hook で止める
  • migrate-to-shoehorn — test の `as` assertion を置き換える
  • scaffold-exercises — exercises / solutions / explainers の雛形生成
  • setup-pre-commit — lint-staged / format / typecheck / tests
4 — 品質ループ
Four failure modes

README は AI エージェント開発の失敗を 4 つに分ける。それぞれに対応スキルを置くことで、会話のズレ、語彙のブレ、壊れたコード、泥団子化に対処する。

失敗モード問題対応
Agent didn't do what I want作る前の認識合わせ不足。grill-me, grill-with-docs
Agent is too verboseドメイン語彙がなく、毎回説明が長くなる。CONTEXT.md と shared language
Code doesn't workエージェントが実行可能な feedback loop を持っていない。tdd, diagnose
Ball of mudコード生成速度が entropy も加速する。to-prd, zoom-out, improve-codebase-architecture
Diagnose loop

/diagnose は「まず高速で決定的な pass/fail signal を作る」ことを中心に置く。コードを眺める前に、再現・最小化・仮説検証・計測を順番に進める。

Feedbacktest / curl / CLI / browser / harness
Reproduceuser symptom を再現
Hypotheses3-5 個を順位付け
Instrument予測に対応する probe
Fixregression test → fix
Postmortempreventive architecture を考える
TDD の特徴: 一括で全テストを書いてから実装する horizontal slice を明確に禁止。1つの振る舞いを RED → GREEN で通し、学びながら次の tracer bullet に進む。
5 — 設計思想

Hard dependency と soft dependency を分ける

to-issues, to-prd, triage は issue tracker と label mapping がないと誤動作するため明示的に setup を要求する。diagnose, tdd, improve-codebase-architecture は domain docs がなくても動くため、柔らかく参照する。

hard soft

Shared language を token 節約に使う

CONTEXT.md は単なる用語集ではなく、エージェントが毎回20語で説明していた概念を1語に圧縮する仕組み。命名、探索、思考トークンの削減に効く。

CONTEXT.md ADR

Issue は state machine で扱う

/triage は needs-triage, needs-info, ready-for-agent, ready-for-human, wontfix の state role を持つ。agent brief はファイルパスや行番号ではなく、振る舞いと interface を契約にする。

Architecture は deep module を探す

/improve-codebase-architecture は module / interface / implementation / depth / seam / adapter / leverage / locality という語彙で、浅い module を深くする候補を探す。

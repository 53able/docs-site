1 — 概要
DESIGN.md とは何か

DESIGN.md は Google Stitch が提唱した新しいコンセプト。AI エージェントが読むためのプレーンテキスト設計書で、カラーパレット・タイポグラフィ・コンポーネントスタイル・レイアウト原則をマークダウンで記述する。特別なツールも JSON スキーマも不要で、プロジェクトルートに置くだけで AI コーディングエージェントが UI の見た目を統一してくれる。

このリポジトリは実在する有名サービスのウェブサイトから抽出した ready-to-use な DESIGN.md ファイルのキュレーションコレクション。Figma エクスポートなし・JSON スキーマなし・特別なツールなし。マークダウンは LLM が最も得意とするフォーマット。

Google Stitch フォーマット準拠 + 拡張

すべての DESIGN.md は Stitch DESIGN.md フォーマットに基づき、9 つのセクションで構成されている。

実際の DESIGN.md 例 — Stripe の Visual Theme より: カスタム sohne-var 可変フォント・weight 300 のヘッドライン・rgba(50,50,93,0.25) を使ったブルーティントのシャドウシステム。「blue-tinted multi-layer shadows」は Stripe らしさを決定づける要素として明記されている。

ブランドごとのファイル

5 — 使い方
2 ステップで導入できる

複雑な設定は一切不要。欲しいブランドの DESIGN.md をコピーしてプロジェクトに置くだけ。

Step 01
DESIGN.md をコピー
欲しいブランドのディレクトリから DESIGN.md をコピーし、プロジェクトのルートに置く。どのブランドのスタイルに近い UI を作りたいかで選ぶ。
Step 02
AI エージェントに指示
AI コーディングエージェントに 「DESIGN.md を参照してこのページを作って」 と伝えるだけ。Google Stitch でもそのまま使える。エージェントが一貫した UI を自動生成する。
Google Stitch との互換性: DESIGN.md フォーマットは Google Stitch が策定した仕様を拡張したもの。Stitch のデザインエージェントはこのファイルをネイティブに解釈できる。

Agent Zero — パーソナル AI エージェント

agent0ai/agent-zero · Python · Docker · SKILL.md compatible · github.com/agent0ai/agent-zero

1 — 概要
何か

Agent Zero は「定義済みのワークフローを持たない」汎用 AI エージェントフレームワーク。 タスクを与えると、必要なツールを自分で作り、サブエージェントを起動し、過去の解決策を記憶しながら成長し続ける。 コードベース全体が読みやすく、プロンプト・ツール・拡張の一切がオープンで変更可能。

動的ハードコードなし
透過的全プロンプト公開
Dockerワンコマンド起動
SKILL.mdクロスプラットフォーム

OS をツールとして使い、Terminal・Python・Node.js を直接実行できる。デフォルトツールは検索・メモリ・通信・コード実行のみで、残りはエージェント自身が必要に応じて生成・習得する。

実際のユースケース
  • 財務分析 & チャート生成 — Bitcoin 価格トレンドと主要ニュースを相関付けてアノテーション付きチャートを出力
  • Excel 自動化パイプライン — 受信ディレクトリのスプレッドシートを検証・クリーニングして経営レポートを生成
  • API 統合の学習と記憶 — Gemini API スニペットを一度与えると、次回以降はメモリから参照して再利用
  • 定期サーバー監視 — 30 分ごとに CPU・ディスク・メモリを確認し、閾値超過をアラート
  • マルチクライアントプロジェクト — クライアントごとに隔離されたメモリ・シークレット・プロジェクト空間

3 — コアコンポーネント
01 — Agents
エージェント

コアアクター。指示を受け、思考し、ツールを使い、サブエージェントを生成する。同一インターフェースで階層を無制限に構築できる。上位エージェントは常に存在し、Agent 0 の場合はユーザーが上位。

02 — Tools
ツール

デフォルト: code_execution_tool / call_subordinate / memory_tool / response_tool / behavior_adjustment。検索は SearXNG 統合（プライバシー重視メタ検索）。カスタムツールはプロンプトで定義可能。

03 — Memory
メモリシステム

VectorDB ベースの永続メモリ。Fragments（会話の断片）・Solutions（成功した解決策）を自動管理。Embedding はローカル小型モデルで生成。コンテキストウィンドウは動的圧縮で最適化。

04 — Prompts
プロンプトシステム

行動全体を prompts/ フォルダ内の Markdown で定義。エージェントプロファイルごとに {{include original}} でデフォルトを継承しつつオーバーライド可能。実行時の振る舞い調整も behaviour.md で永続化。

05 — Skills
スキル（SKILL.md 標準）

コンテキストに応じて動的ロードされる専門知識。システムプロンプトに常駐しないのでトークン効率が高い。Claude Code / Cursor / Goose など他プラットフォームと互換。

06 — Extensions
拡張

python/extensions/ の Python ファイル。アルファベット順に実行されるためファイル名の数値プレフィックスで順序を制御。Message Loop / Memory Management / System Integration の 3 種類。

4 — メモリシステム
ハイブリッドメモリモデル

メモリはフレームワークが自動管理する部分とユーザーが手動で挿入できる部分のハイブリッド。各エントリには ID・タイムスタンプ等のメタデータが付与され、意味的検索でフィルタリングできる。

Fragments
会話の断片

過去の会話から重要な情報を自動抽出・更新。人間が細かいことを忘れるように、Agent Zero もノイズを除いて要点だけを保持する。

Solutions
成功した解決策

過去に成功したコード・手順・統合方法を保存。次回同種タスクに遭遇したとき、学習済みの方法を再利用して時間を短縮する。

User-Provided
ユーザー提供情報

名前・API キー・プロジェクト固有の事実など。「覚えておいて」と伝えるだけで保存され、以降のすべての会話で参照可能。

動的コンテキスト圧縮

長い会話でもコンテキストウィンドウを超えないよう、多段階の圧縮戦略を採用している。

最新メッセージ（原文保持） → 古いメッセージ（要約） → さらに古い（高圧縮要約） → トピックまとめ

要約はユーティリティモデルが担当。埋め込みはローカル小型モデルで生成するためプライバシーと速度を両立。

ベストプラクティス: 重要なセッション後は "memorize learning opportunities from the current session" とエージェントに依頼すること。長期継続のワークフローでは、重要知識をプロンプト化して明示的に埋め込むと信頼性が上がる。
5 — スキル（SKILL.md 標準）
SKILL.md とは

Anthropic が策定したオープンな SKILL.md 標準に準拠した、コンテキスト依存型の専門知識ファイル。 Claude Code Cursor Goose Codex CLI GitHub Copilot と互換性があり、ポータブルに再利用できる。

システムプロンプトに常駐せず、関連するコンテキストが検出されたときだけ動的にロードされる。これによりトークンを節約しながら必要時だけ高度な専門知識を提供できる。

SKILL.md フォーマット 
---
name: "my-skill"
description: "このスキルが何をするか・いつ使うか"
version: "1.0.0"
author: "Your Name"
tags: ["category", "purpose"]
---

# スキルの手順

詳細な指示をここに記述...

## 例
- 使用例 1
- 使用例 2
スキルの追加手順
01
フォルダを作成

usr/skills/my-skill/ ディレクトリを作成する。

02
SKILL.md を配置

YAML frontmatter（name / description / tags）は必須フィールド。

03
スクリプトを追加（任意）

.sh / .py / .js / .ts などを同フォルダに配置してスキルから参照できる。

04
自動検出

エージェントは description / tags に基づいて自動的にスキルを発見・ロードする。

6 — カスタマイズ
プロンプトシステム

prompts/default/ にあるすべての Markdown ファイルがエージェントの行動を定義する。エージェントプロファイル（agents/<profile>/prompts/）に同名ファイルを置くと自動でマージされる。

プロンプトファイル役割
agent.system.main.role.mdエージェントの全体的な役割と能力定義
agent.system.main.communication.md通信スタイルの指定
agent.system.main.solving.mdタスクへのアプローチ方法
agent.system.main.specifics.mdプロファイル固有の追加指示（デフォルト空）
agent.system.tools.mdツールプロンプトのエントリポイント
agent.system.tool.*.md個別ツールの指示
プロンプト継承

{{include original}} を使うとデフォルトプロンプトを継承しつつ差分だけ追記できる。将来のデフォルト更新も自動で取り込める。

{{include original}}

- always explain your reasoning
- include code snippets in responses
動的振る舞い調整

実行中に「UK 英語で返答して」などと伝えると、behaviour_adjustment ツールが呼ばれ、ルールが behaviour.md に永続化されて次回以降の会話にも適用される。

  • 重複・矛盾するルールは自動マージ
  • コアプロンプトを変更せずに挙動を調整できる
  • 拡張側（_20_behaviour_prompt.py）がシステムプロンプトの先頭にルールを注入
7 — セットアップ
クイックスタート

Docker が一番シンプル。ホストマシンへの直接インストールには Python / Conda 環境と RFC 設定が必要。

macOS / Linux

curl -fsSL https://bash.agent-zero.ai | bash

Windows (PowerShell)

irm https://ps.agent-zero.ai | iex

Docker

docker run -p 80:80 agent0ai/agent-zero
ディレクトリ構成（主要）
パス用途
prompts/システム・ツールプロンプトの Markdown テンプレート
agents/<profile>/エージェントプロファイル（プロンプト・ツール・拡張）
python/tools/組み込みツールの実装
python/extensions/モジュール型拡張（Message Loop / Memory / System）
python/helpers/ユーティリティ関数
skills/デフォルトスキル（SKILL.md 形式）
usr/skills/ユーザーカスタムスキル
knowledge/custom/main/カスタム知識ベース（.txt / .pdf / .md 等）
webui/Web インターフェース（CSS / JS / Public assets）
conf/model_providers.yamlモデルプロバイダーのデフォルト設定
セキュリティ注意: Agent Zero はコードを実行し、ファイルを操作し、インターネットにアクセスする。必ず Docker などの隔離環境で動作させること。本番データや重要なシステムに直接アクセスできる環境での実行は推奨しない。

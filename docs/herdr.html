herdr

agent multiplexer that lives in your terminal — ogulcancelik/herdr · v0.7.x · AGPL-3.0-or-later

1 — 概要
何か

herdr は、コーディングエージェント向けのターミナルマルチプレクサ兼ランタイム。ワークスペース・タブ・ペインに端末を整理し、エージェントの識別と状態（blocked / working / done など）を画面検出で把握する。detach してもプロセスは動き続け、別端末や SSH から reattach できる。

Rust単一バイナリ
PTY実端末ビュー
Socketエージェント制御
Plugins拡張マーケット
  • キーバインド（tmux 風 prefix）とマウス操作の両方が第一級。
  • Electron ではなく、既存ターミナル上で動く。
  • エージェント自身が herdr CLI 経由でペインを切ったり出力を読んだりできる（要 HERDR_ENV=1）。
2 — セッション階層
ID と入れ子

公開 ID は短い安定ハンドル。サフィックスは英字を含み、桁が増えることもある。ID は不透明文字列として扱い、サイドバー順から組み立てない。

単位例役割
workspacew1作業の最上位コンテナ
tabw1:t1ワークスペース内の画面単位
panew1:p1エージェント／シェル／ログの制御面
terminalterm_...PTY 側の端末ハンドル
再利用禁止: 閉じた tab / pane ID は再利用されない。別ワークスペースへ移動した pane は新しい公開 pane ID を受け取る。変異のあとは create / split / list / get の JSON を読み直す。 
# マネージド pane に注入されるコンテキスト
printf '%s\n' "$HERDR_WORKSPACE_ID" "$HERDR_TAB_ID" "$HERDR_PANE_ID"
3 — ランタイム境界
サーバ所有へ移行中

AGENTS.md の設計方針: 状態とランタイムを分離し、描画は純関数。現在はサーバ所有ランタイムに寄せており、TUI はクライアントの一つという位置づけ。

flowchart TD
  Client["TUI / CLI client"] --> Socket["Unix socket / IPC"]
  Socket --> Server["server runtime"]
  Server --> State["AppState / Workspace"]
  Server --> Pty["PTY panes"]
  Server --> Detect["detect manifests"]
  Pty --> Agents["coding agents / shells"]
  Detect --> Agents
  Server --> Persist["session persist / handoff"]
層置くもの置かないもの
server / runtimepane・エージェントメタ、プロセス／端末状態、イベント、JSON APIsidebar レイアウト、色、モーダル、マウス選択
TUI / client表示状態、キー／マウス、トークン配置共有ランタイムの真実を TUI 専用ソケットだけで持つこと
検出 (src/detect/)bottom-buffer スナップショットからの証拠ベース判定パーサや viewport 状態への直接タッチ
 
# リポジトリ主要モジュール（抜粋）
src/
  main.rs     # エントリ / HERDR_ENV
  server/     # ランタイム・クライアント受付
  api/        # JSON schema / wait / events
  cli/        # pane workspace tab wait …
  detect/     # エージェント別 manifest
  app/        # AppState / actions（純データ寄り）
  pty/ pane/ workspace/
4 — エージェント状態
status の意味

pane レコードは agent / agent_status とネイティブセッションメタを公開する。検出は src/detect/manifests/（Claude、Codex、Cursor、OpenCode、pi など）に TOML で定義される。

idle
待ち受け。結果は見たとみなす
working
実行中
blocked
入力待ちなど停止
done
完了したが未確認（背景タブなど）
unknown
未検出または判定不能
idle と done: 同じ「完了／待ち」系の意味差は「結果を見たか」。フォーカスで done → idle になる。逆に、既に idle のものをタブ切替だけで done にはしない。

読み取りソースの使い分け:

  • visible — 現在の描画ビューポート
  • recent — ソフトラップ込みの最近の scrollback
  • recent-unwrapped — ログ／トランスクリプト向け（ソフトラップ結合）
  • detection — エージェント検出が使う bottom-buffer
5 — CLI と Socket API
エージェントからの操作

制御前にマネージド pane 内かを確認する。外からフォーカス中セッションをいじらない。

test "${HERDR_ENV:-}" = 1
herdr --help
herdr pane
herdr workspace
herdr tab
herdr wait
herdr terminal

素の herdr は TUI 起動／アタッチなので探索には使わない。多くの制御コマンドは JSON を返すので、ID はレスポンスから取る。

兄弟ペインでエージェントを起動する流れ

herdr pane layout --pane "$HERDR_PANE_ID"
herdr pane split --current --direction right --no-focus
herdr pane rename <pane_id> "reviewer"
herdr pane run <pane_id> "codex"
herdr wait agent-status <pane_id> --status idle --timeout 30000
herdr pane run <pane_id> "Review the current diff and report only actionable findings."
herdr wait agent-status <pane_id> --status done --timeout 120000
herdr pane read <pane_id> --source recent-unwrapped --lines 120
安全ルール: 背景作業は --no-focus。ターゲットは --current か明示 ID。自分が作っていない workspace / tab / pane を閉じない。herdr server stop とメインプロセスの kill は明示依頼があるときだけ。
6 — インストールと開発
入手とビルド 
curl -fsSL https://herdr.dev/install.sh | sh
# または: brew install herdr · mise use -g herdr
herdr
# detach: ctrl+b q → 再アタッチ: herdr

Windows beta は install.ps1、バイナリは GitHub Releases。開発は Rust / just:

git clone https://github.com/ogulcancelik/herdr
cd herdr
cargo build --release
just test        # unit tests
just check       # format + tests + maintenance

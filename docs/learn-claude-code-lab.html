Learn Claude Code Lab

shareAI-lab の learn-claude-code。nano Claude Code 風エージェントを 0 から 1 まで構築する 12 セッション。

1 — 概要
何か

Learn Claude Code（本リポジトリ）は、shareAI-lab が公開する学習用プロジェクト。Claude Code のような AI コーディングエージェントを「1 本のループ + ツール実行」から始め、計画・サブエージェント・コンテキスト圧縮・タスクグラフ・チーム・worktree 分離まで、12 の段階的セッションで 0→1 に組み上げる。各セッションは 1 つのメカニズムを追加し、各メカニズムに 1 つのモットーがある。本番エージェントではポリシー・権限・ライフサイクル層が上乗せされるが、ここではループ自体は変えずに上に重ねる設計。

2 — コアパターン
The Agent Pattern

最小ループ: User → messages[] → LLM → response。stop_reason == "tool_use" ならツールを実行し結果を messages に追加してループ継続、そうでなければテキストを返して終了。本番ではこの上にポリシー・ライフサイクルが載る。

def agent_loop(messages):
    while True:
        response = client.messages.create(
            model=MODEL, system=SYSTEM,
            messages=messages, tools=TOOLS,
        )
        messages.append({"role": "assistant", "content": response.content})
        if response.stop_reason != "tool_use":
            return
        results = [execute_tool(block) for block in response.content if block.type == "tool_use"]
        messages.append({"role": "user", "content": results})

s01 では bash ツール 1 つだけ。「One loop & Bash is all you need」。

セッション一覧（モットー）
  • s01: One loop & Bash is all you need
  • s02: Adding a tool means adding one handler
  • s03: An agent without a plan drifts
  • s04: Break big tasks down; each subtask gets a clean context
  • s05: Load knowledge when you need it, not upfront
  • s06: Context will fill up; you need a way to make room
  • s07: Break big goals into small tasks, order them, persist to disk
  • s08: Run slow operations in the background; the agent keeps thinking
  • s09: When the task is too big for one, delegate to teammates
  • s10: Teammates need shared communication rules
  • s11: Teammates scan the board and claim tasks themselves
  • s12: Each works in its own directory, no interference

6 — スコープ（意図的な省略）
教材であるため

本番で必要な次のような仕組みは、学習を優先するため簡略化または省略している。

  • 完全なイベント / Hook バス（PreToolUse, SessionStart/End 等）。s12 では最小の追記型ライフサイクルイベントのみ。
  • ルールベースの権限ガバナンスと信頼フロー
  • セッション resume/fork や高度な worktree ライフサイクル
  • MCP ランタイムの詳細（transport / OAuth / リソース購読など）

リポジトリ内の JSONL メールボックスプロトコルは教材用実装であり、特定の本番内部実装を主張するものではない。

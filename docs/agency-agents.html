The Agency: 手のひらに収まる AI エキスパート集 — GitHub

1 — 概要
このリポジトリとは

agency-agents は、Reddit の議論から生まれ、数ヶ月の改善を重ねた「AI エージェント人格」のコレクションです。フロントエンドの達人から Reddit コミュニティの達人、ウィットを加える役から現実チェック役まで、各エージェントは専門性・人格・プロセス・成果物を持ったスペシャリストとして設計されています。

一言で言うと：夢のチームを組み立てるが、メンバーは AI スペシャリストで、眠らず、文句を言わず、成果を出し続けるイメージです。

2 — 特徴
各エージェントの共通点
  • 専門特化 — 汎用プロンプトではなく、ドメインに深い知見
  • 人格駆動 — 独自の声・コミュニケーションスタイル・アプローチ
  • 成果物重視 — 実際のコード、プロセス、測定可能なアウトカム
  • 本番向け — 実戦で使われたワークフローと成功指標
一般的な AI プロンプトとの違い：汎用的な「開発者として振る舞え」ではなく、人格とプロセスを持つ専門家。プロンプトライブラリのような単発の羅列ではなく、ワークフローと成果物を持つエージェントシステム。ブラックボックスな AI ツールではなく、フォーク・カスタマイズ可能な透明な人格定義。
3 — クイックスタート
Claude Code で使う（推奨）
# エージェントを Claude Code 用ディレクトリにコピー
cp -r agency-agents/* ~/.claude/agents/

# セッションで任意のエージェントを有効化
# 「Frontend Developer モードで React コンポーネントを作って」のように依頼

各エージェントファイルには、アイデンティティ・人格、ミッション・ワークフロー、技術成果物（コード例）、成功指標・コミュニケーションスタイルが含まれています。必要なものを選んでコピー・改変して利用できます。

4 — リポジトリのディレクトリ構成

クローンした agency-agents のルートは、部門ごとのディレクトリとドキュメントで構成されています。各部門フォルダ内にエージェント定義ファイルが格納されています。

agency-agents/（ルート）
agency-agents/
├── .github/              # GitHub 用設定
├── design/               # Design 部門エージェント
├── engineering/          # Engineering 部門エージェント
├── examples/             # マルチエージェントワークフロー例
├── marketing/            # Marketing 部門エージェント
├── product/              # Product 部門エージェント
├── project-management/   # Project Management 部門エージェント
├── spatial-computing/    # Spatial Computing 部門エージェント
├── specialized/         # Specialized 部門エージェント
├── strategy/             # Strategy 関連エージェント
├── support/              # Support 部門エージェント
├── testing/              # Testing 部門エージェント
├── CONTRIBUTING.md
├── LICENSE
└── README.md

クイックスタートの cp -r agency-agents/* ~/.claude/agents/ では、上記の部門ディレクトリとドキュメントがそのまま ~/.claude/agents/ にコピーされ、Claude Code から部門名でエージェントを指定して利用できます。

6 — ユースケース例
スタートアップ MVP

Frontend Developer（React）、Backend Architect（API/DB）、Growth Hacker（獲得）、Rapid Prototyper（短期イテレーション）、Reality Checker（リリース前品質）の組み合わせで、各段階に専門家を配置して高速にリリース。

マーケティング施策

Content Creator（コンテンツ）、Twitter Engager・Instagram Curator・Reddit Community Builder（プラットフォーム別）、Analytics Reporter（効果測定）で、チャネル横断かつプラットフォーム特化のキャンペーンを構成。

エンタープライズ機能開発

Senior PM（スコープ・タスク）、Senior Developer（実装）、UI Designer（デザインシステム）、Experiment Tracker（A/B）、Evidence Collector（証跡）、Reality Checker（本番可否）で、品質ゲートとドキュメントを伴う企業向け納品を想定。

プロダクトディスカバリ（8 部門並列）

Product / Backend / Brand / Growth / Support / UX Research / Project Shepherd / XR Interface など複数エージェントを同時投入し、市場検証・技術設計・ブランド・Go-to-Market・サポート・UX・実行・XR UI をカバーした統合プロダクト案を 1 セッションで出力する例（Nexus Spatial Discovery Exercise）がリポジトリ内にあります。

7 — 設計思想
  • 強い人格 — 汎用テンプレートではなく、一貫した声とキャラクター
  • 明確な成果物 — 抽象的な助言ではなく、コード・プロセス・成果物
  • 成功指標 — 測定可能なアウトカムと品質基準
  • 実証済みワークフロー — ステップごとの再現可能なプロセス
  • 学習メモリ — パターン認識と継続的改善の考え方

agency-agents リポジトリ

The Agency: 手のひらに収まる AI エキスパート集 — GitHub

1 — 概要
このリポジトリとは

agency-agents は、Reddit の議論から生まれ、数ヶ月の改善を重ねた「AI エージェント人格」のコレクションです。フロントエンドの達人から Reddit コミュニティの達人、ウィットを加える役から現実チェック役まで、各エージェントは専門性・人格・プロセス・成果物を持ったスペシャリストとして設計されています。

一言で言うと：夢のチームを組み立てるが、メンバーは AI スペシャリストで、眠らず、文句を言わず、成果を出し続けるイメージです。

2 — 特徴
各エージェントの共通点
  • 専門特化 — 汎用プロンプトではなく、ドメインに深い知見
  • 人格駆動 — 独自の声・コミュニケーションスタイル・アプローチ
  • 成果物重視 — 実際のコード、プロセス、測定可能なアウトカム
  • 本番向け — 実戦で使われたワークフローと成功指標
一般的な AI プロンプトとの違い：汎用的な「開発者として振る舞え」ではなく、人格とプロセスを持つ専門家。プロンプトライブラリのような単発の羅列ではなく、ワークフローと成果物を持つエージェントシステム。ブラックボックスな AI ツールではなく、フォーク・カスタマイズ可能な透明な人格定義。
3 — クイックスタート
Claude Code（ネイティブ形式）
# 全エージェントを Claude Code 用にインストール
./scripts/install.sh --tool claude-code

# 特定部門だけ入れる例
./scripts/install.sh --tool claude-code --division engineering,security

# セッション例: 「Frontend Developer モードで React コンポーネントを作って」
他ツール（変換 + インストール）
# Step 1: 各ツール向けファイルを生成
./scripts/convert.sh

# Step 2: 対話式インストール（検出済みツールを表示）
./scripts/install.sh

# ツール指定例
./scripts/install.sh --tool cursor
./scripts/install.sh --tool copilot
./scripts/install.sh --tool opencode --division engineering --dry-run

# 部門・エージェント一覧
./scripts/install.sh --list teams

対応ツール: Claude Code、GitHub Copilot、Cursor、Antigravity、Gemini CLI、OpenCode、OpenClaw、Aider、Windsurf、Qwen Code、Kimi Code、Codex など。各エージェントファイルにはアイデンティティ、ワークフロー、コード例、成功指標が含まれる。

参照として使う：インストールせず、README の部門表から必要な *.md を選び、プロンプトやルールにコピーして改変してもよい。
4 — マルチツール連携

scripts/convert.sh がエージェント定義を各ホスト形式へ変換し、scripts/install.sh が配置先へコピーする。詳細は integrations/ 配下の README を参照。

7 — ユースケース例
スタートアップ MVP

Frontend Developer、Backend Architect、Growth Hacker、Rapid Prototyper、Reality Checker を組み合わせ、設計から品質ゲートまで段階ごとに専門家を配置する。

有料広告アカウント引き継ぎ

Paid Media Auditor → Tracking Specialist → PPC Strategist → Search Query Analyst → Ad Creative Strategist → Analytics Reporter の順で、計測確認・無駄削減・構成刷新を 30 日以内に進める。

スマートキャンパス・デジタルツイン

GIS Technical Consultant、BIM/GIS Specialist、Drone Mapping、Web GIS Developer、3D Scene Developer、GeoAI Engineer、GIS QA Engineer で、BIM・ドローン・3D 可視化を一パイプラインにまとめる。

Nexus 並列ディスカバリ（8 エージェント）

Product / Backend / Brand / Growth / Support / UX Research / Project Shepherd / XR Interface を同時投入し、市場・技術・GTM・UX を 1 セッションで統合する。Nexus Spatial Discovery Exercise（examples/）を参照。

8 — 設計思想
  • 強い人格 — 汎用テンプレートではなく、一貫した声とキャラクター
  • 明確な成果物 — 抽象的な助言ではなく、コード・プロセス・成果物
  • 成功指標 — 測定可能なアウトカムと品質基準
  • 実証済みワークフロー — ステップごとの再現可能なプロセス
  • 学習メモリ — パターン認識と継続的改善の考え方

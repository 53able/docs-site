プロダクトの一文

omp（@oh-my-pi/pi-coding-agent）は、対話型 TUI を既定とするコーディングエージェント。編集・検索・LSP・DAP・ブラウザ・サブエージェント・長期記憶まで、ハーネス側に寄せた実装が同梱されている。

公開サイト omp.sh、ランタイムは Bun（推奨 bun ≥ 1.3.14）、ネイティブ処理は Rust の N-API アドオン（pi-natives）経由でプロセス内実行する。

TUI — pi-tui · 差分描画・Markdown
coding-agent — セッション・ツール・MCP・拡張
agent — ループ・コンパクション
ai — 40+ プロバイダ・ストリーム
pi-natives — grep · shell · ast · PTY …
2 — Pi フォークとしての位置づけ
README が強調する差分

ベースは pi-mono の思想（ターミナルファースト・TypeScript・MIT）を継承しつつ、実運用で足りない層を同梱する方針。代表例は次のとおり。

  • 編集 — Hashline（コンテンツハッシュ錨）、ast_edit のプレビュー＋resolve 確定
  • IDE 連携 — LSP（13 操作相当）、DAP（27 操作相当）、ACP で Zed 等から同一エージェントを駆動
  • 並列 — task による worktree 分離サブエージェント、スキーマ検証済み yield
  • 記憶 — Hindsight（retain / recall / reflect）
  • ルール — TTSR（ストリーム途中のルール注入・コンパクション後も維持）
  • 設定の継承 — Cursor / Claude / Codex / Copilot 等の既存ルール・スキル・MCP を初回起動時に読み込み
ベンチ（README 掲載）

Grok Code Fast 1 で編集成功率 6.7%→68.3%、Grok 4 Fast で出力トークン −61% など。編集フォーマットとリトライ設計の影響が大きい、と説明されている。

内部 URL スキーム

pr://、issue://、skill://、agent:// など。FS 系ツールから PR 差分・サブエージェント JSON をパス指定で読める。

Python / JS eval

永続 Python と Bun ワーカー。どちらもループバック経由で read や search を再入可能。

omp commit

作業ツリーを hunk 単位で分析し、依存順にアトミック分割。循環依存はコミット前に拒否。

Rust モジュール（README 抜粋）

他ハーネスが rg や bash を子プロセス起動するのに対し、ホットパスは libuv スレッドプール上の in-process 実装。プラットフォーム別バイナリ（linux/darwin/win × x64/arm64）。

モジュール内容概算 LoC
shell組み込み bash（brush ベンダ）、永続セッション3,700
grep並列 regex 検索・glob1,900
summarize / asttree-sitter 要約・ast-grep~2,000
ptyネイティブ PTY（sudo 等）455
isoworktree 分離（APFS / btrfs / overlay 等）245
補足: python/robomp は GitHub イベント連携など別系統の Python ホスト。python/omp-rpc は RPC クライアント用。エージェント本体の TypeScript ツリーとは責務が分かれている。
4 — 組み込みツール（32）

README のカテゴリ分けに沿った一覧。--tools で表示セットを絞り、search_tool_bm25 で隠れたツールをセッション中に復帰できる。

ファイル・検索

  • write / edit / ast_edit / ast_grep
  • search / find
ランタイム
  • bash · eval · recipe · ssh
コードインテリジェンス
  • lsp — 診断・定義・リネーム等
  • debug — DAP（lldb / dlv / debugpy 等）
協調・外部
  • task · irc · todo_write · ask

設定でデフォルト OFF の例: github, calc, inspect_image, render_mermaid, checkpoint, rewind, Hindsight 系など（README 記載）。

5 — モデルとルーティング

ロール default / smol / slow / plan / commit で用途別モデルを切替。/model スラッシュコマンドと Ctrl+P でセッション中に変更できる。

区分例（README）
Frontier APIAnthropic, OpenAI, Gemini, xAI, Groq, OpenRouter, …
Coding plansCursor, Copilot, Kimi, MiniMax plan, Qwen Portal, …
Self-hostedOllama, LM Studio, vLLM, LiteLLM（OpenAI 互換）

web_search は最大 14 バックエンドのチェーン（auto）と、GitHub / arXiv / 各種レジストリ向けの構造化抽出ハンドラを持つ（README の表参照）。

カスタム定義は ~/.omp/agent/models.yml。フォールバックチェーンは retry.fallbackChains、パス別ロールは modelRoles.paths。

SDK 最小例（README より）
import { ModelRegistry, SessionManager, createAgentSession, discoverAuthStorage } from "@oh-my-pi/pi-coding-agent";

const auth = await discoverAuthStorage();
const models = new ModelRegistry(auth);
await models.refresh();

const { session } = await createAgentSession({
  sessionManager: SessionManager.inMemory(),
  authStorage: auth,
  modelRegistry: models,
});
await session.prompt("list .ts files");
ACP マッピング（README）: bash → ターミナル API、read/write → ファイル API、破壊的操作は session/request_permission。
7 — インストール

bun install -g @oh-my-pi/pi-coding-agent

Windows は irm https://omp.sh/install.ps1 | iex。バージョン固定は mise use -g github:can1357/oh-my-pi（README）。

AiToEarn

OPC・クリエイター向けの AI コンテンツ制作・マルチチャネル配信・収益化 — MIT ライセンスのオープンソース（GitHub: yikart/AiToEarn）。

1 — 概要
位置づけ

AiToEarn は、AI を用いてコンテンツを生成し、複数の SNS・動画プラットフォームへ一括配信し、プロモーション案件やマーケットプレイス経由で収益化するまでを支援するスタックです。公式 README では Monetize · Publish · Engage · Create をスローガンとしており、一人会社（OPC）やブランドのグローバル展開を想定した機能群が揃っています。

10+対応チャネル（例）
Node 20.18推奨ランタイム
MCPClaude / Cursor 等
Docker自己ホスト可

ソースは GitHub の yikart/AiToEarn で公開されています。利用ドメインは地域により aitoearn.cn（中国向け）と aitoearn.ai（国際向け）が案内されています。

2 — 4 本柱（Create → Publish → Engage → Monetize）
ワークフロー上の流れ

README の構成に沿うと、制作から収益までの典型的な流れは次のとおりです。

Create → Publish → Engage → Monetize

Create（内容制作）

エージェントに要件を渡すと、動画・画像・テキストの生成や編集、翻訳、スタイル転送などをパイプライン化して実行します。バックエンド側に aitoearn-ai アプリと Volcengine 等の AI クライアントが配置されています。

Publish（マルチチャネル配信）

単一の下書きから、抖音・小红书・B 站・快手・TikTok・YouTube・Meta 系・X・Pinterest・LinkedIn などへの投稿を自動化します。サーバー側の channel / publishing モジュールと、各プラットフォーム用サービスが担当します。

Engage（インタラクション）

ブラウザ拡張（Electron ベースのデスクトップクライアント）から、いいね・フォロー・コメント返信やブランド監視を自動化します。大規模アカウント運用向けにバッチ操作と LLM 返信が組み合わされています。

Monetize（収益化）

クリエイターがプロモーション案件に参加し、成果に応じて報酬を得るモデルを提供します。README では主に次の 3 区分が説明されています。

区分名称意味
CPSCost Per Sale売上に連動した成果報酬
CPECost Per Engagementエンゲージメント量に応じた報酬
CPMCost Per Milleインプレッション単位の報酬
3 — 利用形態
README に列挙される 5 パターン
方式向き先サーバー要否
Web を開いて利用一般ユーザー利用者側のデプロイ不要
OpenClaw（龙虾）内既存の OpenClaw ユーザー同上
Claude / Cursor 等の MCPエージェント IDE 利用者同上
Docker 一括自社ホストしたいチーム要（コンテナ実行環境）
ソースから開発アプリ拡張・カスタムローカル開発環境
API Key: OpenClaw・MCP・Docker 系の手順では、まず Web の設定画面で API Key を発行する必要があります（README の「如何获取 API Key」節を参照）。
4 — リポジトリ構成（モノレポ）
主要ディレクトリ

ルート直下の project/ にフロント・バックエンド・デスクトップがまとまっています。ルートには docker-compose.yml と nginx/ があり、オンプレ／ VPS 向けの参照構成が置かれています。

project/
├── aitoearn-backend/   # NestJS モノレポ（aitoearn-server, aitoearn-ai, 共有ライブラリ）
├── aitoearn-web/       # Next.js 系の Web クライアント
└── aitoearn-electron/  # デスクトップ＋ブラウザ自動化・各プラットフォーム連携

docker-compose.yml      # 複数サービスの起動定義
nginx/                  # リバースプロキシ設定

バックエンドは MongoDB リポジトリ、MCP ストリーミング、ドラフト生成、素材変換などのドメインサービスに分割されています。Electron 側はプラットフォーム別の electron/plat/ と、補助的な Node サーバー（electron/server）が認証・署名付き API 呼び出しを担います。

デプロイ文書

自己ホスト手順は DOCKER_DEPLOYMENT_CN.md と DOCKER_DEPLOYMENT_EN.md にまとまっています（いずれもリポジトリ直下）。

5 — 連携と拡張ポイント
エージェント・外部ツール
  • MCP: Model Context Protocol 経由で AiToEarn の操作をエージェントから呼び出せます（README の更新履歴 2026-03-26 に記載）。
  • OpenClaw: 同リリースノートで、タスク受領と実行を OpenClaw 内から行える旨が案内されています。
  • スキル定義: バックエンドの apps/aitoearn-ai/src/core/agent/skills/ 以下に、動画編集・生成などの Agent Skill が Markdown で同梱されています。

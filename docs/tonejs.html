Tone.js

A Web Audio framework for creating interactive music in the browser.

1 — 概要
何か

Tone.js は、ブラウザでインタラクティブな音楽を作るための Web Audio フレームワーク。ミュージシャンとオーディオプログラマの両方に馴染みやすい設計で、高レベルでは DAW 的なグローバル Transport（同期・スケジューリング）やプリビルトのシンセ・エフェクトを提供し、低レベルでは自前のシンセ・エフェクト・制御信号を組み立てるための高性能なブロックも用意している。

Transport時間軸・BPM
Synthシンセ・楽器
Effectsエフェクト
Samplesサンプル再生

シグナル処理は Web Audio の GainNode や WaveShaperNode などを多用しており、デスクトップ・モバイル両方のブラウザで動作する。

2 — インストール・Hello Tone
導入

npm でインストールし、ES モジュールで読み込む:

npm install tone
import * as Tone from "tone";

HTML から使う場合は unpkg のスクリプトを読み込む方法もある（simpleHtml 例 参照）。

Hello Tone

シンセを作り、出力（スピーカー）に接続して 1 音鳴らす最小例:

const synth = new Tone.Synth().toDestination();
synth.triggerAttackRelease("C4", "8n");

Tone.Synth は単一オシレータと ADSR エンベロープを持つ基本シンセ。triggerAttackRelease(note, duration) で「ノートオン＋指定時間後にノートオフ」を一括指定できる。音高は "C4" のようなピッチ・オクターブ表記や、Hz 値（例: 440）で指定可能。

3 — Time と再生開始
Time

Web Audio のスケジューリングはサンプル精度。AudioContext の時間はページロード時 0 から秒単位で増加する。Tone.now() で現在時刻を取得できる。Tone.js では「秒」の代わりに "4n"（四分音符）、"8t"（八分三連）、"1m"（1 小節）などのテンポ相対表記が使える（Time encodings 参照）。

再生開始
重要: ブラウザはユーザーが何か（ボタンクリックなど）を操作するまで音を出さない。Tone.js の再生コードは、click や keydown などのユーザー操作で Tone.start() を呼んだ後に実行すること。Tone.start() は Promise を返すので、resolve 後にスケジューリングや再生を行う。 
document.querySelector("button")?.addEventListener("click", async () => {
  await Tone.start();
  console.log("audio is ready");
});
4 — Transport とスケジューリング
Transport

Tone.getTransport() はメインのタイムキーパー。AudioContext の時計と違い、開始・停止・ループ・BPM 変更を動的に行える。DAW のアレンジビューのようなイメージ。複数のイベントやパートを Transport に沿って同期できる。

Tone.Loop は一定間隔でコールバックを実行するループ。コールバックにはサンプル精度の再生時刻が渡されるので、必ずその time を使って発音すること（JavaScript のコールバック自体は時間精度が低い）。

const loopA = new Tone.Loop((time) => {
  synthA.triggerAttackRelease("C2", "8n", time);
}, "4n").start(0);
Tone.getTransport().start();
Tone.getTransport().bpm.rampTo(800, 10);
5 — 楽器
シンセの種類

Tone.Synth、Tone.FMSynth、Tone.AMSynth、Tone.NoiseSynth などはすべてモノフォニック（1 音のみ）。複数音を同時に鳴らすには Tone.PolySynth にモノシンセを渡す。PolySynth は音の割り当てを自動で行い、triggerRelease にはノートまたはノート配列を指定する。

const synth = new Tone.PolySynth(Tone.Synth).toDestination();
synth.triggerAttack("D4", now);
synth.triggerAttack("F4", now + 0.5);
synth.triggerRelease(["D4", "F4", "A4", "C5", "E5"], now + 4);
6 — サンプル
Player と Sampler

音源はシンセに限らない。Tone.Player でオーディオファイルを読み込み再生できる。Tone.loaded() は「すべてのオーディオが読み込み完了」になる Promise を返す。

const player = new Tone.Player(
  "https://tonejs.github.io/audio/berklee/gong_1.mp3"
).toDestination();
Tone.loaded().then(() => { player.start(); });

Tone.Sampler はノートごとのサンプルをまとめた楽器。ピッチシフトで隙間を補間するので、数音分のサンプルから鍵盤全体をカバーできる。Sampler はもともとポリフォニックなので PolySynth は不要。

7 — エフェクトとルーティング
接続の流れ

シンセや Player をそのまま toDestination() せず、エフェクトを挟んでから出力に繋げられる。

Source → Effect → Destination

直列・並列どちらも可能。例: Player を Distortion に connect、あるいは同じ Player を Filter と FeedbackDelay の両方に connect して並列に処理。複数ノードを同一入力に繋いでエフェクトを共有できる。Tone.Gain はルーティングやミキシングのユーティリティとして便利。

const distortion = new Tone.Distortion(0.4).toDestination();
player.connect(distortion);

const filter = new Tone.Filter(400, "lowpass").toDestination();
const feedbackDelay = new Tone.FeedbackDelay(0.125, 0.5).toDestination();
player.connect(filter);
player.connect(feedbackDelay);
8 — Signals・AudioContext
Signal

Web Audio と同様、Tone.js では多くのパラメータがオーディオレートの Signal。サンプル精度で同期・スケジューリングできる。例えば Oscillator の frequency は Signal なので、rampTo でスムーズに周波数を変化させられる。

const osc = new Tone.Oscillator().toDestination();
osc.frequency.value = "C4";
osc.frequency.rampTo("C2", 2);
osc.start().stop("+3");

AudioContext: Tone.js は読み込み時に AudioContext を生成し、standardized-audio-context でブラウザ差を吸収する。Tone.getContext() で取得、Tone.setContext(audioContext) で自前の Context を渡せる。

解決する摩擦

Claude Code の通常応答は、前置き・脱線・締めの定型句が混ざりやすい。ADHD の診断の有無に関わらず、「答えの前に 3 段落」は作業開始の摩擦になる。

症状スキル側の対策
前置きで本題が遅れる1 行目を実行可能なアクションに固定（ルール 1・10）
手順が 1 段落に潰れる複数ステップは番号リスト、1 ステップ 1 アクション（ルール 2）
ターン間で進捗を忘れる「Step 3 of 5 done」のように状態を毎回再提示（ルール 5）
脱線で最初の課題が消える副題は別質問として保留（ルール 4）
完了が埋もれる何が動くようになったかを明示（ルール 7）
「短くする」だけではない: ルールは The Adult ADHD Tool Kit（Ramsay & Rostain）の戦略を、LLM の応答形状向けに写像したもの。診断は不要で、「 preamble を探すスクロール」が苦痛なら対象ユーザー。

flowchart TD
  U["ユーザー: /i-have-adhd"] --> C["commands/i-have-adhd.md"]
  C --> S["skills/i-have-adhd/SKILL.md を読み込み"]
  S --> F["frontmatter description で適用タイミングを指定"]
  F --> R["以降の応答を 10 ルールで整形"]
  R --> X["stop adhd mode / normal mode で解除"]
  1. スラッシュコマンドが SKILL.md の有効化を指示する。
  2. frontmatter の description が「いつルールを適用するか」をモデルに伝える（カジュアルな会話も含む）。
  3. 本文が「どう書くか」を 10 条で定義する。
  4. セッション中は有効。 stop adhd mode または normal mode で解除。
10 のルール

skills/i-have-adhd/SKILL.md が正。以下は README の要約。

#ルール意図
1次のアクションを先頭に「分かった」と「やった」の間の摩擦を減らす
2多段タスクは番号付き1 ステップ 1 境界。1 行に「それから」が 2 回入らない
3末尾は 1 つの具体アクション2 分以内にできる次の一手
4脱線を抑制副題は別質問として提示
5毎ターン状態を再提示作業記憶外の進捗を画面に戻す
6時間見積もりは具体単位「少し」より「15 分」
7完了を可視化何が動くようになったかを前面に
8エラーは事実ベース「Uh oh」禁止。原因と修正を並記
9リストは 5 件まで超えたら now/later や must/nice で分割
10 preamble / recap / closer 禁止「Great question」「Hope this helps!」等を排除

例外（ルールを緩める条件）

  • ユーザーが explain / walk me through を要求 → 長文 OK。ただし preamble と closer は禁止、見出しでスキム可能に。
  • 破壊的操作の直前 → 確認を優先（安全 > 簡潔）。
  • 3 ターン連続でデバッグ失敗 → コード修正ループを止め、仮説を 1 つ質問。
  • 要求が本当に曖昧 → 短い確認質問 1 つ。

送信前チェック（Pre-send）

SKILL.md 末尾のチェックリスト: 冒頭の「これから〜します」、末尾の「他に？」、by the way サイドバー、情報ゼロの hedging を削除。最初と最後の行だけ読んでも (a) 次に何をするか (b) 何が起きたかが分かるか確認する。

インストール

対象は Claude Code のみ。詳細は INSTALL.md。

1. ローカルプラグイン（推奨）

git clone https://github.com/ayghri/i-have-adhd ~/i-have-adhd
claude plugin marketplace add ~/i-have-adhd
claude plugin install i-have-adhd@i-have-adhd

検証: claude plugin list に i-have-adhd が enabled で出る。セッションで /i-have-adhd を実行。

2. プロジェクト単位（marketplace なし）

mkdir -p .claude/skills/i-have-adhd
cp skills/i-have-adhd/SKILL.md .claude/skills/i-have-adhd/SKILL.md
mkdir -p .claude/commands
cp commands/i-have-adhd.md .claude/commands/i-have-adhd.md

作業ディレクトリの .claude/skills/ と .claude/commands/ を自動検出。そのリポジトリ内だけ有効。

3. ユーザー全体（全プロジェクト）

bash install-user.sh

リポジトリルートで実行。~/.claude/skills/i-have-adhd/SKILL.md と ~/.claude/commands/i-have-adhd.md にコピー。$CLAUDE_CONFIG_DIR があればそちらを優先。再実行は上書きで安全。

常時 ON（任意）: ~/.claude/CLAUDE.md に出力スタイルの指示を追記すれば、毎セッション最初からルール適用可能（/i-have-adhd 不要）。INSTALL.md の Always-on 節を参照。
経路スコープ切り替え
プラグインマシン全体（marketplace 経由）/i-have-adhd で ON、stop/normal で OFF
プロジェクト skillその repo のみ同上
ユーザー skill全プロジェクト同上（セッションごとに opt-in）
CLAUDE.md 追記設定に依存常時。OFF にするには記述を削除
カスタマイズ

編集対象は skills/i-have-adhd/SKILL.md のみ。変更は次回 /i-have-adhd 実行時に反映（反映されない場合は Claude Code 再起動）。

  • frontmatter description — モデルが「いつスキルを発火するか」を読む。広すぎるとノイズ、狭すぎると取りこぼし。
  • 本文 Rules — 追加・削除・文言調整は自由。10 条は出発点。
  • Pre-send check — モデルが自己検査する最終ゲート。厳しさを上げたいときはここを厚くする。
FAQ
質問回答
ただの「簡潔モード」？いいえ。10 条は読み方・着手・ドーパミン配分まで含む形状指定。短さ ≠ action-first。
長い説明は壊れる？explain 要求時は長文可。preamble/closer 禁止と見出し構造は維持。
/i-have-adhd なしで使える？CLAUDE.md 追記で常時化可能（INSTALL.md）。
プラグイン vs CLAUDE.mdプラグインはセッション単位 ON/OFF と共有に向く。CLAUDE.md は常時適用向け。
ADHD 診断は必要？不要。preamble 探しのスクロールが負担なら対象。

agentmemory

コーディングエージェント向け永続メモリ — MCP・フック・REST の単一サーバ

1 — 概要
何を解決するか

agentmemory は、セッションをまたいで設計・バグ・好みを再説明しなくてよいように、エージェントの行動を観測して圧縮・索引化し、次回のプロンプトに必要な文脈だけを注入する層だ。CLAUDE.md や .cursorrules のような短い静的メモとは別物で、検索可能な長期記憶とライフサイクル（remember / forget / 監査）を扱う。

4 — 接続方式
起動とクライアント

README のクイックスタートは単一コマンドからサーバを立ち上げる形だ。

npx @agentmemory/agentmemory
  • Claude Code: 複数のライフサイクルフックと MCP・スキル（remember / recall / forget 等）を同梱する想定。
  • その他エージェント: Cursor・Gemini CLI・Codex・OpenCode・Cline など、MCP または REST を喋れるクライアントを README の表で列挙。
  • スタンドアロン MCP: v0.9 以降、実行中サーバへのプロキシでフックとビューアの整合を取る、というリリースノート上の変更がある。
ドキュメント上のランディング

製品サイト agent-memory.dev や、ファイルシステム連携パッケージ @agentmemory/fs-watcher は CHANGELOG / README で案内されている。詳細は上流の該当セクションを参照。

6 — セキュリティと運用
上流の注意喚起

.github/security-advisories/ に個別の脆弱性・ハードニング記事が置かれている（ビューア XSS、デフォルトバインド、メッシュ未認証、エクスポートのパストラバーサル等）。運用前に該当番号を読み、バインドアドレス・認証・エクスポートパスを制限すること。

削除と監査。 README と CHANGELOG では、全削除パスにわたる監査ポリシーの明文化や、ヘルスチェックが小規模 Node プロセスで誤検知しないよう調整した、といった記述がある。本番ではログとアラート閾値を環境に合わせて確認する。

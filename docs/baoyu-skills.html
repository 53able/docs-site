1 — 概要
何ができるか

baoyu-skills は、JimLiu（Baoyu）が公開する Claude Code 向けのスキルコレクション。コンテンツ生成・SNS 投稿・翻訳・AI 画像生成・動画処理など日常の繰り返し作業を、スラッシュコマンド一本で自動化する。

18
スキル総数
3
カテゴリ
MIT
ライセンス

スキルは コンテンツスキル（SNS 向けビジュアル生成・投稿）・AI 生成スキル（多プロバイダ画像生成）・ユーティリティスキル（URL→MD・翻訳・圧縮など）の 3 カテゴリに整理されている。

Utility7YouTube 字幕取得・URL→Markdown・X→Markdown・翻訳・圧縮・フォーマット・HTML 変換
2 — コンテンツスキル
SNS・ビジュアル・投稿

コンテンツ生成と SNS 投稿に特化した 9 スキル。インフォグラフィック・スライドデッキ・コミック・記事カバー画像の生成から、X・WeChat・Weibo への自動投稿まで対応する。

/baoyu-xhs-images

小紅書（Xiaohongshu）向け 1〜10 枚のカートゥーン風インフォグラフィックシリーズ。スタイル 9 種 × レイアウト 6 種の 2 次元システム。

/baoyu-xhs-images article.md --style notion --layout dense
/baoyu-infographic

20 レイアウト × 17 スタイルの組み合わせで構成できるインフォグラフィック。内容を分析してベストな組み合わせを自動推薦する。

/baoyu-infographic content.md --layout pyramid --style technical-schematic
/baoyu-cover-image

記事カバー画像を 5 次元（Type × Palette × Rendering × Text × Mood）で生成。9 カラーパレット × 6 レンダリングスタイルで 54 種の組み合わせ。

/baoyu-cover-image article.md --type conceptual --palette cool
/baoyu-slide-deck

スライドデッキ画像を生成。スタイルは 4 次元（Texture × Mood × Typography × Density）で構築され、16 プリセットを用意。完成後に .pptx / .pdf へ自動結合。

/baoyu-slide-deck article.md --style blueprint --slides 15
/baoyu-comic

知識コミックを生成。アートスタイル（ligne-claire / manga / realistic / ink-brush / chalk）× トーン 7 種。ohmsha・wuxia・shoujo プリセットも搭載。

/baoyu-comic source.md --style ohmsha --layout webtoon
/baoyu-article-illustrator

記事の構造を分析し、図解が必要な箇所を自動識別してイラストを挿入。Type × Style 2 次元アプローチ。

/baoyu-article-illustrator article.md --type flowchart --style blueprint
/baoyu-post-to-x

X（Twitter）にテキスト・画像付き投稿、または長文の X Article（Markdown）を投稿。Chrome CDP でアンチ自動化を回避。ユーザーが内容確認後に手動投稿。

/baoyu-post-to-x --article post.md
/baoyu-post-to-wechat

WeChat 公式アカウント（微信公众号）への投稿。画像テキスト形式と長文記事形式に対応。API 認証または Chrome ブラウザ経由の 2 方式をサポート。

/baoyu-post-to-wechat 文章 --markdown article.md --theme grace
/baoyu-post-to-weibo

Weibo（微博）へのテキスト・画像・動画付き投稿、または頭条文章（長文記事）を投稿。Chrome CDP 経由。

/baoyu-post-to-weibo --article article.md --cover cover.jpg

3 — AI 生成スキル
マルチプロバイダ画像生成

複数の AI 画像生成プロバイダを統一インターフェースで扱う 2 スキル。API キーがあるプロバイダを自動検出し、最適なものを使用する。

/baoyu-image-gen

AI SDK ベースの画像生成。OpenAI・Google・OpenRouter・DashScope（阿里云通义万象）・Jimeng（即梦）・Seedream（豆包）・Replicate に対応。参照画像（--ref）、アスペクト比、品質プリセットをサポート。

/baoyu-image-gen --prompt "A cat" --image cat.png --provider google --ar 16:9
/baoyu-danger-gemini-web

Gemini Web のリバースエンジニアリング API でテキスト・画像を生成。Chrome クッキーで認証。API キー不要だが非公式のため安定性は保証されない。

/baoyu-danger-gemini-web --prompt "A cute cat" --image cat.png

プロバイダ環境変数デフォルトモデル

複数の API キーが設定されている場合は Google を優先。--provider で明示指定も可。

danger スキルについて — baoyu-danger-* プレフィックスのスキルは非公式 API を使用する。サービス側の変更で突然動作しなくなる可能性がある。本番環境での利用は推奨しない。
4 — ユーティリティスキル
コンテンツ処理ツール群

URL 取得・翻訳・字幕抽出・圧縮・フォーマット・HTML 変換など、コンテンツ処理に使う 7 スキル。

/baoyu-youtube-transcript

YouTube 動画のトランスクリプト・字幕・カバー画像をダウンロード。多言語・翻訳・チャプター・話者識別に対応。生データをキャッシュして再フォーマットを高速化。

/baoyu-youtube-transcript https://youtu.be/VIDEO_ID --chapters --speakers
/baoyu-url-to-markdown

Chrome CDP で任意 URL を取得し、Defuddle でクリーンな Markdown に変換。HTML スナップショットも保存。ログイン要のページは --wait モードで対応。

/baoyu-url-to-markdown https://example.com/article -o output.md
/baoyu-danger-x-to-markdown

X（Twitter）のツイートスレッドや X Article を Markdown に変換。非公式リバースエンジニアリング API 経由。

/baoyu-danger-x-to-markdown https://x.com/user/status/123456
/baoyu-compress-image

画像ファイル・ディレクトリを品質を保ちつつ圧縮。ファイルサイズを削減する。

/baoyu-compress-image images/ --quality 80
/baoyu-format-markdown

Markdown のフロントマター・見出し・太字・リスト・コードブロックを整形。タイポグラフィスクリプトも自動実行（ASCII→全角・CJK スペース）。

/baoyu-format-markdown draft.md
/baoyu-markdown-to-html

Markdown を WeChat 互換テーマ付き HTML に変換。シンタックスハイライト・外部リンク脚注対応。

/baoyu-markdown-to-html article.md --theme grace --cite
/baoyu-translate

3 モード翻訳（quick・normal・refined）。対象は Markdown ファイル・URL・テキスト。カスタム用語集・対象読者プロファイル・スタイル指定が可能。長文は並列サブエージェントで自動チャンク処理。

/baoyu-translate article.md --mode refined --to zh-CN --audience technical
baoyu-translate の翻訳モード
モードステップ最適用途
quick直接翻訳短文・インフォーマルなコンテンツ
normal分析 → 翻訳記事・ブログ投稿
refined分析 → 翻訳 → レビュー → 磨き上げ出版品質のドキュメント
5 — インストール
3 つの方法

方法 1：npx skills（推奨）

npx skills add jimliu/baoyu-skills

方法 2：プラグインマーケットプレース登録

# Claude Code 内で実行
/plugin marketplace add JimLiu/baoyu-skills

方法 3：ClawHub から個別インストール

clawhub install baoyu-image-gen
clawhub install baoyu-markdown-to-html
API キーの設定

画像生成スキルなど一部のスキルには API キーが必要。~/.baoyu-skills/.env（ユーザーレベル）または .baoyu-skills/.env（プロジェクトレベル）に記載する。

mkdir -p ~/.baoyu-skills
cat > ~/.baoyu-skills/.env <<'EOF'
# Google Gemini
GOOGLE_API_KEY=xxx

# OpenAI
OPENAI_API_KEY=sk-xxx

# OpenRouter
OPENROUTER_API_KEY=sk-or-xxx
EOF

プロジェクトレベルの .baoyu-skills/.env は .gitignore に追記して秘密情報のコミットを防ぐこと。

カスタマイズ（EXTEND.md）

スキルごとに EXTEND.md を作成してデフォルトを上書きできる。プロジェクトレベル（.baoyu-skills/<skill-name>/EXTEND.md）とユーザーレベル（~/.baoyu-skills/<skill-name>/EXTEND.md）の 2 段階で管理可能。

mkdir -p .baoyu-skills/baoyu-cover-image
# .baoyu-skills/baoyu-cover-image/EXTEND.md にカスタムパレットを定義

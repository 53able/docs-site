ByteStash

byte·stash — self-hosted code snippet manager with multi-fragment, sharing & OIDC.

1 — 概要
何か

ByteStash は、コードスニペットをセキュアに保存・整理・管理するためのセルフホスト型 Web アプリケーション。フロントエンドは React + Tailwind CSS、バックエンドは Node.js + Express、永続化は SQLite。Docker イメージで1コマンドで起動でき、JWT 認証・OIDC（シングルサインオン）・API キー認証を備える。

1 つのスニペットに複数の CodeFragment（ファイル）を持たせられるのが特徴で、プロジェクトをまたぐ関連コードをまとめて管理できる。カテゴリ・ピン留め・お気に入り・公開共有・有効期限付き共有リンクなど、実用的な機能が揃っている。

2 — 主要機能
機能一覧
✏️

スニペット作成・編集

タイトル・説明・カテゴリ・複数フラグメントを持つスニペットをシンタックスハイライト付きエディタで編集。

🔍

検索・フィルタ

言語・キーワード・カテゴリでスニペットを絞り込み。デバウンス付きリアルタイム検索。

🔗

共有リンク

認証要否・閲覧回数上限・有効期限を設定した共有リンクを発行。埋め込み表示にも対応。

🌐

公開スニペット

スニペットを公開設定にするとログインなしで閲覧可能。/public ルートで一覧表示。

📌

ピン留め・お気に入り

スニペットをピン留め（is_pinned）またはお気に入り（is_favorite）に設定して優先表示。

🔐

認証・アクセス制御

JWT セッション・OIDC（外部 IdP）・API キーの 3 方式。新規アカウント登録やアカウント機能自体の無効化も設定可能。

🌍

i18n（多言語）

i18next ベースの国際化対応。新ロケールは i18n:extract コマンドで追加フローが整備されている。

📖

Swagger API ドキュメント

サーバー起動後に /api-docs でインタラクティブな API 仕様を確認・試行できる。

CodeFragment

1 つのスニペットに複数の CodeFragment を持てるのが ByteStash の特徴。関連するファイルをひとまとめで管理できる。

フィールド型説明
idstring?フラグメント ID（新規作成時は省略可）
file_namestringファイル名（例: index.ts）
codestringコード本体
languagestringプログラミング言語（ハイライト識別子）
positionnumber表示順（昇順）
Share（共有リンク）
フィールド型説明
idstring共有トークン ID
snippet_idnumber対象スニペット
requires_authnumber (0/1)閲覧に認証が必要か
view_limitnumber | null閲覧回数上限（null = 無制限）
view_countnumber現在の閲覧回数
expires_atstring有効期限
expirednumber (0/1)期限切れフラグ
 
// TypeScript インターフェース（client/src/types/snippets.ts）
interface CodeFragment {
  id?: string;
  file_name: string;
  code: string;
  language: string;
  position: number;
}

interface Snippet {
  id: string;
  title: string;
  description: string;
  updated_at: string;
  expiry_date?: string;
  categories: string[];
  fragments: CodeFragment[];
  share_count?: number;
  is_public: number;
  is_pinned: number;
  is_favorite: number;
  username?: string;
}
5 — API ルート
主要エンドポイント

詳細な仕様はサーバー起動後に /api-docs（Swagger UI）で確認できる。以下は主要ルートのサマリ。

メソッドパス説明

DELETE/api/snippets/:idスニペット削除

POST/api/auth/loginログイン・JWT 発行

GET/api/share/:shareId共有リンクで閲覧

GET/api/apikeysAPI キー一覧

6 — デプロイ
Docker Compose（最小構成） 
services:
  bytestash:
    image: "ghcr.io/jordan-dalby/bytestash:latest"
    restart: always
    volumes:
      - /your/snippet/path:/data/snippets
    ports:
      - "5000:5000"
    environment:
      BASE_PATH: ""
      JWT_SECRET: your-secret-here
      TOKEN_EXPIRY: 24h
      ALLOW_NEW_ACCOUNTS: "true"
      DISABLE_ACCOUNTS: "false"
      DEBUG: "false"
環境変数リファレンス
変数名デフォルト説明
JWT_SECRET—JWT 署名シークレット（必須）
TOKEN_EXPIRY24hJWT の有効期間
BASE_PATH""サブパスでホストする場合のプレフィックス
ALLOW_NEW_ACCOUNTStrue新規アカウント登録を許可するか
DISABLE_ACCOUNTSfalseアカウント機能を完全に無効化
DISABLE_INTERNAL_ACCOUNTSfalse内部アカウント（非 OIDC）を無効化
ALLOWED_HOSTS—許可ホスト名（カンマ区切り）
DEBUGfalse詳細ログ出力
OIDC_ENABLEDfalseOIDC 認証を有効化
OIDC_ISSUER_URL—OIDC プロバイダーの issuer URL
OIDC_CLIENT_ID—OIDC クライアント ID
OIDC_CLIENT_SECRET—OIDC クライアントシークレット
OIDC_SCOPES—要求するスコープ（スペース区切り）
OIDC_DISPLAY_NAME—ログインボタンに表示する IdP 名

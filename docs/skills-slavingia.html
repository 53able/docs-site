Sahil Lavingia (slavingia) の著書に基づく Claude Code スキル集 — 10 skills / minimalist-entrepreneur

01 — 概要
What is this?

skills-slavingia は、Gumroad 創業者 Sahil Lavingia の著書 The Minimalist Entrepreneur（ミニマリスト起業家）の哲学を Claude Code スキルとして実装したコレクション。コミュニティ発見から持続可能な成長まで、起業家の 10 ステップをエージェントがガイドする。

「プロダクトから始めるな、コミュニティから始めろ」「週末でシップできないなら削れ」——書籍のエッセンスをコンテキストとして持ち、ビジネス上の意思決定を Lavingia の視点でレビューしてくれる。

02 — ジャーニー
起業家の 10 ステップ

書籍の章構成に対応。各ステップが前のステップの上に積み上がる。

STEP 01
Community — 属するコミュニティを見つける
ビジネスアイデアの前にコミュニティ。自分がすでに属する場所、繰り返し聞く不満が起点。
/find-community
STEP 02
Validate — コードを書く前に売る
バリデーションは建設ではなく販売で行う。手作業でも解決できるなら、それが最初の MVP。
/validate-idea
STEP 03
Build — 週末でシップできるものだけ作る
Manual → Processized → Productized の 3 段階。最初は自分が製品。フォームとリストだけで十分。
/mvp
STEP 04
Processize — 手作業を仕組みにする
「休暇中でも誰かが引き継げる紙」を作る。自動化はその後。
/processize
STEP 05
Sell — 最初の 100 人を一人ずつ獲得する
スケールより前に、一人ひとりのカスタマーに向き合う。ファンネルではなく関係性。
/first-customers
STEP 06
Price — 初日から課金する
0 円と 1 円の間には巨大なギャップがある（ゼロプライス効果）。早期割引より正当な価格を。
/pricing
STEP 07
Market — コンテンツでオーディエンスを構築する
プロダクト・マーケット・フィット達成後。広告より、価値あるコンテンツで信頼を積む。
/marketing-plan
STEP 08
Grow — 利益を維持しながら拡大する
採用・支出・スケールの判断を収益ベースで行う。VC ではなく顧客にアカウンタブルに。
/grow-sustainably
STEP 09
Culture — 住みたい家を設計する
採用前に価値観を定義する。採用はカルチャーの希薄化リスク。最初の数人で文化が決まる。
/company-values
STEP 10
Review — すべての決断をミニマリスト原則で見直す
新機能・施策・戦略変更の前に一度立ち止まる。「本当に必要か？」を問い続ける。
/minimalist-review

04 — インストール
Claude Code から

Claude Code のプラグインマーケットプレイスから 2 コマンドで導入できる:

/plugin marketplace add slavingia/skills
/plugin install minimalist-entrepreneur

これだけで 10 スキルすべてが自動登録される。

ローカルクローン から

リポジトリをローカルにクローンして使う場合:

git clone https://github.com/slavingia/skills.git ~/.claude/plugins/skills

その後 Claude Code で:

/plugin marketplace add ~/.claude/plugins/skills
/plugin install minimalist-entrepreneur
スキルの使い方： 各スキルは起動すると Lavingia のビジネス哲学に基づいたアドバイザーとして振る舞う。例えば /validate-idea では、アイデアの詳細を話しかけると「Validated / Needs more validation / Pivot」の判定が返ってくる。

Career-Ops

AI コーディング CLI 向け求職パイプライン — santifer/career-ops

1 — 概要
MIT Claude Code / Gemini / OpenCode Node.js + Go TUI Playwright

Career-Ops は、AI コーディング CLI（Claude Code、Gemini CLI、OpenCode など）を求職コマンドセンターに変えるオープンソースのパイプラインです。スプレッドシートでの手動管理の代わりに、求人評価・ATS 向け CV 生成・ポータルスキャン・バッチ処理・トラッカー整合性を一つのワークフローにまとめます。

キーワードマッチではなく、エージェントが CV と求人票を突き合わせて適合度を推論します。応募の自動送信は行わず、評価と推奨のあと最終判断は常に人間が下します。

3 — 評価フロー（単一求人）

求人 URL または JD テキストを入力すると、Playwright / WebFetch で JD を取得し、アーキタイプ分類のうえ 6 ブロック（A–F）で評価します。10 次元の重み付きスコア（1–5）の平均が最終スコアになります。

ブロック内容
A役割サマリー
BCV マッチ（ギャップと緩和策）
Cレベル戦略
D報酬調査（WebSearch）
ECV パーソナライズ計画
F面接準備（STAR+R ストーリー）

レポートヘッダには Legitimacy（掲載の信頼性ティア）も含まれます。PDF は generate-pdf.mjs（Playwright）で HTML テンプレートから生成し、トラッカー行は merge-tracker.mjs で applications.md にマージされます。

4 — 主要機能
機能説明
自動パイプラインURL を貼ると評価・PDF・トラッカー記録まで一連で処理
面接ストーリーバンク評価ごとに STAR+Reflection を蓄積（interview-prep/story-bank.md）
ATS 向け PDF求人ごとにキーワードを注入した CV（Space Grotesk + DM Sans テンプレート）
ポータルスキャナーscan.mjs が Greenhouse / Ashby / Lever API を直接叩き LLM コストゼロで一覧取得
バッチ処理batch/batch-runner.sh が headless CLI ワーカーを並列起動
ダッシュボード TUIGo 製。スコア・日付・ステータスでフィルタ・ソート
パイプライン整合性merge-tracker / dedup-tracker / verify-pipeline など
多言語モードmodes/ja、modes/de などロケール別の応募フロー

対応 CLI（抜粋）

open agent skill standard に従う CLI なら原則動作します。README では Claude Code、OpenCode、Gemini CLI を明示し、Codex は準備中と記載されています。スキル定義は .agents/skills/career-ops/SKILL.md にあります。

5 — データ契約とカスタマイズ

DATA_CONTRACT.md と AGENTS.md は、ユーザーデータとシステムファイルを分離します。

層例更新
ユーザー層 cv.md、config/profile.yml、modes/_profile.md、portals.yml、data/*、reports/* 自動更新しない。カスタマイズはここへ
システム層 modes/_shared.md、AGENTS.md、*.mjs、templates/*、batch/* update-system.mjs で更新可能
カスタマイズのルール: アーキタイプ・交渉スクリプト・好みなどユーザー固有の変更は modes/_profile.md または config/profile.yml に書く。modes/_shared.md をユーザー内容で上書きしない（システム更新で失われるため）。

初回セッションでは cv.md、profile.yml、_profile.md、portals.yml の有無を静かに確認し、未設定ならオンボーディングに誘導する、と AGENTS.md が定めています。

6 — クイックスタート
git clone https://github.com/santifer/career-ops.git
cd career-ops && npm install
npx playwright install chromium
npm run doctor

cp config/profile.example.yml config/profile.yml
cp templates/portals.example.yml portals.yml
# ルートに cv.md を用意

claude   # または利用中の AI CLI
# URL を貼る /career-ops で評価開始

モード・アーキタイプ・スコア重みは、CLI 上で「バックエンド向けにアーキタイプを変えて」などと依頼すればエージェントが同じファイルを編集する設計です。詳細手順はリポジトリ内の docs/SETUP.md を参照してください。

npm スクリプト（抜粋）

  • npm run doctor — 前提条件の検証
  • npm run verify — パイプライン健全性
  • npm run scan — ポータルスキャン（API 直叩き）
  • npm run pdf — PDF 生成
  • npm run update:check — システム層の更新確認

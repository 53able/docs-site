1 — 概要
Waza / 技

エンジニアリング習慣を
Claude Code スキルに

Waza (技) は武道の術語で「繰り返すことで本能になる動作」を意味する。 優れたエンジニアはコードを書くだけでなく、要件を咀嚼し、自分のコードを見直し、体系的にデバッグし、意図のあるインターフェイスを設計し、一次情報を読む。 AI はスピードを上げてくれるが、思考の明確さや慎重なリリースや深い理解は与えてくれない。 Waza はこれらの習慣のそれぞれを、Claude Code が実行できるスキルに変換する。

コンセプト: Superpowers や gstack のような高機能スキルセットは重い。スキル数が多く、設定が複雑で、学習コストが高い。Waza は逆で、本当に重要な習慣だけを 8 つに絞る。各スキルは 1 つのことを行い、明確な発動条件を持ち、邪魔にならない。

2 — スキル一覧
8 つのスラッシュコマンド
コマンド 発動タイミング 何をするか
/think 新機能・設計・アーキテクチャ判断の前 要件を咀嚼し、設計を圧力テストし、1行もコードを書く前にアーキテクチャを検証する。バグ修正や小さな編集には使わない。
/design UI・コンポーネント・画面を構築するとき 明確な審美的方向性を持つプロダクション品質の UI を生成する。一般的なデフォルトではなく、コミットした視点を持つ。
/check タスク完了後・マージ前 差分を読み、安全な問題を自動修正し、大きな差分にはセキュリティ・アーキテクチャの専門レビュアーを起動し、証拠で検証する。
/hunt エラー・クラッシュ・予期しない挙動・テスト失敗 体系的デバッグ。修正を適用する前に根本原因を特定する。コードレビューや新機能には使わない。
/write 文章の執筆・編集・推敲を明示的に依頼されたとき 中国語・英語の文章を自然に聞こえるよう書き直す。AI 文体のクセを除去する。コメント・コミットメッセージ・インライン docs には使わない。
/learn 未知のドメインへの深入り・研究記事の準備 収集・消化・アウトライン・書き込み・精錬・公開の 6フェーズワークフロー。クイック検索や単一ファイル読み取りには使わない。
/read URL・ウェブページリンク・PDF を読むとき プロキシカスケードスクリプトでコンテンツをクリーンな Markdown として取得・保存。WeChat と Feishu 専用ハンドラーを持つ。
/health Claude が指示を無視したり、動作が不安定なとき 6層設定スタック (CLAUDE.md → rules → skills → hooks → subagents → verifiers) を監査し、問題を深刻度別に報告する。
スキルの構造

各スキルは単一の Markdown ファイルではなく、フォルダとして構成される。SKILL.md のほかに参照ドキュメント・補助スクリプト・スコープ付きフック・実際のプロジェクト失敗から蓄積した Gotchas セクションを含む。

各スキルは YAML フロントマター (name, description, version, allowed-tools) を持ち、marketplace.json のバージョンと一致している必要がある。

3 — 主要スキルの詳細
/think — 設計・検証フロー

コードを書く前に考える。コードもスキャフォールディングも、ユーザーが設計を承認するまで一切実装しない。

Phase 0
深度評価
タスクのスコープを判定: Lightweight / Standard / Deep。外部コントラクト面が判明したら自動でレベルアップ。
Phase 1
問題理解
git log を実行し CLAUDE.md を読む。目的→制約→成功基準の順に一度に一つの質問で検証する。
Phase 2
アプローチ提案
2〜3 の選択肢をトレードオフ付きで提示。最小オプションとアーキテクチャ完全オプションを必ず含める。
Phase 3
アーキテクチャ検証
スコープ・依存関係・テストカバレッジ・リスクを検証。3 コンポーネント以上がデータを交換する場合は ASCII 図を描く。
Phase 4
信頼度チェック
問題の理解・最もシンプルなアプローチか・未解決事項の整理の 3 軸でスコアを付ける。低スコアならループバック。
モード
スコープモード
expand (新機能) / shape (既存追加) / hold (バグ修正) / cut (肥大化したプラン) の 4 モードで姿勢を決める。
/check — レビュー深度分類

差分を読み、問題を見つけ、安全に修正できるものを修正し、残りを確認する。検証が走るまで「完了」と言わない。

深度基準起動するレビュアー
Quick 100 行未満、1〜5 ファイル ベースレビューのみ (このスキル)
Standard 100〜500 行、または 6〜10 ファイル ベース + 条件付き専門レビュアー
Deep 500+ 行、または 10+ ファイル、または auth/payments/データ変更に触れる ベース + 全専門レビュアー + 敵対的パス

修正をクラス分けする: safe_auto (即時適用) / gated_auto (一括確認) / manual (判断が必要) / advisory (情報提供のみ)。

/hunt — 根本原因ファースト

症状に当てたパッチは別のバグを生む。まず根本原因を見つける。1 文で根本原因を述べられるまでコードに触れない。

既知の失敗パターン

形状手がかり調べる場所
タイミング問題断続的、負荷依存共有ステートへの並行アクセス
ガード欠落フィールドやインデックスアクセスでクラッシュオプション値、空コレクション
順序バグ単体では動くが連続では失敗イベントコールバック、トランザクションスコープ
境界失敗タイムアウト、不正なレスポンス形状外部 API、サービスエッジ
環境不一致ローカル通過、CI 失敗環境変数、機能フラグ、シードデータ
古い値古いデータ表示、再起動で解消インメモリキャッシュ、メモ化結果
3 回失敗したら止まる: 仮説が 3 回外れた場合、4 回目を推測しない。何を調べたか・何を除外したか・何が未解明かをユーザーに伝え、アプローチを変えるか確認する。
/health — 6 層設定スタック

Claude が指示を無視したり、動作が不安定なとき、設定の積み重ねを監査する。

Layer 1
CLAUDE.md
グローバル + ローカル。コンテキスト予算・通信スタイル・プロジェクト固有ルール。
Layer 2
rules
.claude/rules/ の Markdown ファイル。起動時にコンテキストへ自動ロード。
Layer 3
skills
オンデマンドで読み込まれるプレイブック。YAML フロントマターで発動条件・許可ツールを制御。
Layer 4
hooks
PreToolUse / PostToolUse フック。check スキルは Bash コマンドの破壊的操作検知フックを持つ。
Layer 5
subagents
並行レビュアーエージェント。health スキルは Agent tool で agent1 / agent2 を同時起動する。
Layer 6
verifiers
scripts/verify.sh。check スキルは全修正後にこれを実行し、「完了」は実行 + 通過を意味する。
共通の落とし穴 (実際の失敗から)
  • パスの思い込み: ファイルを ~/project に移したが、リポジトリは ~/www/project にあった。最初のファイルシステム操作前に必ず pwd か git rev-parse --show-toplevel を実行する。
  • 実行を求められているのに分析した: ユーザーが「帮我做 (やってくれ)」と言ったのに 3 つの選択肢を出した。「やって」「最適化して」「元に戻して」は即実行を意味する。
  • 検証を「些細な変更」でスキップ: 1 行修正でも things を壊す。スキップしたい衝動が湧いたら scripts/verify.sh を実行する。
  • 間違ったコードパスを修正: 症状から場所を推測してクライアントペインを修正したが、問題はローカルペインにあった。ファイルを触る前に実行パスを症状から逆向きに追う。
  • 同じ症状が 4 回現れた: 修正を加えるたびに同じエラーが戻ってきた。同じ症状が修正後に現れたら止まる。前の仮説が間違っていた。実行パスを最初から再度読む。
  • 使えないツールを前提に設計した: MCP 依存のワークフローを計画したが、MCP サーバーがロードされていなかった。外部ツールの可用性を最初の設計ステップの前に確認する。
4 — 追加機能
Statusline

重要なことだけを表示するミニマルな Claude Code ステータスライン。コンテキストウィンドウ使用量、5 時間クォータ、7 日クォータ、それぞれのリセットまでの残り時間を表示する。

カラーコーディング: コンテキストは 70% 未満で緑、70〜85% で黄、85% 超で赤。クォータは青・マゼンタ・赤。プログレスバーなし、ノイズなし。

curl -sL https://raw.githubusercontent.com/tw93/Waza/main/scripts/setup-statusline.sh | bash
English Coaching

英語はすべてのエンジニアが AI と作業する際の第一言語であるべきだ。モデルは英語で考え、最良のリソースは英語で書かれており、英語で明確に書くスキルは時間とともに複利で成長する。

全返答でパッシブ文法チェックが動き、Claude が間違いをパターン名と共にフラグする。

curl -sL https://raw.githubusercontent.com/tw93/Waza/main/templates/english-coaching.md >> ~/.claude/CLAUDE.md
5 — インストール
全スキル一括インストール

Node 18+ と Claude Code が必要。

npx skills add tw93/Waza -g -y
単一スキルのインストール

health を任意のスキル名に置き換える。

npx skills add tw93/Waza -a claude-code -s health -y

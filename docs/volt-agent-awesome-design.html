Awesome DESIGN.md

VoltAgent/awesome-design-md — 開発者向けサイトから抽出した DESIGN.md を、AI コーディングエージェントが読めるプレーンテキストのデザインシステムとして公開するコレクション。

1 — 概要
何か

Awesome DESIGN.md は、Stripe・Vercel・Notion など実在する開発者向けサイトのビジュアル言語を、DESIGN.md という単一 Markdown ファイルに落とし込んだカタログ。Figma エクスポートや JSON スキーマは不要で、プロジェクトルートにコピーするだけでエージェントが色・タイポ・コンポーネント規約を参照できる。

73DESIGN.md 件数
10カテゴリ群
9標準セクション
MITライセンス

各エントリは公開サイトの CSS 値を分析した「インスパイア解釈」であり、トークン・パターン・ルールまで含む実装向けの記述を目指している。README では「build me a page that looks like this」と指示するだけで、表面だけでない UI 生成を狙うと説明されている。

2 — DESIGN.md とは
AGENTS.md との役割分担

Google Stitch が提唱するプレーンテキストのデザインシステム文書。LLM が最も読みやすい Markdown 形式のため、パーサや専用ツールなしでエージェントに渡せる。

ファイル読者定義する内容
AGENTS.mdコーディングエージェントプロジェクトのビルド方法・規約・作業手順
DESIGN.mdデザインエージェント見た目・雰囲気・コンポーネントのルール
フォーマット: 各ファイルは Stitch DESIGN.md 仕様 に沿い、分析結果として拡張セクション（Do's and Don'ts、Responsive、Agent Prompt Guide など）を含む。多くのエントリは先頭に YAML フロントマター（colors / typography など）を持つ。

6 — エージェントへの渡し方
基本フロー 
%%{init: {'theme': 'base', 'themeVariables': { 'primaryColor': '#d4ede0', 'primaryTextColor': '#0f2a1f', 'primaryBorderColor': '#3d9970', 'lineColor': '#3d9970', 'fontSize': '14px'}}}%%
flowchart TD
  A[コレクションからサイトを選ぶ] --> B[DESIGN.md をプロジェクトルートへコピー]
  B --> C[AGENTS.md やスキルで参照を明示]
  C --> D[エージェントに UI 生成を依頼]
  D --> E[preview.html でトークンを目視確認]
  1. 目的のブランドに近い design-md/<slug>/DESIGN.md を選ぶ（または getdesign.md から取得）。
  2. 自分のリポジトリルートに DESIGN.md として配置する。
  3. 「この DESIGN.md に従ってページを作って」とエージェントへ指示する。セクション 9 の Agent Prompt Guide に例文がある。
  4. 必要なら preview.html / preview-dark.html をブラウザで開き、色とタイポの期待値を確認する。

7 — 依頼と貢献
新規サイトと改善
経路内容
getdesign.md/request特定サイトの DESIGN.md 生成を依頼。非公開・優先配送オプションあり。
GitHub Issue（design-md-request テンプレート）URL・配送メール・優先度を記載してオープンリクエスト。
既存ファイルの PR誤った hex・不足トークン・説明の改善は歓迎。ただし新規 DESIGN.md の PR は受け付けない（品質維持のため）。

改善 PR では live サイトと照合し、トークン変更時は preview.html / preview-dark.html も更新する。着手前に Issue で変更内容を共有するよう CONTRIBUTING.md が求めている。

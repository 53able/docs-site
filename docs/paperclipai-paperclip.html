Paperclip

ゼロヒューマン企業のためのオープンソースオーケストレーション（paperclipai/paperclip）。AI エージェントのチームを編成し、ゴールを割り当て、ダッシュボードで作業とコストを追跡する。

1 — 概要
位置づけ

Paperclip は、Node.js サーバーと React UI で、AI エージェントのチームをオーケストレーションして「会社」を運営するオープンソースプロジェクト。OpenClaw が「従業員」なら、Paperclip は「会社」に相当する。自前のエージェントを持ち込み、ゴールを設定し、エージェントの作業とコストを一つのダッシュボードで管理する。

外見はタスクマネージャーのようだが、内部には組織図・予算・ガバナンス・ゴール整合・エージェント間調整がある。プルリクではなくビジネスゴールを管理する。

ロードマップ: Clipmart（プリビルト会社テンプレの公開・ワンクリック投入）は README で COMING SOON として案内されている。

OpenClaw / Claude Code / Codex / Cursor / Bash / HTTP など、ハートビートを受け取れるエージェントなら「雇用」できる。

流れ
  • 01 ゴールを定義する（例: 「AI ノートアプリで $1M MRR を目指す」）
  • 02 チームを雇う（CEO、CTO、エンジニア、デザイナー、マーケター — 任意のボット・プロバイダー）
  • 03 承認して実行 — 戦略をレビューし、予算を設定し、開始。ダッシュボードで監視する
2 — 特徴
主な機能

3 — 解決する問題
Without / With Paperclip
Paperclip なしPaperclip あり
Claude Code のタブが 20 個開いて追えない。再起動で全部消える。タスクはチケットベース、会話はスレッド、セッションは再起動後も永続。
複数ソースから手動でコンテキストを集めてボットに渡している。コンテキストはタスクからプロジェクト・会社ゴールへ流れ、エージェントは常に「何を・なぜ」を把握。
エージェント設定がフォルダに散らばり、タスク管理・連携を毎回自作している。組織図・チケット・委譲・ガバナンスが標準装備。スクリプトの山ではなく会社として運営できる。
暴走ループでトークンが数百ドル消え、気づいたときにはクォータ超過。コスト追跡で予算を可視化し、超過時はエージェントをスロットル。予算で優先度を管理。
定期ジョブ（サポート・SNS・レポート）を手動で起動している。Heartbeats でスケジュール実行。管理職が監督。
アイデアのたびにリポジトリを探し Claude Code を開き、タブを守り続ける。Paperclip にタスクを追加し、コーディングエージェントが完了まで実行。管理側が成果をレビュー。
Paperclip が向いている人
  • 自律 AI 会社を立ち上げたい
  • 複数エージェント（OpenClaw, Codex, Claude, Cursor）を共通ゴールに向けて連携させたい
  • Claude Code を 20 タブ開いていて誰が何をしているか分からなくなる
  • 24/7 自律運用しつつ、監査や必要時の介入を残したい
  • コストを監視し、予算を強制したい
  • タスクマネージャーのような感覚でエージェントを管理したい
  • スマホから自律ビジネスを管理したい
4 — なぜ特別か
オーケストレーションの要点
ポイント説明
Atomic executionタスクチェックアウトと予算強制がアトミック。二重作業と暴走コストを防ぐ。
Persistent agent stateエージェントはハートビートをまたいで同じタスクコンテキストを再開。ゼロからやり直さない。
Runtime skill injectionエージェントは実行時に Paperclip のワークフローやプロジェクトコンテキストを学習可能。再学習不要。
Governance with rollback承認ゲートが強制され、設定はリビジョン管理。悪い変更は安全にロールバック可能。
Goal-aware executionタスクはゴールの親子関係を持ち、エージェントは「なぜ」を一貫して把握できる。
Portable company templates組織・エージェント・スキル等をエクスポート／インポート。シークレットのスクラブと衝突解消付き。
True multi-company isolationすべてのエンティティは会社スコープ。1 デプロイで複数会社を完全に分離したデータ・監査で運用可能。
5 — Paperclip が向いていないもの
README の境界定義（要約）
種別内容
チャットボットではないエージェントは「仕事」を持つ。雑談 UI ではなくチケットと Heartbeat 中心。
エージェントフレームワークではないエージェントの作り方は規定しない。複数エージェントの会社運営にフォーカス。
ワークフロー Builder ではないドラッグ&ドロップのパイプラインではなく、組織図・ゴール・予算・ガバナンスで会社をモデル化。
プロンプトマネージャーではないプロンプト・モデル・ランタイムはエージェント側。Paperclip は組織と作業の座組を管理。
単一エージェント向けではないチーム前提。1 体だけなら過剰になりやすい。
コードレビュー専用ツールではないPR の代替ではなく作業オーケストレーション。レビュープロセスは別途持ち込む。

7 — クイックスタート
セルフホスト・オープンソース

Paperclip アカウントは不要。既定は trusted local（ループバック）で最短初回起動:

npx paperclipai onboard --yes

認証付き／LAN・Tailnet 公開で初めたい場合はバインドプリセットを明示:

npx paperclipai onboard --yes --bind lan
# または
npx paperclipai onboard --yes --bind tailnet

既に設定がある状態で onboard を再実行すると既存設定は維持。変更は paperclipai configure。

手動クローン:

git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev

API は http://localhost:3100。開発時は UI も API サーバのミドルウェア経由で同一オリジン。組み込み PostgreSQL が自動作成され、追加 DB セットアップは不要。

要件: Node.js 20+、pnpm 9.15+（リポジトリの package.json の engines / packageManager に準拠）
8 — 開発
よく使うスクリプト

clone 後のエントリ（README / package.json に整合）:

pnpm dev              # API + UI、ウォッチ
pnpm dev:once         # ウォッチなしで 1 回起動（既定でローカルマイグレーション適用など）
pnpm dev:server       # サーバーのみ
pnpm dev:list         # 管理済み dev ランナーの確認
pnpm dev:stop         # dev ランナー停止
pnpm storybook        # UI Storybook（ポート 6006）
pnpm build            # 全体ビルド
pnpm typecheck        # 型チェック
pnpm test             # 既定は Vitest のみ（README の cheap default）
pnpm test:watch       # Vitest ウォッチ
pnpm test:e2e         # Playwright（ブラウザ。通常フローとは別）
pnpm db:generate      # DB マイグレーション生成
pnpm db:migrate       # マイグレーション適用

pnpm test は Playwright を含まない。ブラウザ系は pnpm test:e2e 等で個別実行。

詳細は上游リポジトリの doc/DEVELOPING.md を参照。

9 — テレメトリ
既定 ON と無効化

匿名の利用状況テレメトリが既定で有効。個人情報・Issue 本文・プロンプト・ファイルパス・シークレットは収集対象外。プライベート repo 参照はインストール単位の salt でハッシュ化される、と README に記載されている。

方法設定
環境変数PAPERCLIP_TELEMETRY_DISABLED=1
標準の追跡拒否DO_NOT_TRACK=1
CICI=true のとき自動無効
設定ファイルtelemetry.enabled: false

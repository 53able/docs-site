oh-my-mermaid

omm — Architecture mirror for vibe coding

1 — 概要
何を解決するか

oh-my-mermaid（omm）は、AI コーディングツール（Claude Code / Codex / Cursor など）と連携し、コードベースの構造を Mermaid 図 + テキストフィールド で継続的に文書化する CLI ツール。

AI writes code in seconds. Humans understand it in hours.
But skip understanding, the codebase becomes a black box — even to you.

omm はこのギャップを埋める。AI に /omm-scan スキルを実行させると、コードベースから パースペクティブ（複数の視点）が生成され、プロジェクトルートの .omm/ ディレクトリに蓄積される。

.omm/ ディレクトリ構造

各パースペクティブはサブディレクトリ。さらにその中に ネスト要素（子ディレクトリ）を持てる。ビュワーはファイルシステムのツリーを自動解析してグループを展開表示する。

.omm/
├── overall-architecture/ ← perspective
│   ├── description.md
│   ├── diagram.mmd
│   ├── context.md
│   ├── cli/ ← nested element
│   │   ├── description.md
│   │   └── diagram.mmd
│   └── server/ ← deeper nesting
│       └── ...
├── data-flow/
└── command-surface/

各要素が持てる 7 つのフィールド:

description diagram context constraint concern todo note

AI スキル（/ コマンド）

スキルはターミナルではなく AI コーディングツール内 で実行するコマンド。omm setup で登録される。

スキル動作
/omm-scanコードベースを解析し、パースペクティブ駆動の再帰的なアーキテクチャ文書を生成・更新する
/omm-pushログイン → リンク → クラウドへプッシュをワンステップで実行
パスの形式: omm write overall-architecture/cli description - <<'EOF' のようにスラッシュで階層を指定。Element IDs は kebab-case で、ディレクトリ名と一致させる。

書き込みパス詳細
AI が omm write 実行 → cli.ts でコマンド解析 → writeNodeField() → prev_diagram を meta.yaml に保存 → field file 書き込み → meta.yaml 更新（created / updated / git_commit） → 親要素の children リスト更新

5 — クラウド連携
ohmymermaid.com

.omm/ をクラウドに同期することで、チーム共有や公開 URL の発行が可能になる。認証トークンは ~/.omm/credentials.json（パーミッション 0600）に保存される。

omm login                    # ブラウザ認証
omm link myorg/my-project    # プロジェクトを slug に紐付け
omm push                     # .omm/ をアップロード
omm share                    # 共有 URL を表示
omm pull                     # クラウドから取得

デフォルトは プライベート。公開共有リンクは omm share で取得できる（例: ohmymermaid.com/share/…）。

6 — クイックスタート
インストールと初回スキャン 
# 1. インストール
npm install -g oh-my-mermaid

# 2. AI ツールへスキルを登録
omm setup         # 全ツールを自動検出
omm setup cursor  # Cursor に限定

# 3. AI ツール内でスキャン実行
/omm-scan

# 4. ブラウザでビュワーを開く
omm view
対応 AI ツール
プラットフォームセットアップコマンド
Claude Codeomm setup claude
Codexomm setup codex

OpenClawomm setup openclaw
Antigravityomm setup antigravity
Web ビュワー

omm view で localhost:3000 にインタラクティブな Web ビュワーが起動する。fs.watch + SSE でファイルの変更を検知しリアルタイムに更新。Mermaid 図はクリックで展開・縮小でき、ネストされた子要素をインタラクティブに探索できる。

# カスタムポートで起動
omm view --port 8080

# バージョン確認
omm --version

# 更新
omm update

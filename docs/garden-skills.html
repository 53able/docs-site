Garden Skills

ConardLi/garden-skills — Claude Code、Cursor、Codex など向けの本番利用を想定した Agent Skills 集（MIT）

4つのスキル

ルートの README で紹介されているカテゴリ別スキル。用途に応じて単体インストールまたは npx skills add で一括取得できる。

web-video-presentation
Web 動画 / プレゼン
Vite + React + TypeScript の 1920×1080 ステージ。章・ステップ単位のナレーション同期、録画向け UI、複数テーマ（paper-press 〜 terminal-green など）。任意で音声合成まで。
web-design-engineer
デザイン / フロント
要件→デザインシステム→v0→本実装→検証の 6 段ワークフロー。AI UI の陳腐化を避ける指針と、React / インライン Babel・oklch()・コンテナクエリ等の実装ルール。
gpt-image-2
画像生成 / プロンプト
Garden ローカル・ホスト委譲・アドバイザのみの 3 モード。18 カテゴリ・80 件超の参照テンプレート、編集ワークフローとスクリプト。公開サイト website/gpt-image2-website でケース閲覧可能。
kb-retriever
ローカル検索 / KB
knowledge/ 階層を data_structure.md で辿り、PDF・Excel は参照ドキュメントを先に読むルール。最大 5 ラウンドで探索を打ち切る。
リリース物: 各スキルの README に Download v<version> .zip リンクがあり、scripts/release/update-readme.mjs によりタグ URL と同期される。CI では npm run validate（一覧・パック・README 整合）が走る。
導入経路

README に定義された 5 方式。用途とピン留め要件で選ぶ。

記号 方法 向いている場面 バージョン固定
A npx skills add ConardLi/garden-skills（skills CLI） エージェント横断の一行導入。単体スキルは -s <name>、グローバルは --global タグ付き tree/ URL でピン留め可
B Claude Code プラグイン（/plugin marketplace add 等） 公式マーケット経由でパック単位（presentation / web-design / knowledge-base / image-generation） マーケットのバージョンに従う
C GitHub Releases の .zip + SHA-256 CI・オフライン・バイト完全一致が必要なとき immutable
D git clone 後に skills/ 以下を手コピー スキル本体をフォーク・改変するとき なし（main 追従）
E Git submodule + シンボリックリンク 大きなプロジェクトにベンダリングし upstream を追うとき サブモジュールのコミット SHA / タグで固定

CLI の代表例

# 4 スキルすべて（最新 main）
npx skills add ConardLi/garden-skills

# 1 スキルのみ
npx skills add ConardLi/garden-skills -s web-design-engineer

# エージェント指定
npx skills add ConardLi/garden-skills -s kb-retriever -a claude-code

# タグでピン留め（例: web-design-engineer のリリースタグ）
npx skills add ConardLi/garden-skills/tree/web-design-engineer-v1.0.0/skills/web-design-engineer

スキルフォルダ規約

各スキルは自己完結ディレクトリ。エージェントは YAML フロントマターの description をトリガ条件として読み込む。

<skill-name>/
├── SKILL.md       # 必須: いつ使うか + 手順
├── README.md      # 人間向け説明
├── manifest.json  # バージョン等（garden-skills では各スキルに付与）
├── references/    # 任意: JiT で読む参照
├── scripts/       # 任意: Node / shell 等
└── assets/        # 任意: テンプレ・フォント等

メンテナ向けのリリース手順・検証はリポジトリ内の CONTRIBUTING.md（日本語は CONTRIBUTING.zh-CN.md）を参照。

agentsview

kenn-io/agentsview — AI コーディングエージェントのセッションをローカルに集約し、閲覧・全文検索・コスト追跡を行う単一バイナリ + Web UI。 アカウント不要、データは端末内に保持する。

初回起動時にマシン上の各エージェントセッションディレクトリを自動検出し、ローカル SQLite に同期したうえで http://127.0.0.1:8080 の Web UI を開く。サーバー不要で agentsview usage だけを使う運用も可能。

概要

agentsview は、Claude Code・Cursor・Codex・Gemini CLI など複数の AI コーディングエージェントが残すセッションログを横断的に扱うためのツール。セッション閲覧、全文検索、トークン使用量とコストの可視化、チーム向け PostgreSQL 同期、分析用 DuckDB ミラーを 1 つのプロダクトにまとめている。

  • 公式サイト: agentsview.io
  • ライセンス: MIT
  • 技術スタック: Go 1.26+（CGO）+ Svelte 5 SPA（Vite）+ 任意で Tauri デスクトップ

アーキテクチャ

データはエージェント固有のセッションファイルから取り込まれ、パーサー層で正規化されたあとストア層（SQLite が主）に格納される。Go サーバーが REST API（Huma）と SSE を提供し、埋め込み済み Svelte フロントエンドが UI を描画する。

インストールと起動

インストール

# macOS / Linux
curl -fsSL https://agentsview.io/install.sh | bash

# Windows
powershell -ExecutionPolicy ByPass -c "irm https://agentsview.io/install.ps1 | iex"

# Homebrew（デスクトップアプリ）
brew install --cask agentsview

クイックスタート

agentsview serve           # サーバー起動 + Web UI を開く
agentsview usage daily     # 直近 30 日の日次コストサマリー
リモートアクセス: デフォルトはループバックバインドで Host ヘッダーを検証する（DNS リバインディング対策）。SSH ポートフォワードやリバースプロキシ経由で開く場合は --public-url にブラウザで開くオリジンを指定する。ループバック外に公開する場合は --require-auth も有効にする。

CLI と API

コスト・使用量（agentsview usage）

ccusage 系ツールの代替として、インデックス済み SQLite から集計するため raw ファイルの毎回パースより高速。LiteLLM レートで自動価格付け（オフライン時はフォールバック）。プロンプトキャッシュトークンもコスト計算に反映。

agentsview usage daily
agentsview usage daily --breakdown --agent claude --since 2026-04-01
agentsview usage daily --all --json
agentsview usage statusline    # シェルプロンプト向け 1 行出力

セッション統計（agentsview stats）

期間内のセッション数、アーキタイプ、ツール/モデル構成、キャッシュ経済性などを JSON v1 スキーマで出力。Git / GitHub 由来の成果指標は --include-git-outcomes 等で明示的に有効化。

REST API

例: セッション単位の使用量は GET /api/v1/sessions/{id}/usage。Huma ベースのルート一覧はリポジトリ内 docs/huma-api-routes.md を参照。

Web UI の機能

  • 全文検索 — メッセージ本文を FTS5 で横断検索
  • コストダッシュボード — セッション/モデル別のトークン・コスト、日次支出チャート
  • アナリティクス — アクティビティヒートマップ、ツール使用、velocity、プロジェクト内訳
  • ライブ更新 — アクティブセッションへの新メッセージを SSE で反映
  • キーボード操作 — j/k、Cmd+K 検索 など
  • エクスポート — HTML エクスポート、GitHub Gist 公開

バックエンドと同期

SQLite（主）

ローカルアーカイブ。ファイル同期、FTS5 検索、書き込み可能な UI。agentsview serve のデフォルト。

PostgreSQL

agentsview pg push でチーム共有 DB へ同期。pg serve は読み取り専用 UI。pg push --watch でデーモン化も可能。

DuckDB / Quack

SQLite からのミラー。duckdb push / duckdb serve。Quack 経由のリモート読み取りはトークン必須・ループバック既定。

役割分担: 取り込みの正は常に SQLite。serve は主アーカイブへ書き込む。PostgreSQL と DuckDB は分析・共有・読み取り専用配信向けのオプション層。

プライバシーとセキュリティ

  • セッションデータは端末内に保持。サーバーはデフォルトで 127.0.0.1 にバインド
  • テレメトリは起動時と 24 時間ごとの匿名 daemon_active イベントのみ（PostHog）。プロンプトやパスは含まない。AGENTSVIEW_TELEMETRY_ENABLED=0 で無効化
  • アップデートチェックは --no-update-check でオフ可能
  • Antigravity CLI の復号サイドカー連携はローカル完結（agy-reader）。信頼モデルは SECURITY.md 参照

開発

要件: Go 1.26+（CGO）、Node.js 22+。

make dev            # Go サーバー（開発モード）
make frontend-dev   # Vite（別ターミナル）
make build          # フロントエンド埋め込みビルド
make test           # Go テスト（fts5 タグ）
make e2e            # Playwright E2E

internal/           Go パッケージ群

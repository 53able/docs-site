Synthadoc

多様な資料を、出典・リンク・状態を持つローカル Markdown wiki へ編成する LLM knowledge compilation engine。

1 — 概要
成果物が主役

Synthadoc は、PDF、表計算、スライド、Web ページ、画像、動画、Word、テキスト、AI セッション記録を読み、相互参照された wiki に編成する。検索時だけ断片を集めるのではなく、取り込み時に内容を統合するため、サーバーを止めても Markdown の知識成果物が残る。

2 — 知識コンパイル
Synthadoc

取り込み時に wiki を更新

新しい資料を要約するだけでなく、既存ページとの関係を調べ、[[wikilinks]]、出典、矛盾、ページ状態を更新する。wiki 自体が検索・編集・監査できる成果物になる。

典型的な RAG

質問時に断片を検索

保存したチャンクから質問に近い断片を取り出し、その場で回答を合成する。Synthadoc はこの query-time synthesis を主役にせず、先に知識を編成しておく。

保護境界: 人が確認して active にしたページは、矛盾する新規ソースで自動上書きしない。矛盾をレビュー対象として残す。
3 — 処理の流れ
source → wiki → answer
01取り込むローカルファイル、URL、Web 検索、YouTube を受け付ける
02安全化する隠し文字や命令上書き表現を LLM 呼び出し前に除去
03編成する内容を統合し、ページ・wikilink・出典を生成
04検証する矛盾、孤立ページ、誇張、切れた引用を lint
05使うBM25 検索、ストリーミング回答、context pack、export
Retrieval

BM25 を基準にし、小規模コーパス向け TF fallback と任意の semantic re-ranking を組み合わせる。

Context

質問を分解し、関連抜粋をトークン予算内へまとめた context pack を生成できる。

Cache

質問と wiki version で回答をキャッシュし、取り込みや状態変更時に無効化する。

5 — 品質とライフサイクル
ページ状態
draft→active→stale/contradicted→archived

lint と人のレビューで状態を動かし、変更履歴を immutable event log に残す。archived ページへの wikilink は関連ページから連鎖的に除去される。

信頼の仕組み
  • 各 claim に元ファイルと行を示す citation を付ける
  • 矛盾、孤立、切れた citation を構造 lint で検出する
  • 別の LLM pass で誇張や根拠の弱い主張を確認する
  • token と cost を job 単位で記録し、hard gate を設定できる
ネットワーク境界: server の既定 bind は 127.0.0.1。外部 interface へ bind する場合、組み込み認証がないため別途保護が必要になる。

7 — 始め方
最短の確認経路

Python package を入れ、LLM 呼び出しなしで閲覧できる History of Computing demo を作る。server を起動したら CLI、Obsidian、Web UI が同じ wiki を参照する。

pip install synthadoc
synthadoc install history-of-computing --target ~/wikis --demo
synthadoc use history-of-computing
synthadoc serve
自分の wiki

domain を指定して空の wiki を作り、provider を設定してから資料を取り込む。

synthadoc install market-notes \
  --target ~/wikis \
  --domain "Market research"
synthadoc use market-notes
synthadoc serve
synthadoc ingest report.pdf
synthadoc query "What changed?"

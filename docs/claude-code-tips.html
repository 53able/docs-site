1 — 概要
何か

claude-code-tips は、YK（ykdojo） による Claude Code 運用 Tips 集。README に Tip 0–45 が並び、同じリポジトリが Claude Code プラグイン dx としても配布される。

対象読者は日々 Claude Code を使う開発者。スラッシュコマンド・コンテキスト管理・Git/gh 連携・隔離実行・Remote Control など、実務で効く手順が中心。

仕組みいつ載るか向き
CLAUDE.md 全会話の先頭に常時 プロジェクト説明・繰り返し指示。短いほどよい
Skills 関連時に自動、または /skill 手順付きの再利用ワークフロー
Slash Commands 明示的に起動 ユーザーがタイミングを制御したい操作
Plugins インストール単位 skills / hooks / MCP などをまとめて配布（例: dx）
注: Claude Code 2.1.3 以降、Skills と Slash Commands は統合方向。意図の違いは「誰が主に起動するか」。Skills は Claude 向け、Slash はユーザー向けとして設計されてきた。

6 — セットアップ
setup.sh（Tip 44）

対話式で複数推奨設定をまとめて入れるスクリプト。スキップ番号を指定できる。

bash <(curl -s https://raw.githubusercontent.com/ykdojo/claude-code-tips/main/scripts/setup.sh)
#区分内容
1InstallDX プラグイン
2–7~/.claude/settings.jsonstatus line / 自動更新オフ / MCP lazy-load / Read(~/.claude) / Read(//tmp/**) / attribution 無効
8–9shellaliases（c/ch/cs）と --fs → --fork-session
ステータス行だけ入れる場合
  1. scripts/context-bar.sh を ~/.claude/scripts/ にコピーして実行可能にする
  2. ~/.claude/settings.json の statusLine.command にパスを指定
  3. 必要ならスクリプト先頭の COLOR（orange / teal / …）を変更。プレビューは scripts/color-preview.sh

表示例: モデル名・cwd・ブランチ・未コミット数・origin 同期・コンテキスト使用率バー・直前メッセージ。

7 — コンテキスト運用
圧縮と分岐
01
新しい話題は新会話
Tip 5。長い履歴は性能を下げる。トピック切替で /clear か新規セッション。
02
handoff してから切る
Tip 8 / /dx:handoff。Goal・進捗・成功失敗・Next を HANDOFF.md に書いてパスだけ渡す。
03
fork / half-clone
Tip 21。/branch や --fork-session で分岐。長い会話は後半だけクローン。
04
85% で半クローン提案
任意 hook: scripts/check-context.sh を Stop hook に。auto-compact はオフ推奨。
half-clone の権限: 会話履歴読み取りのため、グローバル settings に Read(~/.claude) を許可するとプロンプトが減る（Tip 21）。 
{
  "permissions": {
    "allow": ["Read(~/.claude)"]
  }
}
8 — 隔離と並列
実行モードの段階
レベル手段用途
default 手動承認 / auto mode（Shift+Tab） 通常開発。auto は機械的承認疲れを減らす（Tip 31）
container --dangerously-skip-permissions in sandbox 長時間研究・実験。safeclaw 等（Tip 19）
dedicated Mac computer use 可能な専用機 claude-controls-mac（Tip 19）
remote /remote-control / claude remote-control スマホから操作。隔離環境以外は慎重に（Tip 32）

並列作業

  • タブ cascade（Tip 13）: 新規タスクは右にタブ追加。左→右で古い順に巡回。同時は 3–4 本程度。
  • worktree（Tip 14）: ブランチごとに別ディレクトリ。同じプロジェクトの衝突を避ける。
  • tmux ワーカー（Tip 19）: 外側 Claude が tmux send-keys / capture-pane でコンテナ内 Claude を操縦。

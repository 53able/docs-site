Public APIs

Community-maintained list of public APIs for apps and learning — curated README catalog.

1 — 概要
これは何か

public-apis は、ドメイン別に整理された無料／公開 API のカタログを README.md に載せてメンテするコミュニティリポジトリ。プロダクトへの組み込みや学習用の入口として、リンク・認証要否・HTTPS・CORS を一覧できる。スポンサーである APILayer の製品紹介セクションが先頭にあり、その後にカテゴリ別の巨大なマークダウンテーブルが続く。

3 — Auth / HTTPS / CORS
検証スクリプトと一致する許容値

scripts/validate/format.py は行をパースし、Auth・HTTPS・CORS が規定集合に入っているか、説明長やアルファベット順などをチェックする。ここを外すと CI のフォーマット検証で落ちる。

フィールド許容値
AuthOAuth, apiKey, X-Mashape-Key, No, User-Agent
HTTPSYes, No
CORSYes, No, Unknown
CORS: ブラウザから直接叩く用途では CORS 未対応だと使いづらい。CONTRIBUTING でも、適切な CORS 設定がない API はサーバーサイド向けになる旨が注意書きされている。
4 — コントリビューション
PR 前に押さえるルール
  • マーケティング PR は不可。 有料製品の宣伝目的や、デバイス購入が前提の「無料 API」は避ける。
  • 無料利用の実態: フル無料かフリーティアがあること。コミュニティ向けの一覧であり営業ツールではない（CONTRIBUTING 冒頭）。
  • 1 PR 1 リンク。 既存 API の「更新版」で差し替えない（旧版は非推奨扱いにしない）。
  • カテゴリ内アルファベット順（検証でカテゴリごとにタイトル昇順を確認）。
  • 列のパディング: セルは左右 1 スペース（| Foo | のように）。
  • 命名: TLD を名前に含めない、名前末尾を API にしない。
  • ブランチ: upstream への PR は master ブランチをターゲットにする（CONTRIBUTING 記載）。
  • コミット: 開く前にスカッシュ。レビュー修正後もスカッシュを求められる。
format.py が見ること（抜粋）
  • カテゴリごとに最低 3 エントリ（min_entries_per_category）。
  • 説明の先頭は大文字、末尾に句読点を付けない。
  • タイトルは [Title](url) 形式必須。
5 — 検証パイプライン
ローカルと CI

scripts/README.md に従い、リポジトリルートで Python 依存を入れたうえで検証コマンドを実行する。PR ではリンクチェックを含むビルドが走り、失敗時はログを見て README を直す流れになる。

format — README の表形式
python -m pip install -r scripts/requirements.txt
python scripts/validate/format.py README.md
links — リンク到達性（時間がかかる場合あり）
python scripts/validate/links.py README.md
python scripts/validate/links.py README.md -odlc

最後の -odlc（重複リンクのみチェック）は、全リンクの生き確認を省略したいとき用。

Zero（vercel-labs/zero）

エージェント向けシステム言語 · 明示的エフェクト · 構造化コンパイラ出力 · Apache-2.0

1 — 概要
位置づけ

Zero は、エージェントが扱う小さなネイティブツール向けのシステムプログラミング言語。明示的なエフェクト、予測可能なメモリ、構造化されたコンパイラ出力を重視する（README の説明に基づく）。言語・標準ライブラリ・コンパイラはまだ実験段階で、安定版ではない。

.0ソース拡張子
CLIcheck / run / build
JSON機械可読な診断・グラフ

公式サイト zerolang.ai からインストーラを取得し、zero CLI でチェック・実行・ビルドする。人間向けの長文ドキュメントは同梱の docs-site/（Next.js）が担う。

2 — 設計思想（エージェントが読み取りやすくするため）
明示性

グローバルな隠れ I/O を避け、ランタイムが渡す World 能力オブジェクト経由で出力などを行う。失敗可能な操作は check で伝播し、関数は raises で失敗しうることを宣言する。

World 能力 → check / raises → 構造化診断 → zero explain / fix
実験中: 言語仕様・CLI・標準ライブラリは変わりうる。本番依存の前提にしない（README）。

ターゲットと成果物

ネイティブ実行ファイル、オブジェクト、WASM などを zero build --emit で直接生成する。生成 C バックエンドにフォールバックする経路は削除済み（zero-builds スキル）。クロスビルド用の --target（例: linux-musl-x64、wasm32-wasi）を指定できる。

3 — 言語の骨格
最小プログラム

エントリは pub fun main(world: World) -> Void raises。トップレベルには use、const、shape、enum、choice、fun、test などがある（zero-language スキル）。

pub fun main(world: World) -> Void raises {
    check world.out.write("hello from zero\n")
}

制御と型

  • 束縛は原則 let、変更が必要なときだけ let mut
  • 条件は Bool のみ（整数や文字列の真偽値は使わない）
  • 構造体は shape、列挙は enum、結果の和型は choice
  • match は網羅的。フォールバックは _ 臂

パッケージ

zero.json で実行可能パッケージやライブラリを定義する。zero new cli|lib|package で雛形を生成し、use helpers のようなパッケージローカル import は src/ から解決する（zero-packages スキル）。

{
  "package": { "name": "hello", "version": "0.1.0" },
  "targets": { "cli": { "kind": "exe", "main": "src/main.0" } }
}

5 — CLI（開発ループ）
よく使うコマンド
コマンド用途
zero check <file|package>型・診断の検査（--json 可）
zero run <file|package>ホスト上で実行（-- 以降はプログラム引数）
zero build --emit exe|obj|wasm成果物を直接生成
zero graph --json依存・モジュール構造の可視化用データ
zero size --jsonサイズプロファイル
zero routes --jsonWeb ターゲットのルート一覧
zero explain <code>診断コードの説明
zero fix --plan --json修復プランの取得
zero doctor --json環境・ツールチェーンの健全性

インストールと最初の実行

curl -fsSL https://zerolang.ai/install.sh | bash
export PATH="$HOME/.zero/bin:$PATH"
zero --version
zero check examples/hello.0
zero run examples/add.0

examples/add.0 を実行すると、README 記載のとおり math works が標準出力に出る。

6 — バージョン一致スキル
エージェント向け配布

skills/zero/SKILL.md は発見用の薄いスタブ。実ワークフローは、インストール済みコンパイラが返す版一致コンテンツを使う（同梱スキル）。

zero skills list
zero skills get zero
zero skills get zero --full

複数バイナリがある場合は、実際にプロジェクトを走らせる zero と同じパスで skills を呼ぶ。リポジトリ checkout 上の作業では bin/zero を優先する（skills/zero/SKILL.md）。

7 — リポジトリ内の検証
npm スクリプト

ルート package.json から、サンドボックス付きの conformance・ネイティブテスト・ドキュメントサイトテストなどを実行できる。

npm run docs:dev
npm run docs:test
npm run conformance
npm run native:test
npm run command-contracts
npm run bench

ドキュメントサイトは pnpm --prefix docs-site dev でローカル起動。Playground 用 WASM は docs:wasm スクリプトでビルドされる。

学習順: docs-site/articles/getting-started.md → learn-zero.md → language-reference.md → examples/README.md（README の Learn Zero 節）。

モノレポ基盤

コードの読み書き、コマンド実行、Git 操作に加え、SQLite FTS5 によるプロジェクトメモリとチェックポイントでセッションをまたいだ文脈を保持する。OpenCode をフォークし、メモリ・サブエージェント・compose ワークフロー・自己改善コマンドを追加している。

概要

MiMoCode はターミナルで動く AI コーディングアシスタント。初回起動時に MiMo Auto（期間限定無料）や Xiaomi MiMo Platform、Claude Code からの認証インポート、カスタム OpenAI 互換 API などを選べる。

  • 公式: mimo.xiaomi.com/en/mimocode
  • ライセンス: MIT（利用は Use Restrictions も参照）
  • ランタイム: Bun 1.3+、Turbo ワークスペース、SolidJS / OpenTUI ベースの TUI

エージェントと compose

Tab でプライマリエージェントを切り替える。サブエージェントはセッション文脈を共有しつつ並列実行・キャンセル・バックグラウンド実行に対応する。

エージェント役割
buildデフォルト。開発向けのフルツール権限
plan読み取り専用。調査と設計
compose仕様駆動開発とスキル連携のオーケストレーション

compose モードに含まれるスキル例

plan / execute
仕様から実装計画と実行フェーズへ接続
review / verify
コードレビューと検証ループ
tdd / debug
テスト駆動とデバッグ手順の組み込み
merge / worktree
ブランチ統合と worktree 連携

自律停止の判定

/goal でセッションの停止条件を設定すると、エージェントが終了しようとしたタイミングで独立した judge モデルが条件達成を評価する。楽観的な早期終了を抑える。

メモリとコンテキスト管理

永続メモリ
ファイル用途
MEMORY.mdプロジェクト知識・ルール・アーキテクチャ決定
checkpoint.mdcheckpoint-writer サブエージェントが維持する状態スナップショット
notes.mdエージェント用の一時メモ
tasks/<id>/progress.mdタスク単位の進捗ログ

コンテキスト再構築

  • モデルのコンテキスト上限に応じてチェックポイント保存タイミングを自動判断
  • 上限接近時は最新チェックポイント・プロジェクトメモリ・タスク進捗・直近メッセージから文脈を再構築
  • トークン予算で注入量を制御し、重要度順にランク付け

タスクツリー

T1、T1.1、T1.2 のような木構造タスク ID をチェックポイントと連携。セッション再開後も進捗が復元される。

自己改善コマンド

/dream
直近セッションから永続知識を抽出し、古いエントリを整理
/distill
繰り返し手順をスキル・サブエージェント・コマンドへ昇格
/voice
TenVAD + MiMo ASR によるストリーミング音声入力（MiMo ログイン時）
Max Mode
experimental.maxMode で best-of-N 推論と judge 選択

設定

プロジェクト直下の .mimocode/mimocode.jsonc、またはグローバル ~/.config/mimocode/mimocode.json で設定する。

カテゴリ主な項目
Providerモデル選択、カスタムエンドポイント
Agent権限、カスタムエージェント定義
Memory / Checkpoint注入予算、チェックポイント動作
MCP外部ツールサーバー接続
TUIキーバインド、テーマ（.mimocode/themes/）
ExperimentalmaxMode など

プロジェクト固有のエージェント定義・スキル・コマンドは .mimocode/agent/、.mimocode/skills/、.mimocode/command/ に Markdown で置ける。

OpenCode との関係

MiMoCode は OpenCode のフォーク。複数プロバイダ、TUI、LSP、MCP、プラグインなど OpenCode の中核機能を維持したうえで、次を追加している。

  • SQLite FTS5 によるセッション横断メモリ
  • チェックポイント駆動のコンテキスト再構築と予算付き注入
  • サブエージェントのライフサイクル管理
  • /goal による停止条件と judge 評価
  • compose ワークフローとバンドル済みスキル群
  • /dream・/distill による自己改善

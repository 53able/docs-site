Meta Wearables DAT iOS

facebook/meta-wearables-dat-ios — Device Access Toolkit for iOS

1 — 概要
何か

Meta Wearables Device Access Toolkit (DAT) は、Meta の AI グラス（Ray-Ban Meta など）と iOS アプリを接続し、映像ストリーミングや写真撮影などのハンズフリー体験を組み込むための SDK だ。Swift Package Manager で配布され、開発者プレビュー段階にある。

0.7.0SDK バージョン
iOS 16+デプロイターゲット
Xcode 15+ビルド要件
9プラグインスキル

アプリは Meta AI コンパニオンアプリ経由でグラスとペアリングし、MWDATCore / MWDATCamera などのモジュールでセッション管理やカメラ機能にアクセスする。物理デバイスが無い開発では MockDeviceKit でエミュレートできる。

開発者プレビュー: SDK とドキュメントは公開されているが、本番リリース前の段階。Wearables Developer Center で組織・リリースチャネルを登録し、テストユーザーと共有する運用が想定されている。

3 — SDK 導入
Swift Package Manager
  1. Xcode で File > Add Package Dependencies...
  2. URL に https://github.com/facebook/meta-wearables-dat-ios を指定
  3. タグからバージョン（例: 0.7.0）を選択
  4. ターゲットに MWDATCore と MWDATCamera を追加

Info.plist の必須設定

getting-started スキルが示す最低限の構成は次のとおり。

  • CFBundleURLTypes — Meta AI からのコールバック用 URL スキーム
  • クエリスキームに fb-viewapp を許可
  • UISupportedExternalAccessoryProtocols に com.meta.ar.wearable
  • UIBackgroundModes — bluetooth-peripheral / external-accessory
  • NSBluetoothAlwaysUsageDescription
  • MWDAT 辞書 — AppLinkURLScheme / MetaAppID（開発時は 0 と Developer Mode）

アプリ起動時に Wearables.configure() を一度呼び出す。詳細は Wearables Developer Center の API リファレンス（iOS Swift DAT 0.7）を参照。

アナリティクスオプトアウト

Info.plist の MWDAT > Analytics > OptOut を YES にするとデータ収集を無効化できる。キーが無いか NO の場合は収集が有効（オプトイン）となる。

6 — 接続フロー（概略）
ランタイム

エンドユーザー視点での典型的な流れ。実装詳細は session-lifecycle / permissions-registration スキルを参照。

flowchart TD
  A[iOS アプリ起動] --> B[Wearables.configure]
  B --> C[Meta AI 経由ペアリング]
  C --> D{セッション確立}
  D -->|成功| E[カメラ / ディスプレイ API]
  D -->|未接続| F[MockDeviceKit または再試行]
  E --> G[ストリーム / 写真 / UI]
  C --> H[権限・App 登録]
  H --> D
前提: テスト端末に Meta AI アプリが入っていること。実機グラス利用時は Meta AI 内で Developer Mode（Settings > Your glasses）を有効にする。

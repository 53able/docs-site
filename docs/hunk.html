Hunk

review-first terminal diff viewer — OpenTUI × Pierre diffs

1 — 概要
何をするか

Hunk は、エージェントが書いた変更一式をレビュー優先の UI で読むためのターミナル diff ビューア。UI 基盤は OpenTUI、差分表示は Pierre diffs を利用する。

Multi-fileサイドバー付きストリーム
InlineAI / エージェント注釈
Layoutssplit / stack / 自動
Watchファイル・Git 追従
  • キーボード・マウス・pager・Git difftool との連携
  • 未追跡ファイルを含むワーキングツリーのレビュー
  • 任意コミットやパッチ stdin の閲覧
2 — インストールと要件
グローバル CLI 
npm i -g hunkdiff
  • Node.js 18 以上
  • macOS または Linux
  • 多くのワークフローで Git を推奨
hunk           # ヘルプ
hunk --version # バージョン表示
3 — Git 作業ツリーと raw ファイル
Git まわり

サブコマンドは Git の diff 系に寄せており、結果をプレーン text ではなくレビュー UI に載せる。

hunk diff                      # 現在の変更（未追跡含む）
hunk diff --watch              # ワーキングツリー変更で自動再読込
hunk show                      # 最新コミット
hunk show HEAD~1               # 過去コミット
2 ファイル比較とパッチ 
hunk diff before.ts after.ts
hunk diff before.ts after.ts --watch
git diff --no-color | hunk patch -
4 — エージェント連携
ライブセッション
  1. 別ターミナルで hunk diff または hunk show を起動する。
  2. リポジトリ付属の skills/hunk-review/SKILL.md をエージェントに読ませる。
  3. ライブセッション向けにスキルを実行させる。
注: hunk skill path はリリース 0.10.0 以降で整備予定の記述がある。それまでは上記 SKILL パスを直接指定する。

詳しい手順は上流の docs/agent-workflows.md を参照。

6 — 設定ファイル
永続化の場所
  • ~/.config/hunk/config.toml
  • リポジトリ直下の .hunk/config.toml

例（TOML）

theme = "graphite"   # graphite, midnight, paper, ember
mode = "auto"        # auto, split, stack
exclude_untracked = false
line_numbers = true
wrap_lines = false
agent_notes = false
7 — リポジトリ構成（処理の層）
ざっくりデータフロー

CLI エントリ（src/core/cli.ts ほか）が Git / ファイル入力を解釈し、セッションと broker（packages/session-broker*）を介して TUI（src/ui）へ差分とメタデータを流すイメージ。

flowchart TD A[CLI hunk / hunk show] --> B[src/core Git・パス解決] B --> C[src/session プロトコル] C --> D[session-broker デーモン] D --> E[src/ui OpenTUI + Pierre] C --> E

1 — 概要
位置づけ

Rowboat は、メールや会議メモと接続し、長寿命のナレッジグラフをローカルに蓄え、その文脈でミート準備・メール下書き・スライド生成などを支援する ローカルファーストの AI コワーカー（Electron デスクトップアプリが中心）。データは プレーン Markdown とバックリンクで保持され、Obsidian 互換の保管形式を採用する。

Local-firstデータは自機の Vault
Markdown透明な作業記憶
MCP外部ツール接続
BYOMモデルは差し替え可
検索型との違い: 都度トランスクリプトを探すのではなく、関係・決定・コミットメントがグラフとして積み上がり、ユーザーがノートを直接編集できる点が README で強調されている。
2 — データの流れ
取り込み → グラフ → 行動

連携ソースから同期された情報が Vault 内のノート・リンクとして蓄積され、アシスタントがブリーフ作成や資料生成に利用する、というパイプラインを概念的に示す。

3 — モノレポ構成
リポジトリ内の主なアプリ

開発者向けコンテキスト（CLAUDE.md）に基づく。デスクトップは apps/x の Electron ネストワークスペースが現行の焦点。

パス役割

Electron 側のビルド順（依存関係）

shared → core → preload / renderer → main。ワークスペースのシンボリックリンク対策でメインは esbuild 単一バンドルにまとめる、と CLAUDE.md に記載がある。

4 — メモリとグラフ
Vault と Live notes
  • Obsidian 互換: バックリンク付き Markdown の Vault が「透明な作業記憶」として機能する。
  • Live notes: ノート上で @rowboat を指定すると、トピックや人物などを横断して自動更新されるライブノートを作れる（README）。
  • 編集可能性: モデル内部に隠れたコンテキストではなく、ユーザーがファイルとして検査・編集できる。
モデル設定（開発者向け）

単一プロバイダの例として ~/.rowboat/config/models.json に provider / model を置く形式が CLAUDE.md に示されている。

6 — 拡張
MCP とモデル
  • MCP: Model Context Protocol で検索・DB・CRM・Slack / GitHub など外部ツールを接続可能（README の例）。
  • モデル: Ollama / LM Studio などローカル、またはホスト型 API（独自キー）に対応。Vault はローカルに残る。
7 — 任意設定
API キー（README）

音声入力（Deepgram）、音声出力（ElevenLabs）、Web 検索（Exa）、Composio などは任意。~/.rowboat/config/ 配下の JSON に apiKey を置く形式が共通。

{
  "apiKey": "<key>"
}

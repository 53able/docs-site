Hermes Workspace

Your AI agent's command center — v2.0.0 / MIT / Node 22+

1 — 概要
位置づけ

Hermes Workspace は、NousResearch/hermes-agent 向けのデスクトップ型 Web ワークスペース。チャット、ファイル、メモリ、スキル、ターミナル、Conductor によるオーケストレーションを 1 つの UI に集約する（単なるチャットラッパーではなく、完全なワークスペースを目指す設計）。v2 では zero-fork — 上流の hermes-agent を Nous 公式インストーラで入れ、パッチなしで接続する。

3 — アーキテクチャ
接続の概念

ブラウザ上の Hermes Workspace は、環境変数 HERMES_API_URL で指した バックエンドに HTTP で接続する。Hermes ゲートウェイが有効で API サーバーが立ち上がっていると、セッション・メモリ・スキル・ジョブなどの 拡張 API が利用可能になる。純粋な OpenAI 互換（チャット＋モデル一覧）だけに繋いだ場合は、チャットは動くが拡張パネルは制限付き（README が「ポータブル」として説明する挙動）。

4 — 起動と接続
例: 既存ゲートウェイに取り付ける

ゲートウェイが 127.0.0.1:8642 で待ち受けている前提の最小手順（README より）。

git clone https://github.com/outsourc-e/hermes-workspace.git
cd hermes-workspace
pnpm install
cp .env.example .env
echo 'HERMES_API_URL=http://127.0.0.1:8642' >> .env
# ゲートウェイに API_SERVER_KEY がある場合は HERMES_API_TOKEN に同じ値
pnpm dev   # http://localhost:3000

エージェント側の前提: ゲートウェイが到達可能なアドレスに束縛され、API_SERVER_ENABLED=true（~/.hermes/.env 等）。API_SERVER_KEY を使う場合はワークスペース側のトークンと一致させる。検証: curl http://127.0.0.1:8642/health。

1 行インストール

README の install.sh は hermes-agent（PyPI）の導入、リポジトリ取得、.env、依存インストールまで一括で行い、その後 hermes gateway run と pnpm dev の 2 ターミナル運用を想定している。

6 — Docker
構成

docker compose up は hermes-agent イメージ（ポート 8642）と hermes-workspace イメージ（ポート 3000）を併起する。永続は hermes-data ボリューム。本番想定のイメージ例: ghcr.io/outsourc-e/hermes-workspace:latest（linux/amd64 / arm64）。

注意: ゲートウェイにモデルが到達可能であること。API キーまたはローカル推論のどちらかが必須（README）。

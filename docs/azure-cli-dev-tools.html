azure-cli-dev-tools

azdev — Azure CLI コマンドモジュール・拡張機能の開発支援ツールキット

1 — 概要
何か

azdev は、Azure CLI のコマンドモジュールや拡張機能を開発・テスト・公開するためのオールインワン CLI ツール。スタイルチェック・静的解析・テスト実行・バージョン管理・拡張機能ビルドをひとつのコマンド体系にまとめている。

30+コマンド数
PythonPyPI: azdev
knackCLI フレームワーク
MITライセンス

開発者は azdev style → azdev linter → azdev test という標準ループで品質を担保し、azdev extension publish で拡張機能を Azure CLI Extension Index に公開できる。また azdev command-change サブグループで API の破壊的変更を検出できる。

構成

リポジトリは azdev/ 本体と azure-cli-diff-tool/ サブパッケージで構成される。azdev/operations/ 配下に各コマンドグループの実装が格納され、azdev/commands.py で knack の CommandGroup を使ってコマンドテーブルに登録している。

2 — セットアップ
インストール手順
01

Python 仮想環境を作成

Python 3.8+ を使い、python -m venv env で venv を作成。source env/bin/activate（Linux/macOS）または env\Scripts\activate.bat（Windows）で有効化。

02

リポジトリをフォーク・クローン

azure-cli（および必要なら azure-cli-extensions）をフォークし、git remote add upstream でアップストリームを登録する。

03

azdev をインストール

pip install azdev で PyPI から取得。Linux では事前に gcc python3-dev（apt）または gcc python3-devel（rpm）が必要。

04

azdev setup を実行

azdev setup で対話的セットアップ。非対話的には azdev setup --cli /path/to/azure-cli --repo /path/to/extensions。

コマンド・モジュールの雛形作成

CLI モジュールや拡張機能の雛形は以下で生成できる。aaz-dev-tools を使うと REST API スペックからコマンドを自動生成できる。

# 拡張機能の雛形を作成
azdev extension create <extension-name>

# CLI モジュールの雛形を作成
azdev cli create <module-name>

# 拡張機能をソースからインストール（開発時）
azdev extension add <extension-name>

4 — 開発フロー
標準開発サイクル

コードを変更したら、以下の順番でチェックを回す。CI パイプライン（azure-pipelines.yml）でも同じ順序で実行される。

azdev style → azdev linter → azdev test → azdev cmdcov → azdev verify license

テストモード

モード挙動用途
once（デフォルト） 記録ファイルなし → ライブ実行して YAML 生成。あり → プレイバック 初回記録 & 通常の CI
--live 常に Azure に接続してライブ実行、記録を再生成 API 仕様変更後の再記録
playback のみ 記録済み YAML をリプレイ。Azure 接続不要 オフライン CI・高速検証
破壊的変更の検出フロー

command-change サブグループを使って API 変更を追跡する。azure-cli-diff-tool（azure_cli_diff_tool/meta_change_detect.py）が差分解析のコアを担う。

meta-export (before) → meta-export (after) → meta-diff → generate-breaking-change-report
記録ファイルの場所: テスト記録は YAML 形式で保存される。--live を付けずに実行した場合、既存の YAML が見つかれば常にプレイバックモードになる。再記録するには --live フラグが必要。
5 — 拡張機能管理
拡張機能の開発から公開まで
extension create → extension add → style / linter / test → extension build → extension publish

主要コマンド詳細

コマンド説明
azdev extension create <name> 拡張機能の雛形ディレクトリを生成。azext_<name>/ 以下に __init__.py・commands.py・_params.py 等を自動作成
azdev extension add <name> ソースコードから拡張機能を開発用インストール。az <cmd> で即テスト可能な状態になる
azdev extension build <name> wheel ファイルをビルド。dist/ 以下に .whl が生成される
azdev extension publish <name> Azure Blob にアップロードし、azure-cli-extensions リポジトリの index.json を更新。PR 作成まで自動化できる
azdev extension cal-next-version <name> semver ルールに基づき次バージョン番号を計算（command-change meta-diff の結果を利用）
azdev extension repo add <path> 拡張機能が含まれる追加リポジトリを登録。複数リポジトリ横断での開発に使う
最新インデックスの検証: azdev latest-index verify は生成済みインデックスとチェックイン済みファイルを比較し、差異があれば非ゼロ終了する。CI に組み込むと PR で自動検証できる。

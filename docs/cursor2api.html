Cursor2API

Cursor ドキュメントの無料 AI を Anthropic Messages API と OpenAI Chat Completions API にプロキシ。Claude Code と Cursor IDE で利用可能。

1 — 概要
何をするか

Cursor2API は、Cursor のドキュメントページが提供する無料の AI 対話インターフェースを、Anthropic Messages API および OpenAI Chat Completions API 形式に変換するプロキシサーバーです。Claude Code（Anthropic CLI）や Cursor IDE、ChatBox / LobeChat などの OpenAI 互換クライアントから、Cursor のバックエンドを経由して Claude と対話できます。

/v1/messagesAnthropic
/v1/chat/completionsOpenAI
/v1/responsesCursor IDE
Node + Express実装
2 — アーキテクチャ
データフロー

クライアント（Claude Code / Cursor IDE / OpenAI 互換クライアント）が cursor2api にリクエストを送り、プロキシが Cursor の /api/chat に変換して転送。レスポンスを Anthropic / OpenAI 形式に戻して返します。

┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│ Claude Code  │────▶│              │────▶│              │
│ (Anthropic)  │     │  cursor2api  │     │  Cursor API  │
│              │◀────│  (代理+変換)  │◀────│  /api/chat   │
└─────────────┘     └──────────────┘     └──────────────┘
       ▲                    ▲
       │                    │
┌──────┴──────┐     ┌──────┴──────┐
│  Cursor IDE  │     │ OpenAI 互換  │
│ /v1/responses│     │ /v1/chat/   │
│ + Agent 模式  │     │ completions │
└─────────────┘     └─────────────┘

プロキシ層では認知リフレーミング（「API ドキュメントを書いている」とモデルに伝え、ツール呼び出しを JSON 例として出力させる）や、ツールパラメータの自動修正（file_path → path など）、拒否テキストの検出・置換により、Cursor の制限付きドキュメント助手を実質フルツール対応として振る舞わせます。

4 — 主な機能
一覧
  • Anthropic Messages API 互換 — /v1/messages のストリーミング・非ストリーミング。Claude Code と直接接続。
  • OpenAI Chat Completions 互換 — /v1/chat/completions。ChatBox / LobeChat などで利用可能。
  • Cursor IDE Agent 対応 — /v1/responses、フラットツール形式、インクリメンタルストリーミングのツール呼び出し。
  • ツールパラメータ自動修正 — file_path → path、スマート引用符置換、曖昧マッチ修復。
  • 視覚の降級 — ローカル OCR（Tesseract）または外部視覚 API（Gemini 等）で画像をテキスト化してから送信。
  • 認知リフレーミング — 「API ドキュメントを書いている」と伝え、ツール呼び出しを JSON 例として出力させることで Cursor の制限を回避。
  • 拒否の多層防御 — コンテキスト洗浄・XML 分離・出力インターセプト・レスポンス洗浄で Cursor の「Documentation Assistant」拒否を抑制。
  • Thinking 対応 — <thinking> タグの抽出、Anthropic / OpenAI 両パスで出力。3 行・120 語制限で output 予算を節約。
  • 截断の段階的復旧 — Tier 1（Bash/分割誘導）→ Tier 2（強制分割）→ Tier 3–4（従来の続き書き）で長いツール応答の切断を復旧。
5 — 使い方
セットアップ

config.yaml で cursor_model（例: anthropic/claude-sonnet-4.6）、fingerprint.user_agent、vision.enabled / vision.mode（ocr または api）を設定。

起動

npm install
npm run dev

Claude Code で使う

export ANTHROPIC_BASE_URL=http://localhost:3010
claude

Cursor IDE で使う

設定で OPENAI_BASE_URL=http://localhost:3010/v1 を指定し、/v1/models で表示される Claude モデル名を選ぶ（GPT 名より Claude 名を推奨）。

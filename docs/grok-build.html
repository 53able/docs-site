Grok Build

xai-org/grok-build — SpaceXAI のターミナル AI コーディングエージェント（grok）

xai-org/grok-build は SpaceXAI のフルスクリーン TUI コーディングエージェント。コードベース理解・ファイル編集・シェル実行・Web 検索・長時間タスクを、対話 TUI / headless（CI・スクリプト）/ ACP（エディタ埋め込み）で扱う。製品サイトは x.ai/cli。公開ツリーは社内モノレポから定期同期。

実行モード
3 つの入口
Interactive TUI
既定起動。スクロールバック・プロンプト・モーダル。初回はブラウザ認証。
Headless
grok -p "…"（または --prompt-json / --prompt-file）。stdout に結果、終了コードで完了。出力は plain / json / streaming-json。
Agent mode (ACP)
grok agent stdio。JSON-RPC でセッション・プロンプト・ツール可視化・権限承認。Zed / Neovim 等との統合。
grok --version
grok -p "Summarize this repo"
grok agent --model grok-build stdio

Agent 定義

xai-grok-agent は Markdown + YAML frontmatter のエージェント定義を .grok/agents/（プロジェクト）または ~/.grok/agents/（ユーザー）から読み、ツール集合・システムプロンプト・compaction / model 設定を束ねる。

インストール
リリースバイナリ 
curl -fsSL https://x.ai/cli/install.sh | bash   # macOS / Linux / Git Bash
irm https://x.ai/cli/install.ps1 | iex          # Windows PowerShell
grok --version

changelog: x.ai/build/changelog。特定版は install スクリプトに版番号を渡すか、GROK_VERSION（PowerShell）を指定。

ソースビルド
要件とコマンド
  • Rust — rust-toolchain.toml（channel 1.92.0）。rustup が自動導入。
  • protoc — bin/protoc（dotslash）または PATH / $PROTOC。
  • ビルドホストは macOS / Linux。Windows はこのツリーからは best-effort。
cargo run -p xai-grok-pager-bin              # build + launch TUI
cargo build -p xai-grok-pager-bin --release  # target/release/xai-grok-pager
cargo check -p xai-grok-pager-bin
cargo test -p xai-grok-config                # クレート単位
cargo clippy -p <crate>
cargo fmt --all

フルワークスペースビルドは遅いので、常に -p <crate> を付ける（README）。

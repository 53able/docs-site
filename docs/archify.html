Archify

agent skill for architecture · workflow · sequence · dataflow · lifecycle — tt-a1i/archify v2.11

1 — 概要
何をするか

Archify は Claude、Codex CLI、opencode 向けの Agent Skill。システムの説明や Mermaid 断片を受け取り、自己完結の HTML 技術図を返す。出力はブラウザで開き、ダーク / ライト切替（T）とエクスポート（E）が使える。リポジトリは tt-a1i/archify。

5 modes図タイプ
JSON IR型付き中間表現
≤4×ラスタ解像度
0 deps検証もスタンドアロン
  • 一般的な描画エディタや Mermaid テーマではなく、ストーリーに合わせた配置判断をエージェントが行う
  • v2.11 で 5 スキーマをコミット済みスタンドアロンバリデータにコンパイル。インストール後は npm install なしでスキーマ契約を強制できる
  • postgres / aws.lambda / github-actions などのセマンティックラベルで視覚カテゴリを決める
  • 生成 HTML は依存ゼロ。dual-theme SVG は埋め込み先の prefers-color-scheme に追従

3 — 図の種類
質問に合わせたモード選択

1 枚でリポジトリ全体を説明しようとせず、見たい観点を 1 つ選ぶ。トリガー語は SKILL.md に整理されている。

architecture

コンポーネント・クラウド資源・境界。主経路と trust boundary。例: examples/web-app.html、examples/archify-repo.html。

workflow

CI/CD・承認・ツール呼び出し・ランブック。スイムレーンと happy path。例外レーンは variant: "exception"。

sequence

API 呼び出し連鎖・キャッシュフォールバック・非同期トレース。参加者・メッセージ・activation。

dataflow

パイプライン・系譜・PII 境界。ステージ横断の flow ラベルと classification。

lifecycle

状態機械・待機・リトライ・終端。レーン id は意味付き（必須 main、終端は terminal）。他レーンは中央イベント帯を共有する。

Mermaid 入力

貼られた Mermaid を機械パースして描画するのではなく、構造だけ読んで archify スタイルで置き直す。flowchart → workflow（またはコンポーネントなら architecture）、sequenceDiagram → sequence、stateDiagram → lifecycle。スタイル指定は捨て、トポロジと意味だけ残す。

4 — 処理の流れ
JSON IR → 検証 → レンダ → 検査

5 モードとも同じループ。レンダラを直接いじらず、JSON を直して再実行する。

flowchart TD
  A["Generate JSON IR"] --> B["Validate schema"]
  B --> C["Render HTML/SVG"]
  C --> D["Check layout & artifact"]
  D -->|fail| E["Fix JSON IR"]
  E --> B
  D -->|pass| F["Deliver self-contained HTML"]
01
スキーマと例を読む

schemas/<type>.schema.json と examples/*.<type>.json を先に見る。フィールド形状は推測しない。

02
JSON を書く

<name>.<type>.json。必須は schema_version（const 1）、diagram_type、meta.title、モード固有の配列。

03
render / validate

node bin/archify.mjs render <type> … と validate … --json。エラーは JSON path 付きで出る。

04
check HTML

node bin/archify.mjs check <out>.html。非有限 SVG、対角線矢印、凡例越えなどを検出する。

手置きフォールバック: Node が使えないときは assets/template.html に SVG を手配置。色はインラインではなく CSS クラス（c-backend など）必須。テーマ切替が壊れない。スキーマ違反は path 付きで非ゼロ終了し、重なりや凡例越えはレンダラ側のレイアウト検査が担当する。
5 — レイアウト原則
空間ナラティブ

可読性は「全依存を矢印で描くこと」ではなく、1 本の物語が辿れる配置から来る。

  1. 主経路は 1 本 — architecture は左→右、workflow はレーン→列。happy path が交差せず追えること。
  2. ラベル付きエッジは少なく — 境界越えや非自明な遷移だけ。隣接ステップは無ラベルでよい。
  3. 枝は短く上下へ — 権限・ストレージ・CI は主経路の最近傍から縦に繋ぐ。斜め横断は避ける。
  4. 詳細はカードへ — ポリシーや技術メモは summary cards。矢印を増やさない。
  5. モード適合 — プロセス系は workflow / sequence。コンポーネント ≤12 は architecture。エッジが 20+ なら削る。
architecture の配置
  • Free: pos: [x, y]（左上）、既定サイズ [120, 60]。
  • Grid: layout.mode: "grid" と row / col。固定セル計算で、dagre 自動配置ではない。
  • 境界: wraps に囲む id を列挙。renderer が padding 付きボックスを計算する。
  • 検査: node bin/archify.mjs inspect architecture … で HTML を開かずに矩形をダンプできる。
6 — CLI と出力
スキルフォルダでの操作 
cd archify
node bin/archify.mjs doctor
node bin/archify.mjs demo /tmp/archify-demo
node bin/archify.mjs render workflow examples/agent-tool-call.workflow.json /tmp/workflow.html
node bin/archify.mjs validate workflow examples/agent-tool-call.workflow.json --json
node bin/archify.mjs check /tmp/workflow.html
node bin/archify.mjs examples

生成 HTML の操作

操作内容
Theme（T）ダーク / ライト。localStorage に保存し prefers-color-scheme も尊重
Export（E）PNG をクリップボードへコピー、または PNG / JPEG / WebP / SVG をダウンロード
ラスタ最大 4× でネイティブ描画。巨大図は 3× / 2× に自動ステップダウン
SVG両テーマ変数 + @media (prefers-color-scheme)。README 埋め込み向け
URL パラメータ?theme=light|dark、?openExport=1
アニメーション: meta.animation: "trace" はデモ時のみ。省略時は静的。prefers-reduced-motion を尊重する。
7 — プロンプト例
スコープを絞る 
Analyze this repository, then use archify to create a high-level runtime architecture diagram.
Show 8–12 core components, one primary request or data path, external dependencies, and trust boundaries.
Put supporting detail in cards instead of adding more edges.

フロー特化

Use archify to draw this login flow: Browser -> Web App -> API -> JWT validation ->
Redis session lookup -> PostgreSQL fallback. Make the cache-miss path secondary.

データ系譜

Draw a data-flow diagram from Web and Mobile events through Consent Gate, Kafka,
Warehouse, and Feature Store to Dashboards and ML consumers. Mark the PII boundary.

チャットでの修正は JSON IR を残したまま「add Redis」「move auth to the left」のように局所変更する。

ロードマップ上の非目標

v2.11 は zero-install 検証と doctor / demo を追加したうえで、5 モードすべてが型付き JSON IR。今後の焦点はローカル編集の安定化とレイアウト診断。自動 Mermaid パース、汎用 auto-layout、ホスティング共有、WYSIWYG エディタは現時点の目標ではない（ROADMAP.md）。

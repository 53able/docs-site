1 — 概要
What is nanobot

nanobot は HKUDS が開発する超軽量パーソナル AI エージェント。OpenClaw にインスパイアされ、フルスタック代替比で 99% 少ないコード行数 でコアエージェント機能を実現している。

3 — クイックスタート
2 分でエージェント起動
1

インストール — pip / uv / source から選択:

# PyPI（安定版）
pip install nanobot-ai

# uv（推奨 — 高速）
uv tool install nanobot-ai

# ソースから（最新機能）
git clone https://github.com/HKUDS/nanobot.git
cd nanobot && pip install -e .
2

初期化 — config と workspace を生成:

nanobot onboard
# または対話型ウィザード
nanobot onboard --wizard
3

設定 — ~/.nanobot/config.json を編集:

{
  "providers": {
    "openrouter": {
      "apiKey": "sk-or-v1-xxx"
    }
  },
  "agents": {
    "defaults": {
      "model": "anthropic/claude-opus-4-5",
      "provider": "openrouter"
    }
  }
}
4

チャット — CLI で即座に対話:

# 対話モード
nanobot agent

# 1 回だけ
nanobot agent -m "README を要約して"

# ゲートウェイ（チャンネル接続）
nanobot gateway
API キーについて: グローバルユーザーには openrouter（全モデルアクセス）が推奨。環境変数 ${VAR_NAME} 参照もサポートしており、シークレットを config.json に直書きしなくて済む。

4 — チャンネル連携

6 — 高度な機能
コア機能群
メモリシステム（Dream）

3 層メモリ: SOUL.md（エージェント人格）, USER.md（ユーザー情報）, memory/MEMORY.md（長期記憶）。memory/history.jsonl に追記式で要約履歴を保存。

Dream は定期実行（またはチャット内 /dream コマンド）でメモリを統合。/dream-restore <sha> で過去状態への復元も可能。

MCP（Model Context Protocol）

Stdio（npx / uvx 等）と HTTP（SSE）の両方に対応。Claude Desktop / Cursor の config 形式と互換。

{
  "tools": {
    "mcpServers": {
      "filesystem": {
        "command": "npx",
        "args": ["-y",
          "@modelcontextprotocol/server-filesystem",
          "/path"]
      }
    }
  }
}

enabledTools で登録ツールを絞り込み可能。

Python SDK

CLI なしでライブラリとして利用できる。セッションキーで会話を独立管理。

from nanobot import Nanobot

bot = Nanobot.from_config()
result = await bot.run(
    "README を要約して",
    session_key="user-alice"
)
print(result.content)

AgentHook でツール呼び出し前後のライフサイクルフックも追加できる。

OpenAI 互換 API

nanobot serve で 127.0.0.1:8900 にエンドポイントを公開。既存の OpenAI クライアントからそのまま利用できる。

  • GET /health
  • GET /v1/models
  • POST /v1/chat/completions

"session_id" で会話を分離（省略時は共有デフォルトセッション）。

Docker / Linux Service

公式 Docker イメージは非 root ユーザー（UID 1000）で動作。bubblewrap（bwrap）によるサンドボックスが事前インストール済み。

docker compose run --rm nanobot-cli onboard
vim ~/.nanobot/config.json
docker compose up -d nanobot-gateway

systemd ユーザーサービスとして登録すればログアウト後も常時稼働可能（loginctl enable-linger）。

セキュリティ・サンドボックス

本番運用向け推奨設定:

{
  "tools": {
    "restrictToWorkspace": true,
    "exec": { "sandbox": "bwrap" }
  }
}

bwrap で exec をサンドボックス化（Linux のみ）。channels.*.allowFrom は空配列で全拒否がデフォルト。

チャット内コマンド: /new（会話リセット）, /stop（タスク停止）, /restart（再起動）, /status（状態表示）, /dream（記憶統合即時実行）, /dream-restore（メモリ復元）, /help。すべての対応チャンネルと CLI モードで共通動作。

visual-explainer

v0.6.3 — 複雑なターミナル出力を、自己完結 HTML の図・表・レビュー・スライドに変換する Agent Skill。

1 — 概要
何を解決するか

コーディングエージェントは、図や比較表を頼まれると ASCII アートや罫線テーブルへ寄りがち。visual-explainer はその出力を、ブラウザで読める スタイル付き HTML ページ に変換する。

対象はアーキテクチャ説明、diff レビュー、計画レビュー、プロジェクト復帰用 recap、ファクトチェック、データ表、スライドデッキ。生成物はビルド不要で、基本はブラウザだけで開ける。

自動発動: 4 行以上または 3 列以上の複雑な表をターミナルに出しそうな場面では、HTML テーブルとしてレンダリングする方針。
0.6.3
current version

Claude Code Marketplace / Pi / Codex に対応。Mermaid は zoom・pan・expand 付きの diagram-shell を正とする。

2 — 最新更新点
v0.6.3

bare Mermaid を禁止

<pre class="mermaid"> だけでは図が小さくなりやすく、ズーム・パンもない。新しい説明では templates/mermaid-flowchart.html の diagram-shell パターンを丸ごと使う。

v0.6.0

multi-diagram 対応

ページ内に複数の Mermaid 図を置けるよう、図ごとの状態をクロージャで持つ initDiagram(shell) 方式へ移行。ID 衝突を避け、タッチ操作や double-click fit も扱う。

v0.6.1

Pi のワンコマンド導入

install-pi.sh が追加され、スキル本体とプロンプトを ~/.pi/agent 配下へコピーし、{{skill_dir}} を実インストール先にパッチする。

v0.5.x

Marketplace 構造へ整理

Claude Code plugin marketplace 仕様に合わせ、スキルは plugins/visual-explainer/ に配置。Claude Code ではコマンドが /visual-explainer:diff-review のように名前空間付きになる。

3 — インストール
導入方法

Claude Code（マーケットプレース）

/plugin marketplace add nicobailon/visual-explainer
/plugin install visual-explainer@visual-explainer-marketplace

コマンドは /visual-explainer:コマンド名 のように名前空間付きになります。

Pi（ワンコマンド）

curl -fsSL https://raw.githubusercontent.com/nicobailon/visual-explainer/main/install-pi.sh | bash

ローカルに clone している場合は、リポジトリ直下で ./install-pi.sh を実行しても同じ導入フローになる。

git clone --depth 1 https://github.com/nicobailon/visual-explainer.git
cd visual-explainer && ./install-pi.sh

OpenAI Codex

git clone --depth 1 https://github.com/nicobailon/visual-explainer.git /tmp/visual-explainer
cp -r /tmp/visual-explainer/plugins/visual-explainer ~/.agents/skills/visual-explainer
mkdir -p ~/.codex/prompts
cp /tmp/visual-explainer/plugins/visual-explainer/commands/*.md ~/.codex/prompts/
rm -rf /tmp/visual-explainer

$visual-explainer で呼び出すか、プロンプトを入れた場合は /prompts:diff-review などを利用する。Codex の slash prompts は deprecated 扱いだが、まだ動作する。

4 — コマンド一覧
コマンド 説明
/generate-web-diagram 任意のトピックで HTML 図を生成
/generate-visual-plan 機能・拡張のビジュアル実装計画を生成
/generate-slides 雑誌品質のスライドデッキを生成
/diff-review アーキテクチャ比較とコードレビュー付きのビジュアル diff レビュー
/plan-review 計画とコードベースの比較とリスク評価
/project-recap プロジェクトに戻るためのメンタルモデルスナップショット
/fact-check ドキュメントの内容を実際のコードと照合して検証
/share HTML を Vercel にデプロイして公開 URL を取得
5 — 図解生成のルール
Mermaid は shell ごと使う

v0.6.3 の重要なルールは、Mermaid 図を裸の <pre class="mermaid"> で置かないこと。ズーム、パン、reset、expand、クリック展開まで含む diagram-shell 構造が canonical pattern になっている。

.diagram-shell
└── .mermaid-wrap
    ├── .zoom-controls
    └── .mermaid-viewport
        └── .mermaid-canvas
            └── script.diagram-source
Flow / Sequence Mermaid。5+ nodes や分岐は TD 優先。
Class / C4 Class は classDiagram。C4 は native C4 ではなく graph TD + subgraph。
Tables 本物の <table>。ASCII 表や CSS Grid 代用は避ける。
Large Architecture 15+ 要素は hybrid。小さな Mermaid overview と詳細カードを分ける。
6 — 構成

スキルは用途に応じてアプローチを切り替える。フローチャート・シーケンス・ER・Class・C4 は Mermaid、アーキテクチャ概要は CSS Grid、データは HTML テーブル、ダッシュボードは Chart.js が基本。

.claude-plugin/
├── plugin.json           ← マーケットプレース用
└── marketplace.json      ← プラグインカタログ
plugins/
└── visual-explainer/
    ├── .claude-plugin/
    │   └── plugin.json   ← プラグインマニフェスト
    ├── SKILL.md          ← ワークフローとデザイン原則
    ├── commands/         ← スラッシュコマンド
    ├── references/       ← 生成前にエージェントが参照
    │   ├── css-patterns.md
    │   ├── libraries.md
    │   ├── responsive-nav.md
    │   └── slide-patterns.md
    ├── templates/
    │   ├── architecture.html
    │   ├── mermaid-flowchart.html
    │   ├── data-table.html
    │   └── slide-deck.html
    └── scripts/
        └── share.sh      ← Vercel へデプロイ用

出力先: ~/.agent/diagrams/ファイル名.html に保存され、ブラウザで開く。docs-site 用に転用する場合は、このページのように docs/<name>.html へ配置する。

7 — 共有フロー
/share

/share は生成済み HTML を Vercel にデプロイし、ライブ URL と claim URL を返す共有用コマンド。内部では HTML を一時ディレクトリへ index.html としてコピーし、vercel-deploy skill で claimable deployment を作る。

bash {{skill_dir}}/scripts/share.sh ~/.agent/diagrams/my-diagram.html
8 — 制限
  • 表示にはブラウザが必要
  • OS のテーマ切り替え後、Mermaid の SVG を反映するにはページの再読み込みが必要
  • 結果はモデルの能力に依存する
  • 共有 URL は公開デプロイになるため、機密情報を含む HTML には向かない

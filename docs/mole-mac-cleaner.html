Mole

macOS ディープクリーン＆最適化オールインワン CLI — mo

1 — 概要
What is Mole

Mole（コマンド名: mo）は、CleanMyMac・AppCleaner・DaisyDisk・iStat Menus の機能を 1 つのバイナリに統合した macOS 向けシステムメンテナンス CLI。Homebrew または 1 行スクリプトでインストールでき、Go 製の TUI（bubbletea）により インタラクティブな操作が可能。

1単一バイナリ
Go実装言語
TUIインターフェース
MITライセンス

矢印キーや Vim キーバインド（h/j/k/l）でナビゲート可能。--dry-run フラグで安全にプレビューしてから実行できる。

2 — インストール
Homebrew（推奨） 
brew install mole
スクリプトインストール

バージョン指定オプション付き：

# 最新版
curl -fsSL https://raw.githubusercontent.com/tw93/mole/main/install.sh | bash

# バージョン指定
curl -fsSL https://raw.githubusercontent.com/tw93/mole/main/install.sh | bash -s -- -s 1.17.0
起動 
mo                 # インタラクティブメニュー
mo clean           # ディープクリーン
mo status          # リアルタイムシステム監視

4 — 機能詳細
ディープクリーン

ユーザーキャッシュ・ブラウザキャッシュ・開発ツール（Xcode, Node.js, npm）・システムログ・アプリ固有キャッシュ・ゴミ箱をまとめて削除。

$ mo clean
  ✓ User app cache              45.2GB
  ✓ Browser cache               10.5GB
  ✓ Developer tools             23.3GB
  ✓ System logs                  3.8GB
============================
Space freed: 95.5GB
スマートアンインストーラー

アプリ本体の削除に加え、Application Support・Caches・Preferences・Logs・WebKit storage・Extensions・Launch daemons など 12 箇所以上 の関連ファイルを自動的に探して除去。

ディスクアナライザー

インタラクティブなディレクトリブラウザ。ファイルサイズ・最終使用日時を確認しながらゴミ箱移動（Finder 経由）。mo analyze /Volumes で外部ドライブも対象可。

$ mo analyze
 ▶ ███████████  48.2%  Library   75.4GB  >6mo
   ████████░░   22.1%  Downloads 34.6GB
   ↑↓←→ Navigate | ⌫ Delete | Q Quit
ライブシステム監視

CPU・GPU・メモリ・ディスク・ネットワーク・プロセスをリアルタイム表示。ヘルススコア（0–100）付きのダッシュボード。--json やパイプで機械可読 JSON 出力も対応。

$ mo status --json | jq '.health_score'
92
プロジェクト成果物クリーン

node_modules・target・build・dist・venv など複数言語の古いビルド成果物を検索・削除。7 日以内の最近のプロジェクトはデフォルトで非選択。

クイックランチャー連携

Raycast / Alfred 向けスクリプトコマンドを 1 行でセットアップ。Mole Clean・Mole Status 等 5 コマンドをランチャーから即起動できる。

curl -fsSL https://raw.githubusercontent.com/tw93/Mole/main/scripts/setup-quick-launchers.sh | bash

6 — 安全設計
安全ファースト原則

Mole はシステムに対する破壊的操作を含むため、以下の安全設計を採用している：

  • パス検証 — 操作対象パスを厳格に検証し、保護ディレクトリへの誤操作を防ぐ
  • 保護ディレクトリルール — システム領域への削除操作を禁止
  • 保守的なクリーン境界 — 不確実な場合は削除しない
  • 高リスク操作の明示的確認 — 重要な操作は強い確認を求める
  • 不確実性への対応 — リスクが高い・不明な場合はスキップ/拒否
破壊的操作のリスト: clean・uninstall・purge・installer・remove は削除操作を伴う。必ず --dry-run でプレビューし、必要に応じて --debug を加えて詳細ログを確認してから実行すること。操作ログは ~/Library/Logs/mole/operations.log に記録される。無効にするには MO_NO_OPLOG=1 を設定。
analyze の安全性

mo analyze はファイルを直接削除せず、Finder 経由でゴミ箱へ移動する。即時削除の他コマンドと異なり、手動でアドホッククリーンを行う際はこちらが安全。

# 文体・トーン（docs-site 解説ページ）

## 目的

依頼された読み心地（丁寧さ・歓迎感・技術寄りなど）を、**本文の書き方と具体的手順**で出す。見出しやカード文言で「雰囲気のラベル」だけを貼らない。

## 実行すること

- 見出しは **何のセクションか** が一目で分かる名詞・手順ベースにする（例: 概要、処理の流れ、テンプレ対応、参照）。
- 短い文・能動態・固有名とファイル名を具体的に書く。読者に必要な操作をそのまま書く。
- `index.html` の **カードタイトル**を、ページの `<title>` および主題（プロダクト名・スキル名・リポジトリ）と揃える。キャッチコピーだけのカードタイトルにしない。
- メタ description / og:description は、煽りや招待状調より **内容の要約**を優先する。

## 避けること

- セクション名やリードで、トーンを**単語として宣言**する（例: 日本語で「おもてなし」「心遣い」を見出しに据える、英語で "Hospitality checklist" だけを飾りにする）。
- 「ようこそ」「やさしくご案内」「遠慮なく〜」など、**定型の歓迎文**で帯を埋める。
- 体験や価値を短いスローガンだけにし、**中身のない見出し**にする。

## ユーザーが違和感を指摘したら

1. 宣言系の見出し・リード・カード文言を列挙する。
2. 各ブロックを **事実・手順・制約・参照**のいずれかに置き換える。
3. 必要ならユーザーが添付する **stop-slop** スキルに従い、副詞・受動態・引き伸ばし表現を削る。

# 出力テンプレート

> **目的**: コミット計画と実行結果の一貫したフォーマットを提供する

## コミット計画テンプレート

複数コミットに分割する場合の計画出力形式:

```markdown
## コミット計画

| # | カテゴリ | 概要 | ファイル | 行数 |
|---|---------|------|----------|------|
| 1 | style | フォーマット統一 | file1.ts | +5/-3 |
| 2 | refactor | 関数分離 | file1.ts, file2.ts | +20/-15 |
| 3 | feat | 新機能追加 | file3.ts | +50/-0 |

---

### コミット 1/3: [style] フォーマット統一

**対象ファイル**: 
- file1.ts

**変更内容**: 
- インデントを統一
- 不要な空行を削除

**評価**:
| 観点 | スコア | 理由 |
|------|--------|------|
| 原子性 | 5/5 | フォーマットのみ |
| レビュー容易性 | 5/5 | 差分8行 |
| 履歴可読性 | 5/5 | 意図明確 |
| **総合** | **15/15** | ✅ 合格 |

**コミットメッセージ**:
```
style: file1.tsのコードフォーマットを統一する
```

---

### コミット 2/3: [refactor] 関数分離
...（同様の形式）

---

### コミット 3/3: [feat] 新機能追加
...（同様の形式）
```

## 単一コミット実行テンプレート

単純なケースでの出力形式:

### 🅶 Git 環境

```markdown

**カテゴリ**: feat

**対象ファイル**: 
- src/components/UserList.tsx
- src/hooks/useUsers.ts

**コミットメッセージ**:
```

なぜ変えたか:
- 大量のユーザーから特定ユーザーを探しやすくする

```
実行しますか？ [Y/n]
```

## エラー報告テンプレート

### コミット失敗

```markdown
## コミット失敗 ❌

**原因**: pre-commit hookでlintエラー

**エラー詳細**:
```
error: src/components/UserList.tsx
  Line 42: 'unused' is defined but never used
```

**推奨アクション**:
1. lintエラーを修正
2. 再度コミットを試行
```

### 分割不可

```markdown
## 分割不可 ⚠️

**理由**: 変更間の依存関係が強く、分割するとビルドが通らない

**詳細**:
- file1.ts で新しい型を定義
- file2.ts でその型を即座に使用
- 分割すると file2.ts が型エラーになる

**推奨アクション**:
- 1コミットとして実行（原子性を優先）
```


```
【利用可能なGitButler操作】

- `but rub <file-id> <commit-id>`: 変更をコミットにアメンド
- `but rub <commit1> <commit2>`: コミットのスカッシュ
- `but rub <commit-id> 00`: コミット取り消し
- `but rub <file-id> <branch-id>`: ファイルをブランチに割当

【期待する出力】
```

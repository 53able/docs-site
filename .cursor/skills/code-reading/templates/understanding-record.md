# 理解記録テンプレート

## 基本テンプレート（モジュール単位）

```markdown
## [モジュール名]

**責務**: [1文で]
**入力**: [主要な入力]
**出力**: [主要な出力]
**依存**: [主要な依存先]
**注意点**: [ハマりポイント、暗黙の前提]
```

### 使用例

```markdown
## TokenValidator

**責務**: JWTトークンの検証と有効期限チェック
**入力**: token: string, options?: ValidateOptions
**出力**: ValidatedToken | ValidationError
**依存**: JwtService, ConfigService
**注意点**: 環境変数JWT_SECRETが未設定だと起動時にクラッシュ
```

## 拡張テンプレート（詳細な理解が必要な場合）

```markdown
## [モジュール名]

### 概要
**責務**: [1文で]
**設計意図**: [なぜこの設計か]

### インターフェイス
**入力**: 
- [パラメータ1]: [説明]
- [パラメータ2]: [説明]

**出力**: 
- 成功時: [型と内容]
- 失敗時: [エラー型と条件]

### 依存関係
```
[このモジュール] --uses--> [依存先1]
                --uses--> [依存先2]
                <--used by-- [利用元]
```

### 重要な実装詳細
- [ポイント1]
- [ポイント2]

### 注意点・ハマりポイント
- [注意1]
- [注意2]

### 未解決・要確認
- [ ] [疑問点1]
- [ ] [疑問点2]
```

## セッション記録テンプレート

```markdown
# コードリーディングセッション: [日付]

## ゴール
[1文で明文化]

## 対象範囲
- [ファイル/ディレクトリ1]
- [ファイル/ディレクトリ2]

## 理解したモジュール
[基本テンプレートを並べる]

## 依存関係図
```
[図をここに]
```

## 発見したパターン
- [パターン1]
- [パターン2]

## 未解決事項
- [ ] [要確認1]
- [ ] [要確認2]

## 次のアクション
[1つだけ明確に]
```

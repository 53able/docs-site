# Layer 3: 管理 (Management)

> 理解した情報をどう記録・参照可能にするか

## 3-1. 情報の階層化

### 短期記憶（セッション中）

- 現在のゴール
- 現在探索中のパス
- 直近で読んだ3-5ファイル

### 中期記憶（タスク完了まで）

- 圧縮テンプレートで記録したモジュール理解
- 依存関係図
- 発見したパターンやハマりポイント

### 長期記憶（プロジェクト知識）

- README、設計ドキュメント
- 過去のPRやIssueの議論
- チームの暗黙知（コメントで残す）

## 3-2. 記録戦略

**原則**: 記憶に頼らず、いつでも参照できる形で残す

```
DO:
├─ 図をテキストで残す（Mermaid、ASCII art）
├─ 理解を短いメモで言語化する
└─ 疑問点や未解決事項を明示的にリスト化

DON'T:
├─ 頭の中だけで関係性を管理する
└─ 「後で思い出せるだろう」と記録を省略する
```

## 3-3. 記録のタイミング

| タイミング | 記録内容 |
|------------|----------|
| モジュール理解完了時 | 圧縮テンプレート |
| 依存関係発見時 | 依存グラフに追記 |
| ハマりポイント発見時 | 注意点として記録 |
| セッション終了時 | 次のアクションを1つ |

## 3-4. 参照可能性の確保

### 図解のパターン

**依存関係図**:
```
[Module A] --uses--> [Module B]
           --uses--> [Module C]
```

**データフロー図**:
```
Input --> [Validate] --> [Transform] --> [Save] --> Output
                            ↓
                      [ErrorHandler]
```

**責務境界図**:
```
┌─────────────────────────────────────┐
│ AuthService                         │
│ ├─ 担当: 認証・認可                 │
│ └─ 担当外: ユーザー管理、永続化     │
└─────────────────────────────────────┘
```

## 3-5. 未解決事項の管理

```markdown
## 未解決・要確認

- [ ] TokenValidatorのキャッシュ戦略が不明
- [ ] エラー時のリトライロジックの有無
- [ ] 環境変数 `JWT_SECRET` のローテーション方法
```

**原則**: 「わからない」も重要な情報。明示的に記録する。
